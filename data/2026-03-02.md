### [1] [Speak Now: Safe Actor Programming with Multiparty Session Types](https://arxiv.org/abs/2602.24054)
*Simon Fowler,Raymond Hu*

Main category: cs.PL

TL;DR: Maty是首个支持静态多方会话类型和完整actor能力的actor语言设计，结合了会话类型错误预防机制与actor语言的可扩展性和容错性。


<details>
  <summary>Details</summary>
Motivation: Erlang和Elixir等actor语言广泛用于实现可扩展可靠的分布式应用，但actor通信模式的非正式规范性质使系统容易受到通信不匹配和死锁等代价高昂的错误影响。多方会话类型(MPSTs)可以在开发早期排除通信错误，但actor通信的多发送者、单接收者特性使其难以受益于会话类型。

Method: 通过流敏感效应系统强制执行会话类型，结合事件驱动编程风格和一等消息处理器。使用MPSTs保证通信安全性：进程永远不会发送或接收意外消息，会话也不会因为actor等待永远不会发送的消息而卡住。扩展Maty支持Erlang风格的监督和级联故障。

Result: 实现了Maty的Scala版本，使用API生成方法。通过实现广泛使用的Savina actor基准套件的代表性样本、行业提供的工厂场景和聊天服务器，展示了模型的表达能力。

Conclusion: Maty首次将静态多方会话类型与完整actor能力相结合，为actor语言提供了会话类型的错误预防机制，同时保持了actor语言的可扩展性和容错性优势。

Abstract: Actor languages such as Erlang and Elixir are widely used for implementing scalable and reliable distributed applications, but the informally-specified nature of actor communication patterns leaves systems vulnerable to costly errors such as communication mismatches and deadlocks. Multiparty session types (MPSTs) rule out communication errors early in the development process, but until now, the many-sender, single-receiver nature of actor communication has made it difficult for actor languages to benefit from session types.
  This paper introduces Maty, the first actor language design supporting both static multiparty session typing and the full power of actors taking part in multiple sessions. Maty therefore combines the error prevention mechanism of session types with the scalability and fault tolerance of actor languages. Our main insight is to enforce session typing through a flow-sensitive effect system, combined with an event-driven programming style and first-class message handlers. Using MPSTs allows us to guarantee communication safety: a process will never send or receive an unexpected message, nor will a session get stuck because an actor is waiting for a message that will never be sent. We extend Maty to support Erlang-style supervision and cascading failure, and show that this preserves Maty's strong metatheory. We implement Maty in Scala using an API generation approach, and demonstrate the expressiveness of our model by implementing a representative sample of the widely-used Savina actor benchmark suite; an industry-supplied factory scenario; and a chat server.

### [2] [CIll: CTI-Guided Invariant Generation via LLMs for Model Checking](https://arxiv.org/abs/2602.23389)
*Yuheng Su,Tianjun Bu,Qiusong Yang,Yiwei Ci,Enyuan Tian*

Main category: cs.LO

TL;DR: CIll是一个结合LLM和传统模型检查技术的框架，通过CTI引导LLM合成归纳不变量，提高模型检查的效率和可扩展性。


<details>
  <summary>Details</summary>
Motivation: 传统IC3方法基于CNF表示，对于某些设计可能需要大量子句，影响可扩展性。需要更高效自动生成归纳不变量的方法。

Method: CIll框架交替进行正确性检查和归纳性检查，使用BMC验证不变量的有界正确性，当归纳性失败时提取CTI提供给LLM，LLM基于设计和CTI提出新不变量，结合IC3和K归纳作为补充引擎，采用局部证明和重用IC3学习的不变量来优化性能。

Result: CIll在RISCV-Formal框架中证明了完全合规性，在NERV和PicoRV32中证明了所有非M指令的完全准确性，M扩展针对RISCV-Formal提供的RVFI ALTOPS替代语义进行了验证。

Conclusion: CIll通过LLM引导的CTI分析有效合成归纳不变量，解决了传统方法的可扩展性问题，在RISC-V验证中表现出色。

Abstract: Inductive invariants are crucial in model checking, yet generating effective inductive invariants automatically and efficiently remains challenging. A common approach is to iteratively analyze counterexamples to induction (CTIs) and derive invariants that rule them out, as in IC3. However, IC3's clause-based learning is limited to a CNF representation. For some designs, the resulting invariants may require a large number of clauses, which hurts scalability. We present CIll, a CTI-guided framework that leverages LLMs to synthesize invariants for model checking. CIll alternates between (bounded) correctness checking and inductiveness checking for the generated invariants. In correctness checking, CIll uses BMC to validate whether the generated invariants hold on reachable states within a given bound. In inductiveness checking, CIll checks whether the generated invariants, together with the target property, become inductive under the accumulated strengthening. When inductiveness fails, CIll extracts CTIs and provides them to the LLM. The LLM inspects the design and the CTI to propose new invariants that invalidate the CTIs. The proposed invariants are then re-validated through correctness and inductiveness checks, and the loop continues until the original property strengthened by the generated invariants becomes inductive. CIll also employs IC3 to work with the LLM for automatically discovering invariants, and uses K-Induction as a complementary engine. To improve performance, CIll applies local proof and reuses invariants learned by IC3, reducing redundant search and accelerating convergence. In our evaluation, CIll proved full compliance within RISCV-Formal framework and full accuracy of all non-M instructions in NERV and PicoRV32, whereas M extensions are proved against the RVFI ALTOPS substitute semantics provided by RISCV-Formal.

### [3] [Zero-Incoherence Capacity of Interactive Encoding Systems: Achievability, Converse, and Side Information Bounds](https://arxiv.org/abs/2602.23520)
*Tristan Simas*

Main category: cs.IT

TL;DR: 论文提出了零不相干容量概念，用于交互式多位置编码系统，证明了精确容量定理C₀=1，给出了紧致的边信息下界，并展示了速率-复杂度分离现象。


<details>
  <summary>Details</summary>
Motivation: 研究多位置编码系统中如何保证复制编码之间零概率不一致的问题，为分布式系统、复制数据一致性等场景提供理论基础。

Method: 将编码位置建模为多终端信源编码模型，使用信息论方法推导容量定理，通过混淆性/不相干图连接形式化分析，并用Lean 4进行机器验证。

Result: 证明了零不相干容量C₀=1，k路不相干需要≥log₂k比特的边信息，在容量处修改成本为O(1)而超过容量时为Ω(n)。

Conclusion: 只有完全推导（单一独立源）才能实现零不相干，论文提供了紧凑的信息论结果和构造性实例，核心定理已通过机器验证。

Abstract: We introduce the zero-incoherence capacity for interactive multi-location encoding systems: the maximum encoding rate that guarantees exactly zero probability of disagreement among replicated encodings. Our main information-theoretic results are compact and self-contained: an exact capacity theorem ($C_0=1$), a tight side-information lower bound for resolution ($\geq\log_2 k$ bits for $k$-way incoherence), and a rate--complexity separation (modification cost $O(1)$ at capacity vs $Ω(n)$ above).
  The paper frames encoding locations as terminals in a multi-terminal source-coding model. Derivation (automatic deterministic dependence) is interpreted as perfect correlation that reduces effective rate; only complete derivation (one independent source) achieves zero incoherence. We give concise achievability and converse proofs in IT style, formalize the confusability/incoherence graph connection, and present an explicit mutual-information argument for the side-information bound.
  Theoretical contributions are supplemented by constructive instantiations (programming-language patterns and a software case study). For TIT submission we move detailed language evaluation, extended code examples, and the full Lean proof corpus to supplementary material; the main text contains brief instantiations only. Core theorems (capacity, realizability, bounds) are machine-checked in Lean 4; entropy arguments apply standard Fano-inequality techniques.

### [4] [Mixed Choice in Asynchronous Multiparty Session Types](https://arxiv.org/abs/2602.23927)
*Laura Bocchi,Raymond Hu,Adriana Laura Voinea,Simon Thompson*

Main category: cs.DC

TL;DR: 提出支持异步混合选择的多方会话类型框架，确保分布式参与者最终达成一致状态，并实现Erlang工具链验证


<details>
  <summary>Details</summary>
Motivation: 现有的多方会话类型框架在处理异步混合选择时存在限制，需要一种能够处理分布式参与者间暂时不一致但最终确保一致性的方法

Method: 提出核心构造处理混合选择，允许分布式参与者间的暂时状态不一致，但确保最终一致性；建立全局类型与分布式局部类型投影间的操作对应关系证明正确性；实现Erlang/OTP工具链

Result: 证明了系统的正确性（进展属性和操作对应性），实现了实用的工具链，并通过重新实现RabbitMQ的amqp_client部分验证了框架有效性

Conclusion: 成功开发了支持异步混合选择的多方会话类型框架，提供了理论保证和实用工具，能够有效处理分布式协议中的复杂交互模式

Abstract: We present a multiparty session type (MST) framework with asynchronous mixed choice (MC). We propose a core construct for MC that allows transient inconsistencies in protocol state between distributed participants, but ensures all participants can always eventually reach a mutually consistent state. We prove the correctness of our system by establishing a progress property and an operational correspondence between global types and distributed local type projections. Based on our theory, we implement a practical toolchain for specifying and validating asynchronous MST protocols featuring MC, and programming compliant gen_statem processes in Erlang/OTP. We test our framework by using our toolchain to specify and reimplement part of the amqp_client of the RabbitMQ broker for Erlang.

</details>
