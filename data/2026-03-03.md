### [1] [Tracking Capabilities for Safer Agents](https://arxiv.org/abs/2603.00991)
*Martin Odersky,Yaoyu Zhao,Yichen Xu,Oliver Bračevac,Cao Nguyen Pham*

Main category: cs.AI

TL;DR: 提出使用Scala 3类型系统构建AI代理安全约束框架，通过能力追踪防止信息泄露和恶意副作用


<details>
  <summary>Details</summary>
Motivation: AI代理通过工具调用与现实世界交互时存在安全挑战：可能泄露隐私信息、造成意外副作用或被提示注入攻击操纵

Method: 将代理置于基于编程语言的"安全约束"中：代理以Scala 3代码表达意图，利用捕获检查追踪能力，类型系统静态追踪能力变量，实现细粒度访问控制和局部纯度

Result: 实验表明代理能够生成能力安全的代码，任务性能无显著损失，类型系统可靠防止信息泄露和恶意副作用等不安全行为

Conclusion: 通过利用具有追踪能力的强类型系统，可以构建可扩展的代理安全约束框架，有效解决AI代理的安全挑战

Abstract: AI agents that interact with the real world through tool calls pose fundamental safety challenges: agents might leak private information, cause unintended side effects, or be manipulated through prompt injection. To address these challenges, we propose to put the agent in a programming-language-based "safety harness": instead of calling tools directly, agents express their intentions as code in a capability-safe language: Scala 3 with capture checking. Capabilities are program variables that regulate access to effects and resources of interest. Scala's type system tracks capabilities statically, providing fine-grained control over what an agent can do. In particular, it enables local purity, the ability to enforce that sub-computations are side-effect-free, preventing information leakage when agents process classified data. We demonstrate that extensible agent safety harnesses can be built by leveraging a strong type system with tracked capabilities. Our experiments show that agents can generate capability-safe code with no significant loss in task performance, while the type system reliably prevents unsafe behaviors such as information leakage and malicious side effects.

### [2] [Bentō: Optimizing Persistent Memory Programs](https://arxiv.org/abs/2603.01889)
*Sebastião Amaro,João Gonçalves,Miguel Matos*

Main category: cs.ET

TL;DR: Bentō是一个自动化的二进制重写工具，通过优化持久内存程序中的持久性指令组合，在不改变崩溃一致性语义的前提下提升性能达15%


<details>
  <summary>Details</summary>
Motivation: 持久内存(PM)编程需要显式使用flush和fence指令来保证崩溃一致性，但程序员往往过度保守地使用这些指令，导致不必要的性能损失

Method: 研究不同持久性指令组合对PM程序的影响，发现特定组合能在保持崩溃一致性语义的同时提升性能，基于此开发了Bentō——一个自动的黑盒二进制重写器

Result: Bentō能够将现有PM程序的性能提升高达15%，且只需最小的程序员努力

Conclusion: 通过自动优化持久性指令组合，可以在不改变程序语义的情况下显著提升PM程序性能，Bentō为此提供了一种实用的解决方案

Abstract: Persistent Memory (PM) is a new storage technology thatbrings high performance, byte addressability, and persistency for a lesser cost than DRAM. Due to cache volatility and store reordering, developers must use explicit instructions (e.g.: flush and fence) to guarantee that the application state remains consistent upon crashes. This is difficult to get right and, in fact, several tools have been created to detect bugs in PM programs. To overcome this difficulty, programmers tend to be overly conservative, for instance, by enforcing unnecessary ordering constraints, which partially forfeits the performance benefits of using PM. In this paper, we study the impact that different combinations of persistency instructions have in several PM programs and found that a specific combination can lead to performance improvements while preserving the original crash-consistency semantics. Based on these results we developed Bentō an automatic and black-box binary rewriter that can boost the performance of existing PM programs by up to 15% with minimal programmer effort.

</details>

### [3] [Agentic Code Reasoning](https://arxiv.org/abs/2603.01896)
*Shubham Ugare,Satish Chandra*

Main category: cs.SE

TL;DR: 论文提出半形式推理方法，通过结构化提示让LLM代理在不执行代码的情况下进行代码语义分析，在补丁等价验证、故障定位和代码问答任务上显著提升准确性。


<details>
  <summary>Details</summary>
Motivation: 研究LLM代理能否在不执行代码的情况下探索代码库并进行代码语义推理，这种能力被称为代理代码推理。当前的无结构思维链方法存在跳过案例或做出无支持声明的风险。

Method: 提出半形式推理：一种结构化提示方法，要求代理构建明确的前提、追踪执行路径并推导形式结论。与无结构思维链不同，半形式推理作为证书，确保代理不能跳过案例或做出无支持声明。

Result: 在三个任务上评估：补丁等价验证准确率从78%提升到88%（精选示例），在真实世界代理生成补丁上达到93%；代码问答在RubberDuckBench上达到87%准确率；故障定位在Defects4J上Top-5准确率比标准推理提升5个百分点。

Conclusion: 结构化代理推理使得无需执行代码即可进行有意义的语义代码分析成为可能，为RL训练管道、代码审查和静态程序分析等实际应用开辟了道路。

Abstract: Can LLM agents explore codebases and reason about code semantics without executing the code? We study this capability, which we call agentic code reasoning, and introduce semi-formal reasoning: a structured prompting methodology that requires agents to construct explicit premises, trace execution paths, and derive formal conclusions. Unlike unstructured chain-of-thought, semi-formal reasoning acts as a certificate: the agent cannot skip cases or make unsupported claims. We evaluate across three tasks (patch equivalence verification, fault localization, and code question answering) and show that semi-formal reasoning consistently improves accuracy on all of them. For patch equivalence, accuracy improves from 78% to 88% on curated examples and reaches 93% on real-world agent-generated patches, approaching the reliability needed for execution-free RL reward signals. For code question answering on RubberDuckBench Mohammad et al. (2026), semi-formal reasoning achieves 87% accuracy. For fault localization on Defects4J Just et al. (2014), semi-formal reasoning improves Top-5 accuracy by 5 percentage points over standard reasoning. These results demonstrate that structured agentic reasoning enables meaningful semantic code analysis without execution, opening practical applications in RL training pipelines, code review, and static program analysis.

</details>

### [4] [OBASE: Object-Based Address-Space Engineering to Improve Memory Tiering](https://arxiv.org/abs/2603.00378)
*Vinay Banakar,Suli Yang,Kan Wu,Andrea C. Arpaci-Dusseau,Remzi H. Arpaci-Dusseau,Kimberly Keeton*

Main category: cs.OS

TL;DR: OBASE系统通过地址空间工程解决内存分层中的热碎片问题，将热对象和冷对象分别聚类到不同页面，使现有内存分层机制能有效工作，显著减少内存占用。


<details>
  <summary>Details</summary>
Motivation: 当前数据中心过度配置DRAM的根本原因是热碎片问题：内存分配器按大小而非访问模式放置对象，导致热对象和冷对象在同一页面中交错分布。单个热对象会使其所在页面被标记为活跃，从而将周围的冷数据也困在昂贵的DRAM中。分析显示Google生产工作负载中活跃页面中高达97%的字节是冷的且无法回收。

Method: 提出地址空间工程方法：动态重组虚拟内存，使热对象聚类到统一的热页面，冷对象聚类到统一的冷页面。开发OBASE系统，这是一个用于非托管语言的编译器-运行时系统，作为页面感知操作系统后端的对象感知前端。OBASE通过轻量级指针插桩跟踪访问，并使用无锁协议在运行时迁移对象，确保并发安全性。

Result: 通过重组地址空间，OBASE使未修改的后端（kswapd、TMO、TPP、Memtis）能够有效分层内存。在十个并发数据结构、六个后端以及来自Meta和Twitter的生产跟踪中，OBASE将页面利用率提高了2-4倍，内存占用减少了高达70%，仅带来2-5%的开销。

Conclusion: OBASE通过地址空间工程有效解决了内存分层中的热碎片问题，显著提高了内存效率，使现有内存分层机制能够充分发挥作用，为数据中心内存管理提供了实用解决方案。

Abstract: Hardware and OS mechanisms for memory tiering are widely deployed, yet datacenters still overprovision DRAM. The root cause is hotness fragmentation: allocators place objects by size rather than access pattern, so hot and cold objects become interleaved within the same pages. A single hot object marks its page as active, trapping surrounding cold data in expensive DRAM. Our analysis of Google production workloads shows that up to 97% of the bytes in active pages are cold and unreclaimable. We propose address-space engineering: dynamically reorganizing virtual memory so that hot objects cluster into uniformly hot pages and cold objects into uniformly cold pages. We present OBASE, a compiler-runtime system for unmanaged languages that serves as an object-aware frontend for page-aware OS backends. OBASE tracks accesses via lightweight pointer instrumentation and migrates objects at runtime using a lock-free protocol that is safe under concurrency. By reorganizing the address space, OBASE enables unmodified backends (kswapd, TMO, TPP, Memtis) to tier memory effectively. Across ten concurrent data structures, six backends, and production traces from Meta and Twitter, OBASE improves page utilization by 2-4x and reduces memory footprint by up to 70%, with only 2-5% overhead.

</details>
