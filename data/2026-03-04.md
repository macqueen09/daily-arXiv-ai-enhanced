*Ethan Chu,Yiyang Guo,Jan Hoffmann*

Main category: cs.PL

TL;DR: 首个支持异常/效应处理程序非本地控制转移的自动资源边界分析，扩展了基于类型的自动摊销资源分析（AARA），在Standard ML中实现并验证了低开销。


<details>
  <summary>Details</summary>
Motivation: 现有自动资源边界分析技术对大多数程序和语言特性有效，但异常或效应处理程序所需的非本地控制转移一直是个挑战。需要支持这类控制流模式的分析方法。

Method: 扩展基于类型的自动摊销资源分析（AARA），采用摊销分析潜力法。为包含列表和线性潜力函数的简单函数式语言设计新类型系统，支持异常和效应。提出新颖的语法类型安全性定理，确保推导边界相对于基于堆栈的抽象机器的正确性。

Result: 实现了首个支持异常/效应处理程序非本地控制转移的自动资源边界分析。实验评估表明，新方法能分析现有方法无法处理的程序，且支持异常和效应处理程序的开销很低。

Conclusion: 成功开发了首个支持异常和效应处理程序非本地控制转移的自动资源边界分析系统，扩展了AARA框架，为复杂控制流程序的资源分析提供了有效解决方案。

Abstract: There exist many techniques for automatically deriving parametric resource (or cost) bounds by analyzing the source code of a program. These techniques work effectively for a large class of programs and language features. However, non-local transfer of control as needed for exception or effect handlers has remained a challenge.
  This paper presents the first automatic resource bound analysis that supports non-local control transfer between exceptions or effects and their handlers. The analysis is an extension of type-based automatic amortized resource analysis (AARA), which automates the potential method of amortized analysis. It is presented for a simple functional language with lists and linear potential functions. However, the ideas are directly applicable to richer settings and implemented for Standard ML and polynomial potential functions.
  Apart from the new type system for exceptions and effects, a main contribution is a novel syntactic type-soundness theorem that establishes the correctness of the derived bounds with respect to a stack-based abstract machine. An experimental evaluation shows that the new analysis is capable of analyzing programs that cannot be analyzed by existing methods and that the efficiency overhead of supporting exception and effect handlers is low.

### [2] [Highly Incremental: A Simple Programmatic Approach for Many Objectives (Extended Version)](https://arxiv.org/abs/2603.02405)
*Philipp Schröer,Joost-Pieter Katoen*

Main category: cs.PL

TL;DR: 提出一种统一的程序化方法，通过添加奖励语句和程序变换，使用单调函数处理累积奖励，从而支持分析概率程序中的多种目标（高阶矩、阈值概率、预算超额期望等）。


<details>
  <summary>Details</summary>
Motivation: 现有方法需要针对概率程序的不同分析目标（如高阶矩、阈值概率、预算超额期望等）开发专门的验证技术，缺乏统一框架。本文旨在提供一种通用的程序化方法，能够统一处理这些多样的分析目标。

Method: 1. 在概率程序语言中添加奖励语句；2. 定义程序变换，将单调函数应用于程序的累积奖励；3. 关键创新：利用奖励的增量差异进行变换；4. 将各种分析目标统一转化为概率wp推理问题。

Result: 该方法能够表达和分析多种目标：高阶矩、奖励的阈值概率、预算超额期望、矩生成函数等。通过Caesar演绎验证器实现自动化验证，并在多个示例中成功应用了该变换方法。

Conclusion: 提出了一种简单优雅的统一框架，通过程序变换将多种概率程序分析目标转化为标准的概率wp推理问题，实现了"一法通，万法通"的分析能力，并通过自动化工具验证了其有效性。

Abstract: We present a one-fits-all programmatic approach to reason about a plethora of objectives on probabilistic programs. The first ingredient is to add a reward-statement to the language. We then define a program transformation applying a monotone function to the cumulative reward of the program. The key idea is that this transformation uses incremental differences in the reward. This simple, elegant approach enables to express e.g., higher moments, threshold probabilities of rewards, the expected excess over a budget, and moment-generating functions. All these objectives can now be analyzed using a single existing approach: probabilistic wp-reasoning. We automated verification using the Caesar deductive verifier and report on the application of the transformation to some examples.

Main category: cs.PL

TL;DR: 提出短竞争检测问题，开发高效监控框架，针对happens-before和sync-preserving竞争提供算法，在资源使用和预测能力间取得平衡


<details>
  <summary>Details</summary>

Abstract: We introduce and study the problem of detecting short races in an observed trace. Specifically, for a race type $R$, given a trace $σ$ and window size $w$, the task is to determine whether there exists an $R$-race $(e_1, e_2)$ in $σ$ such that the subtrace starting with $e_1$ and ending with $e_2$ contains at most $w$ events. We present a monitoring framework for short-race prediction and instantiate the framework for happens-before and sync-preserving races, yielding efficient detection algorithms. Our happens-before algorithm runs in the same time as FastTrack but uses space that scales with $\log w$ as opposed to $\log |σ|$. For sync-preserving races, our algorithm runs faster and consumes significantly less space than SyncP. Our experiments validate the effectiveness of these short-race detection algorithms: they run more efficiently, use less memory, and detect significantly more races under the same budget, offering a reasonable balance between resource usage and predictive power.

Method: SpecLoop采用代理框架，首先生成候选规格说明，然后从这些规格说明重构RTL；使用形式等价检查工具验证重构RTL与原始设计的功能一致性。当检测到不匹配时，将反例反馈给系统以迭代优化规格说明，直到证明等价性或无法进一步改进。

### [7] [CuTe Layout Representation and Algebra](https://arxiv.org/abs/2603.02298)
*Cris Cecka*

Main category: cs.MS

TL;DR: CuTe是一个用于表示和操作张量的数学规范，通过层次化布局表示和丰富的布局代数运算，解决了现代硬件张量指令的复杂数据布局管理问题。


<details>
  <summary>Details</summary>
Motivation: 现代高性能计算和深度学习架构越来越多地采用专门的张量指令（如张量核心矩阵乘法和硬件优化的多维数据复制操作），这些指令规定了固定且复杂的数据布局，需要在整个执行流水线中正确传播以确保正确性和最佳性能。

Method: CuTe提出了两个关键创新：1）层次化布局表示，直接扩展传统的平面形状和平面步幅张量表示，能够表示现代硬件指令所需的复杂映射；2）丰富的布局代数运算（包括连接、合并、组合、补集、除法、平铺和反转），支持复杂的布局操作、推导、验证和静态分析。

Result: CuTe已成功部署在生产系统中，成为NVIDIA CUTLASS库的基础，并支持CuTe DSL等相关工作。相比传统方法，CuTe的抽象显著帮助了软件开发，促进了架构规定布局的编译时验证，实现了可泛化到广泛应用的算法原语，并简洁表达了现代专门张量指令所需的平铺和分区模式。

Conclusion: CuTe提供了一个管理GPU内核中数据布局和线程安排的框架，其布局代数支持强大的编译时推理和通用张量变换表达，为现代硬件张量指令的复杂布局管理提供了有效的数学规范解决方案。

These instructions prescribe fixed, often complex data layouts that must be correctly propagated through the entire execution pipeline to ensure both correctness and optimal performance. We present CuTe, a novel mathematical specification for representing and manipulating tensors. CuTe introduces two key innovations: (1) a hierarchical layout representation that directly extends traditional flat-shape and flat-stride tensor representations, enabling the representation of complex mappings required by modern hardware instructions, and (2) a rich algebra of layout operations -- including concatenation, coalescence, composition, complementation, division, tiling, and inversion -- that enables sophisticated layout manipulation, derivation, verification, and static analysis. CuTe layouts provide a framework for managing both data layouts and thread arrangements in GPU kernels, while the layout algebra enables powerful compile-time reasoning about layout properties and the expression of generic tensor transformations.
  In this work, we demonstrate that CuTe's abstractions significantly aid software development compared to traditional approaches, promote compile-time verification of architecturally prescribed layouts, facilitate the implementation of algorithmic primitives that generalize to a wide range of applications, and enable the concise expression of tiling and partitioning patterns required by modern specialized tensor instructions.
  CuTe has been successfully deployed in production systems, forming the foundation of NVIDIA's CUTLASS library and a number of related efforts including CuTe DSL.
