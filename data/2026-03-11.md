### [1] [Turn: A Language for Agentic Computation](https://arxiv.org/abs/2603.08755)
*Muyukani Kizito*

Main category: cs.PL

TL;DR: Turn是一种编译型、基于actor的编程语言，专为需要LLM推理的智能代理软件设计，通过语言级构造确保认知类型安全、确定性控制流、隔离上下文和凭证安全。


<details>
  <summary>Details</summary>
Motivation: 现有方法在通用语言上添加框架，将关键不变性（有界上下文、类型化推理输出、凭证隔离、持久状态）编码为应用级约定而非语言保证。Turn旨在通过语言级构造填补这一空白，为智能代理软件提供更强的安全保障。

Method: Turn引入五个语言级构造：1) 认知类型安全：将LLM推理作为类型化原语，编译器从结构定义生成JSON Schema，VM在绑定前验证模型输出；2) 置信度运算符：基于模型确定性启用确定性控制流；3) 基于actor的进程模型：为每个代理提供隔离的上下文窗口、持久内存和邮箱；4) 基于能力标识系统：从VM主机返回不透明、不可伪造的句柄；5) 编译时模式吸收：从外部规范合成类型化API绑定。

Result: Turn实现了语言设计、类型规则、模式语义和基于Rust的字节码VM，并针对代表性智能代理工作负载进行了评估。项目已在GitHub开源。

Conclusion: Turn通过语言级构造为智能代理软件提供了更强的安全保障，将关键不变性从应用级约定提升为语言保证，为构建可靠、安全的自主推理系统提供了新的编程语言基础。

Abstract: We present \textbf{Turn}, a compiled, actor-based programming language -- statically typed for schema inference, dynamically typed at the value level -- for agentic software: programs that reason and act autonomously by delegating inference to large language models (LLMs). Existing approaches augment general-purpose languages with frameworks, encoding critical invariants (bounded context, typed inference output, credential isolation, durable state) as application-level conventions rather than language guarantees.
  Turn introduces five language-level constructs that address this gap. \emph{Cognitive Type Safety} makes LLM inference a typed primitive: the compiler generates a JSON Schema from a struct definition and the VM validates model output before binding. The \emph{confidence operator} enables deterministic control flow gated on model certainty. Turn's \emph{actor-based process model}, derived from Erlang, gives each agent an isolated context window, persistent memory, and mailbox. A \emph{capability-based identity system} returns opaque, unforgeable handles from the VM host, ensuring raw credentials never enter agent memory. Finally, \emph{compile-time schema absorption} (\texttt{use schema::<protocol>}) synthesizes typed API bindings from external specifications at compile time; the \texttt{openapi} adapter is shipped with \texttt{graphql}, \texttt{fhir}, and \texttt{mcp} in active development.
  We describe the language design, type rules, schema semantics, and a Rust-based bytecode VM, and evaluate Turn against representative agentic workloads. Turn is open source at https://github.com/ekizito96/Turn.

</details>


### [2] [Idempotent Slices with Applications to Code-Size Reduction](https://arxiv.org/abs/2603.09726)
*Rafael Alvarenga de Azevedo,Daniel Augusto Costa de Sa,Rodrigo Caetano Rocha,Fernando Magno Quintão Pereira*

Main category: cs.PL

TL;DR: 该论文形式化了幂等后向切片的概念，提出了从GSA形式程序中提取幂等切片的算法，并展示了其在代码大小优化中的应用。


<details>
  <summary>Details</summary>
Motivation: Guimaraes等人之前使用幂等切片将急切求值转换为严格求值，但该算法不足以正确处理一般控制流图。需要形式化幂等切片概念并开发更通用的算法。

Method: 形式化幂等后向切片的概念，提出从Gated Static Single Assignment (GSA)形式程序中提取幂等切片的正确高效算法。

Result: 在LLVM测试套件中，该算法在特定基准测试中实现了高达-7.24%的代码大小减少（使用clang 17的-Os优化序列）。

Conclusion: 形式化的幂等切片算法能够正确处理一般控制流图，为稀疏代码大小优化提供了理论基础和实用工具。

Abstract: Given a value computed within a program, an idempotent backward slice with respect to this value is a maximal subprogram that computes it. An informal notion of an idempotent slice has previously been used by Guimaraes et al. to transform eager into strict evaluation in the LLVM intermediate representation. However, that algorithm is insufficient to be correctly applied to general control-flow graphs. This paper addresses these omissions by formalizing the notion of idempotent backward slices and presenting a sound and efficient algorithm for extracting them from programs in Gated Static Single Assignment (GSA) form. As an example of their practical use, the paper describes how identifying and extracting idempotent backward slices enables a sparse code-size reduction optimization; that is, one capable of merging non-contiguous sequences of instructions within the control-flow graph of a single function or across functions. Experiments with the LLVM test suite show that, in specific benchmarks, this new algorithm achieves code-size reductions up to -7.24% on programs highly optimized by the -Os sequence of passes from clang 17.

### [3] [Arbiter: Detecting Interference in LLM Agent System Prompts](https://arxiv.org/abs/2603.08993)
*Tony Mason*

Main category: cs.SE

TL;DR: Arbiter框架通过结合形式化评估规则和多模型LLM扫描，检测编码代理系统提示中的干扰模式，在三个主要编码代理中发现大量问题，成本仅0.27美元。


<details>
  <summary>Details</summary>
Motivation: LLM编码代理的系统提示作为控制代理行为的软件构件，缺乏传统软件应有的测试基础设施，需要系统化的评估方法来发现其中的干扰模式和安全漏洞。

Method: 提出Arbiter框架，结合形式化评估规则和多模型LLM扫描方法，对Claude Code、Codex CLI和Gemini CLI三个主要编码代理的系统提示进行无定向扫描和定向分析。

Result: 发现152个无定向扫描结果和21个手动标记的干扰模式；提示架构（单体式、扁平式、模块化）与观察到的故障类别强相关但与严重性无关；多模型评估能发现单模型分析无法发现的漏洞类别；其中一个发现与Google已修补的问题一致。

Conclusion: 系统提示需要像传统软件一样进行严格测试，Arbiter框架能以低成本有效发现编码代理系统提示中的干扰模式和漏洞，多模型评估比单模型分析更全面。

Abstract: System prompts for LLM-based coding agents are software artifacts that govern agent behavior, yet lack the testing infrastructure applied to conventional software. We present Arbiter, a framework combining formal evaluation rules with multi-model LLM scouring to detect interference patterns in system prompts. Applied to three major coding agent system prompts: Claude Code (Anthropic), Codex CLI (OpenAI), and Gemini CLI (Google), we identify 152 findings across the undirected scouring phase and 21 hand-labeled interference patterns in directed analysis of one vendor. We show that prompt architecture (monolithic, flat, modular) strongly correlates with observed failure class but not with severity, and that multi-model evaluation discovers categorically different vulnerability classes than single-model analysis. One scourer finding was structural data loss in Gemini CLI's memory system was consistent with an issue filed and patched by Google, which addressed the symptom without addressing the schema-level root cause identified by the scourer. Total cost of cross-vendor analysis: \$0.27 USD.

</details>
