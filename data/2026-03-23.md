### [1] [Defusing Logic Bombs in Symbolic Execution with LLM-Generated Ghost Code](https://arxiv.org/abs/2603.19239)
*Dimitrios Stamatios Bouras,Sergey Mechtaev*

Main category: cs.PL

TL;DR: Gordian是一个混合符号执行框架，使用LLM生成轻量级幽灵代码来帮助SMT求解器处理求解器不友好的代码片段，同时保留其精确的全局推理能力。


<details>
  <summary>Details</summary>
Motivation: 传统符号执行受限于求解器不友好的程序片段、复杂数值推理和无界堆结构。现有LLM方法难以分析真实世界代码库，因为深度执行路径需要跨多个交互约束的全局一致性推理。

Method: 提出三种幽灵代码类型：(1)通过迭代双向约束传播反转困难代码片段；(2)使用求解器友好的代理建模同时保留相关行为；(3)无界堆空间的语义分区。在KLEE符号执行引擎上实现Gordian框架。

Result: 在合成"逻辑炸弹"、数学库FDLibM和三个结构化输入程序(libexpat、jq、bc)上评估。相比传统符号执行基线，覆盖率平均提高52-84%；相比LLM方法提高86-419%，同时LLM令牌使用量平均减少90-96%。

Conclusion: Gordian展示了在真实世界环境中使用LLM选择性生成幽灵代码来增强符号执行的实用性和有效性，实现了传统符号执行和LLM方法的优势结合。

Abstract: Symbolic execution is a powerful program analysis technique, but its effectiveness is fundamentally limited by solver-hostile program fragments, complex numerical reasoning, and unbounded heap structures. Recent work proposed replacing constraint solvers with large language models (LLMs) to bypass these limitations, but such approaches struggle to analyze real-world codebases, where deep execution paths require globally consistent reasoning across many interacting constraints. We present Gordian, a hybrid symbolic execution framework that uses LLMs selectively to generate lightweight ghost code that aids an SMT solver in handling solver-hostile code fragments, while preserving its precise, global reasoning capability. In particular, we propose three types of ghost code: (1) inversion of difficult code fragments with iterative bidirectional constraint propagation, (2) modeling via solver-friendly surrogates while preserving relevant behavior, and (3) semantic partitioning of unbounded heap spaces. We implemented Gordian on top of the KLEE symbolic execution engine and evaluated it on synthetic "logic bombs" capturing distinct symbolic reasoning challenges, a popular mathematical library FDLibM, and three structured-input programs (libexpat, jq, and bc). Across all benchmarks, Gordian improves coverage on average by 52-84% over traditional symbolic execution baselines, and by 86-419% over LLM-based techniques, while reducing LLM token usage by an average of 90-96%. This highlights the practicality and effectiveness of this approach in real-world settings.

### [2] [Incremental Live Programming via Shortcut Memoization](https://arxiv.org/abs/2603.19560)
*Marisa Kirisame,Thomas J. Porter,Ruqing Yang,Jianqiu Zhao,Yudi Wu,Ivan Wei,Cyrus Omar,Pavel Panchekha*

Main category: cs.PL

TL;DR: Chordata是一个基于快捷记忆化的增量程序解释器，通过观察程序执行历史学习计算模式，在程序编辑后重用这些模式来加速重新执行，相比基线获得13.03倍加速。


<details>
  <summary>Details</summary>
Motivation: 实时编程系统需要在程序编辑后快速重新执行以显示动态影响，但当程序变得大型或复杂时，完全重新执行计算成本高昂，需要增量执行机制来提高效率。

Method: 基于快捷记忆化技术，通过观察先前版本程序的执行来学习重复计算模式（称为快捷方式），当相同或结构相似的程序片段重新执行时应用这些快捷方式。提供了基于重写语义的形式化语义，并实现了Hazel实时编程系统的变体。

Result: 在学生解决简单编程问题的编辑轨迹上评估，相比基线获得13.03倍加速，内存开销为19.97倍。对于较小的更改和更复杂的程序，Chordata获得更大的加速比，甚至能在单次执行内提供加速。

Conclusion: Chordata通过快捷记忆化有效实现了增量程序解释，显著提高了实时编程系统的执行效率，特别是在处理复杂程序和频繁小修改时表现更优。

Abstract: Live programming systems aim to quickly show programmers the dynamic impacts of program edits. To do so, they re-execute the program whenever it is edited, which poses a computational challenge when programs become large or complex. This has led to the need for incrementality in the implementation of live program interpreters. This paper introduces Chordata, an incremental program interpreter based on shortcut memoization, which learns repeated patterns of computation, called shortcuts, by observing executions of previous versions of a program. It can then apply these shortcuts when the same or a structurally similar program fragment is re-executed. This paper contributes a formal semantics of shortcut memoization for any language with a rewrite-based semantics, with mechanized proofs of key correctness properties. We then express a variant of the Hazel live programming system, expressed as a CEK machine, in Chordata, and develop a number of practical heuristics to learn high-value shortcuts. We evaluate the resulting system on editing traces of students solving simple programming problems. Chordata achieves a speedup of 13.03\times compared to baseline with a 19.97\times memory overhead. For smaller changes and for more complex programs, Chordata achieves even greater speedups. Furthermore, we show that Chordata is capable of providing a speedup even within a single execution, with a faster speedup on a larger input.

### [4] [Sound State Encodings in Translational Separation Logic Verifiers (Extended Version)](https://arxiv.org/abs/2603.20001)
*Hongyi Ling,Thibault Dardinier,Ellen Arlt,Peter Müller*

Main category: cs.PL

TL;DR: 提出了首个用于证明具有非平凡状态编码的翻译式分离逻辑验证器正确性的形式化框架，解决了现有方法需要前后端状态模型相同的强假设问题。


<details>
  <summary>Details</summary>
Motivation: 现有翻译式验证器的形式化方法要求前端和后端具有相同的状态模型，这在实践中不现实，特别是当状态模型定义了支持的分离逻辑资源时。需要支持非平凡状态编码的验证器正确性证明框架。

Method: 提出了一个形式化框架，仅假设前端和中间验证语言（IVL）状态模型之间存在同态编码关系。核心是一个新颖的条件——向后可满足性，用于保证前端翻译的正确性。该框架针对基于并发分离逻辑的前端验证器和分离逻辑IVL（如Raven、VeriFast、Viper）进行了形式化。

Result: 框架在Isabelle/HOL中形式化并证明了所有证明。通过证明三种常见状态编码的正确性，展示了框架的表达能力。

Conclusion: 这是首个支持非平凡状态编码的翻译式分离逻辑验证器正确性证明的形式化框架，解决了实际验证器中状态模型差异的现实问题，为各种前端和IVL提供了通用解决方案。

Abstract: Automated program verifiers are often organized into a front-end, which encodes an input program into an intermediate verification language (IVL), and a back-end, which proves that the IVL program is correct. Soundness of such translational verifiers requires that the back-end verification is sound and that correctness of the IVL program implies correctness of the input program. Existing formalizations for translational verifiers based on separation logic target the former, but support the latter only under the strong assumption that there exists a separation logic for the input program with the same state model as the IVL. This assumption is unrealistic in practice, especially since the state model also defines the supported separation logic resources.
  We present the first formal framework for proving the soundness of translational separation logic verifiers with non-trivial state encodings. To be applicable to various front-ends and IVLs, our framework only assumes the existence of a homomorphic encoding relation between the front-end and IVL state models. At the core of our framework is a novel condition, backward satisfiability, which is crucial to guarantee the soundness of the front-end translation. We formalize our framework for front-end verifiers based on concurrent separation logic and separation logic IVLs, such as Raven, VeriFast, and Viper. We demonstrate its expressiveness by proving soundness for three common state encodings. Our framework and all proofs are formalized in Isabelle/HOL.

### [5] [Analyzing Decoders for Quantum Error Correction](https://arxiv.org/abs/2603.20127)
*Abtin Molavi,Feras Saad,Aws Albarghouthi*

Main category: cs.PL

TL;DR: 本文提出了一种新的量子纠错解码器系统化分析方法，替代传统的蒙特卡洛模拟方法，通过结构化错误空间搜索和约束多项式优化来量化解码器准确性和鲁棒性。


<details>
  <summary>Details</summary>
Motivation: 量子纠错解码器的准确性直接影响量子计算的规模，但目前主要依赖蒙特卡洛模拟进行评估，这种方法需要大量样本且方差较大，存在明显局限性。

Method: 1) 为QEC程序开发了形式化语义，基于Stim电路格式；2) 结合结构化错误空间搜索和约束多项式优化内核；3) 构建验证器来量化解码器准确性和对物理错误率漂移的鲁棒性。

Result: 该方法在低错误率区域表现优于模拟方法，能够量化解码器在物理错误率区间内的鲁棒性，为量子纠错系统设计提供了更可靠的评估工具。

Conclusion: 本文提出的系统化分析方法为量子纠错解码器评估提供了理论基础和实践工具，能够更高效、准确地评估解码器性能，特别在低错误率场景下具有优势。

Abstract: Quantum error correction (QEC) enables reliable computation on noisy hardware by encoding logical information across many physical qubits and periodically measuring parities to detect errors. A decoder is the classical algorithm that uses these measurements to infer which error most likely occurred, so that the system can correct it. The decoder's accuracy-how rarely it makes the wrong guess-directly determines the scale of quantum computation that can be reliably executed. With a wealth of competing decoding algorithms, a QEC system designer needs reliable methods to evaluate them. Today, the dominant approach is to evaluate decoders using Monte Carlo simulation. However, simulation has several drawbacks such as requiring many samples to produce low variance estimates.
  In this work, we develop a new systematic analysis for evaluating decoders. We introduce a novel formal semantics of a core language for QEC programs that captures the de facto standard Stim circuit format, providing a principled theoretical foundation for the emerging space of fault-tolerant quantum systems design. Given a QEC program and a decoder, our verifier can quantify both the decoder accuracy and the decoder robustness to drift in physical error rate. Our approach has two key components: (i) a structured search over the space of possible errors; and (ii) a constrained polynomial optimization kernel. A thorough empirical evaluation of our approach suggests that it can outperform simulation, especially in low error rate regimes, and that it can be deployed to quantify decoder robustness over an interval of physical error rates.

</details>
