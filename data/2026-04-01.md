*Michael Kifer,Theresa Swift*

Main category: cs.PL

TL;DR: ErgoAI是一个高级多范式逻辑编程语言和系统，作为Flora-2的增强版和继任者，专注于可扩展的知识表示和推理，支持结构化知识和外部源知识（如向量嵌入）。


<details>
  <summary>Details</summary>
Motivation: 开发ErgoAI是为了解决Flora-2系统的局限性，创建一个更先进的多范式逻辑编程系统，专注于可扩展的知识表示和推理，能够处理结构化知识和外部源知识，同时保持良好基础语义。

Method: ErgoAI采用多范式设计，基于良好基础语义，支持对象逻辑（F-logic）与非单调继承、HiLog风格的高阶语法、规则可废止性、语义清晰的交易更新、广泛使用子目标延迟处理不安全查询和提升性能，以及在模块级别可选的有界理性支持。

Result: ErgoAI是一个全新的语言和系统，虽然Flora-2程序被编译到XSB并采用许多Prolog特性，但ErgoAI在架构和设计上完全不同，提供了更强大的知识表示和推理能力。

Conclusion: ErgoAI作为Flora-2的继任者，代表了逻辑编程领域的重要进展，特别在可扩展知识表示和推理方面，能够有效整合结构化知识和外部源知识，为复杂知识处理提供了强大的多范式解决方案。

Abstract: ErgoAI is a high level, multi-paradigm logic programming language and system developed by Coherent Knowledge Systems as an enhancement of and a successor to the popular Flora-2 system. ErgoAI is oriented towards scalable knowledge representation and reasoning, and can exploit both structured knowledge as well as knowledge derived from external sources such as vector embeddings. From the start, ErgoAI (and Flora-2 before it) were designed to exploit the well-founded semantics for reasoning in a multi-paradigm environment, including object-based logic (F-logic) with non-monotonic inheritance; higher order syntax in the style of HiLog; defeasibility of rules; semantically clean transactional updates; extensive use of subgoal delay for handling unsafe queries and for better performance; and optional support for bounded rationality at a module level. Although Flora-2 programs are compiled into XSB and adopt many Prolog features, ErgoAI is altogether a different language and system.
  Under consideration in Theory and Practice of Logic Programming (TPLP).

### [2] [Self-Improving Code Generation via Semantic Entropy and Behavioral Consensus](https://arxiv.org/abs/2603.29292)
*Huan Zhang,Wei Cheng,Wei Hu*

Main category: cs.SE

TL;DR: ConSelf：一种无需外部监督的代码生成自改进方法，通过代码语义熵进行课程构建，结合共识驱动的偏好优化来提升LLM的代码生成能力。


<details>
  <summary>Details</summary>
Motivation: 现有代码生成改进方法依赖昂贵的监督微调或偏好优化，需要强大的教师模型或测试单元等外部资源。然而在实际场景中，获取参考解决方案和测试预言远比获取问题描述和测试输入困难。本文旨在探索代码语言模型能否在没有优秀教师和测试预言的情况下实现自我改进。

Method: 提出ConSelf方法，包含两个核心创新：1) 代码语义熵——通过评估程序行为的功能多样性来衡量问题级不确定性的新指标，用于构建最具学习价值问题的课程；2) 共识驱动的直接偏好优化(Con-DPO)——基于行为共识对每个偏好对进行加权的偏好微调方法，减轻噪声自生成监督的影响。

Result: 在多个基准测试和骨干LLM上的实验表明，ConSelf显著优于基线方法，验证了基于语义熵的课程构建和共识驱动优化在无需外部监督情况下改进代码生成的有效性。

Conclusion: ConSelf成功证明了代码语言模型可以在没有外部教师和测试预言的情况下实现自我改进，为解决实际场景中监督资源稀缺的代码生成改进问题提供了有效方案。

Abstract: Improving the code generation capabilities of large language models (LLMs) typically relies on supervised fine-tuning or preference optimization, both of which require costly external resources such as powerful teacher models or reliable test units. However, in real-world scenarios, it is much harder to obtain reference solutions and test oracles than problem descriptions and test inputs. In this paper, we tackle a challenging yet realistic question: Can a code language model improve itself without access to a superior teacher and a test oracle? To answer this, we propose ConSelf, a self-improving approach built upon two key ideas. First, we introduce code semantic entropy, a novel metric that measures problem-level uncertainty by assessing the functional diversity of program behaviors, enabling a curriculum construction with the most learnable problems. Second, we present consensus-driven direct preference optimization (Con-DPO), a preference-based fine-tuning method that weights each preference pair by its behavioral consensus, thereby mitigating the impact of noisy self-generated supervision. Experiments on various benchmarks and backbone LLMs demonstrate that ConSelf significantly outperforms baselines, validating the effectiveness of semantic entropy-based curriculum construction and consensus-driven optimization in improving code generation without external supervision.

### [3] [Phyelds: A Pythonic Framework for Aggregate Computing](https://arxiv.org/abs/2603.29999)
*Gianluca Aguzzi,Davide Domini,Nicolas Farabegoli,Mirko Viroli*

Main category: cs.SE

TL;DR: Phyelds是一个用于聚合编程的Python库，填补了Python在数据科学和机器学习生态系统中缺乏聚合编程实现的空白。


<details>
  <summary>Details</summary>
Motivation: 聚合编程已有十多年发展历史，并在多个领域成功应用，但现有实现（如Protelis、ScaFi、FCPP）不针对数据科学从业者。Python是数据科学和机器学习的实际标准语言，拥有丰富的生态系统，但在聚合编程方面存在空白。

Method: 开发Phyelds库，提供完全功能但轻量级的场计算模型实现，采用Pythonic API，架构设计旨在与Python机器学习生态系统无缝集成。

Result: Phyelds展示了跨领域的多功能性，从经典的聚合计算模式到联邦学习协调，以及与广泛使用的多智能体强化学习模拟器的集成。

Conclusion: Phyelds填补了Python生态系统中聚合编程的空白，为数据科学从业者、教育工作者和机器人开发者提供了强大的工具，支持大规模分布式学习和机器学习算法的实现。

Abstract: Aggregate programming is a field-based coordination paradigm with over a decade of exploration and successful applications across domains including sensor networks, robotics, and IoT, with implementations in various programming languages, such as Protelis, ScaFi (Scala), and FCPP (C++). A recent research direction integrates machine learning with aggregate computing, aiming to support large-scale distributed learning and provide new abstractions for implementing learning algorithms. However, existing implementations do not target data science practitioners, who predominantly work in Python--the de facto language for data science and machine learning, with a rich and mature ecosystem. Python also offers advantages for other use cases, such as education and robotics (e.g., via ROS). To address this gap, we present Phyelds, a Python library for aggregate programming. Phyelds offers a fully featured yet lightweight implementation of the field calculus model of computation, featuring a Pythonic API and an architecture designed for seamless integration with Python's machine learning ecosystem. We describe the design and implementation of Phyelds and illustrate its versatility across domains, from well-known aggregate computing patterns to federated learning coordination and integration with a widely used multi-agent reinforcement learning simulator.

### [4] [A Graded Modal Dependent Type Theory with Erasure, Formalized](https://arxiv.org/abs/2603.29716)
*Andreas Abel,Nils Anders Danielsson,Oskar Eriksson*

Main category: cs.LO

TL;DR: 本文提出了一种分级模态类型理论，这是一种带有分级系统的依赖类型理论，可用于强制执行代码的各种属性。该理论支持多种类型构造，并通过模态结构参数化，重点关注定量属性特别是擦除功能。


<details>
  <summary>Details</summary>
Motivation: 开发一种能够通过分级系统跟踪变量使用情况的类型理论，特别关注定量属性如擦除功能，使得函数参数可以被标记为可擦除，从而优化程序执行。

Method: 构建基于模态结构（部分有序半环）的分级模态类型理论，支持Π类型、弱/强Σ类型、自然数、空类型、宇宙等。使用Agda进行形式化，核心采用句法克里普克逻辑关系，证明元理论性质如主题归约、一致性、正规化等。

Result: 成功形式化了该类型理论，证明了主要元理论性质：主题归约、一致性、正规化、定义相等的可判定性。还证明了分级分配替换定理和归约下分级保持性。对于特定模态类，证明了提取函数的正确性：自然数类型的程序在提取前后具有相同值。

Conclusion: 该分级模态类型理论提供了一个强大的框架，通过分级系统精确控制变量使用，特别在擦除优化方面具有实际应用价值。形式化验证确保了理论的可靠性，提取函数的正确性证明为程序优化提供了理论基础。

Abstract: We present a graded modal type theory, a dependent type theory with grades that can be used to enforce various properties of the code. The theory has $Π$-types, weak and strong $Σ$-types, natural numbers, an empty type, and a universe, and we also extend the theory with weak and strong unit types and graded $Σ$-types. The theory is parameterized by a modality structure, a kind of partially ordered semiring, whose elements (grades) are used to track the usage of variables in terms and types. Different modalities are possible. We focus mainly on quantitative properties, in particular erasure: with the erasure modality one can mark function arguments as erasable.
  The theory is fully formalized in Agda. The formalization, which uses a syntactic Kripke logical relation at its core and is based on earlier work, establishes major meta-theoretic properties such as subject reduction, consistency, normalization, and decidability of definitional equality. We also prove a substitution theorem for grade assignment, and preservation of grades under reduction. Furthermore we study an extraction function that translates terms to an untyped $λ$-calculus and removes erasable content, in particular function arguments with the "erasable" grade. For a certain class of modalities we prove that extraction is sound, in the sense that programs of natural number type have the same value before and after extraction. Soundness of extraction holds also for open programs, as long as all variables in the context are erasable, the context is consistent, and erased matches are not allowed for weak $Σ$-types.

</details>
