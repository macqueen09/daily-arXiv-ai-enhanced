### [1] [M$^\star$: Every Task Deserves Its Own Memory Harness](https://arxiv.org/abs/2604.11811)
*Wenbo Pan,Shujie Liu,Xiangyang Zhou,Shiwei Zhang,Wanlu Shi,Mirror Xu,Xiaohua Jia*

Main category: cs.PL

TL;DR: M$^\star$ 是一种通过可执行程序演化自动发现任务优化内存系统的方法，将智能体内存系统建模为Python程序，通过反射式代码演化优化数据模式、存储逻辑和工作流指令，在对话、具身规划和专家推理等任务上优于固定内存基线。


<details>
  <summary>Details</summary>
Motivation: 当前大型语言模型智能体通常采用针对特定领域设计的固定内存系统（如对话的语义检索或编码的技能重用），但这些优化后的内存系统难以迁移到其他任务。需要一种能够自动发现任务优化内存系统的方法。

Method: 将智能体内存系统建模为包含数据模式、存储逻辑和智能体工作流指令的Python程序（内存程序）。采用基于种群的搜索策略和反射式代码演化方法，通过分析评估失败来迭代优化候选程序。

Result: 在对话、具身规划和专家推理四个不同基准测试中，M$^\star$ 在所有评估任务上都稳健地优于现有的固定内存基线。演化出的内存程序在不同领域展现出结构上不同的处理机制。

Conclusion: 为特定任务专门化内存机制能够探索广阔的设计空间，相比通用内存范式提供更优的解决方案。自动发现的内存系统能够适应不同任务需求，展现出结构上的多样性。

Abstract: Large language model agents rely on specialized memory systems to accumulate and reuse knowledge during extended interactions. Recent architectures typically adopt a fixed memory design tailored to specific domains, such as semantic retrieval for conversations or skills reused for coding. However, a memory system optimized for one purpose frequently fails to transfer to others. To address this limitation, we introduce M$^\star$, a method that automatically discovers task-optimized memory harnesses through executable program evolution. Specifically, M$^\star$ models an agent memory system as a memory program written in Python. This program encapsulates the data Schema, the storage Logic, and the agent workflow Instructions. We optimize these components jointly using a reflective code evolution method; this approach employs a population-based search strategy and analyzes evaluation failures to iteratively refine the candidate programs. We evaluate M$^\star$ on four distinct benchmarks spanning conversation, embodied planning, and expert reasoning. Our results demonstrate that M$^\star$ improves performance over existing fixed-memory baselines robustly across all evaluated tasks. Furthermore, the evolved memory programs exhibit structurally distinct processing mechanisms for each domain. This finding indicates that specializing the memory mechanism for a given task explores a broad design space and provides a superior solution compared to general-purpose memory paradigms.

### [2] [Polyregular equivalence is undecidable in higher-order types](https://arxiv.org/abs/2604.11935)
*Mikołaj Bojańczyk,Grzegorz Fabiański,Rafał Stefański*

Main category: cs.PL

TL;DR: 字符串到字符串的多正则函数等价性判定问题未解决，本文研究其基于λ演算的高阶扩展，证明等价性不可判定


<details>
  <summary>Details</summary>
Motivation: 字符串到字符串的多正则函数等价性（f=g）是否可判定仍是一个开放问题，本文旨在研究其高阶扩展下的可判定性

Method: 基于Bojańczyk (2018)中λ演算定义的多正则函数，构建高阶扩展，通过从铺砖问题归约来证明不可判定性

Result: 在基于λ演算的高阶扩展设置下，多正则函数的等价性是不可判定的

Conclusion: 多正则函数的高阶扩展使等价性判定变得不可判定，这为理解此类函数的计算复杂性提供了新视角

Abstract: It is open whether equivalence ( f = g ) is decidable for string-to-string polyregular functions. We consider their higher-order extension based on the λ-calculus definition of polyregular functions from Bojańczyk (2018). In this setting, equivalence is undecidable by reduction from the tiling problem.

### [3] [Modular Verification of Differential Privacy in Probabilistic Higher-Order Separation Logic (Extended Version)](https://arxiv.org/abs/2604.12713)
*Philipp G. Haselwarter,Alejandro Aguirre,Simon Oddershede Gregersen,Kwing Hei Li,Joseph Tassarotti,Lars Birkedal*

Main category: cs.PL

TL;DR: 提出一种适用于验证复杂通用编程语言中差分隐私实现的新型程序逻辑，支持隐私预算作为分离逻辑资源的一流推理，能够处理现代DP库中的常见编程模式。


<details>
  <summary>Details</summary>
Motivation: 现代差分隐私库的设计模式和语言特性以及库设计者希望建立的保证类别往往超出了先前验证方法的范围，需要一种能够验证复杂通用编程语言中DP实现的方法。

Method: 引入一种程序逻辑，支持将隐私预算作为分离逻辑资源进行推理，能够处理隐私过滤器、缓存等常见DP库编程模式，支持模块化推理原语、高阶组合器和交互式算法。

Result: 实现了一个经过验证的差分隐私机制库，包括稀疏向量技术的在线版本和受OpenDP启发的隐私过滤器，展示了规范的可重用性，所有结果都在Rocq Prover中得到基础性验证。

Conclusion: 该程序逻辑能够有效验证复杂通用编程语言中的差分隐私实现，处理现代DP库的常见模式，为DP库的可靠性提供了形式化保证。

Abstract: Differential privacy is the standard method for privacy-preserving data analysis. The importance of having strong guarantees on the reliability of implementations of differentially private algorithms is widely recognized and has sparked fruitful research on formal methods. However, the design patterns and language features used in modern DP libraries as well as the classes of guarantees that the library designers wish to establish often fall outside of the scope of previous verification approaches.
  We introduce a program logic suitable for verifying differentially private implementations written in complex, general-purpose programming languages. Our logic has first-class support for reasoning about privacy budgets as a separation logic resource. The expressiveness of the logic and the target language allow our approach to handle common programming patterns used in the implementation of libraries for differential privacy, such as privacy filters and caching. While previous work has focused on developing guarantees for programs written in domain-specific languages or for privacy mechanisms in isolation, our logic can reason modularly about primitives, higher-order combinators, and interactive algorithms.
  We demonstrate the applicability of our approach by implementing a verified library of differential privacy mechanisms, including an online version of the Sparse Vector Technique, as well as a privacy filter inspired by the popular Python library OpenDP, which crucially relies on our ability to handle the combination of randomization, local state, and higher-order functions. We demonstrate that our specifications are general and reusable by instantiating them to verify clients of our library. All of our results have been foundationally verified in the Rocq Prover.

### [4] [Hyper Separation Logic (extended version)](https://arxiv.org/abs/2604.12870)
*Trayan Gospodinov,Thibault Dardinier,Peter Müller*

Main category: cs.PL

TL;DR: HSL是首个支持任意量词交替的分离逻辑，能够对堆操作程序的超性质进行模块化推理


<details>
  <summary>Details</summary>
Motivation: 现有分离逻辑只能处理特定类别的超性质（如∀∀-超性质或∃∃-性质），不支持量词交替，无法表达GNI等复杂性质。Hyper Hoare Logic虽能处理量词交替，但不支持堆操作程序。

Method: 提出Hyper Separation Logic (HSL)，通过新颖的超分离合取将标准分离合取提升到状态集合，实现超性质的广义框架规则，扩展Hyper Hoare Logic以支持堆操作。

Result: 在Isabelle/HOL中证明了HSL的正确性，展示了其在表达超出现有分离逻辑能力范围的超性质方面的表达能力。

Conclusion: HSL是首个支持任意量词交替且适用于堆操作程序的程序逻辑，填补了现有分离逻辑在复杂超性质推理方面的空白。

Abstract: Many important functional and security properties--including non-interference, determinism, and generalized non-interference (GNI)--are hyperproperties, i.e., properties relating multiple executions of a program. Existing separation logics allow one to reason about specific classes of hyperproperties, e.g., $\forall\forall$-hyperproperties such as non-interference and $\exists\exists$-properties such as non-determinism. However, they do not support quantifier alternation, which is for instance needed to express GNI. The only existing logic that can reason about such properties is Hyper Hoare Logic, but it does not support heap-manipulating programs and, thus, is not applicable to common imperative programs.
  This paper introduces Hyper Separation Logic (HSL), the first program logic that supports modular reasoning about hyperproperties with arbitrary quantifier alternation over programs that manipulate the heap. HSL generalizes Hyper Hoare Logic with a novel hyper separating conjunction that lifts the standard separating conjunction to sets of states, enabling a generalized frame rule for hyperproperties. We prove HSL sound in Isabelle/HOL and demonstrate its expressiveness for hyperproperties that lie beyond the reach of existing separation logics.

</details>

Motivated by this opportunity, we introduce a pleasingly parallel virtual machine and benchmark its performance by evaluating millions of concurrent array programs, observing speedups up to $147\times$ relative to serial evaluation.
