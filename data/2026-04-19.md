### [1] [Filament: Denning-Style Information Flow Control for Rust](https://arxiv.org/abs/2604.14357)
*Jeffrey C. Ching,Quan Zhou,Danfeng Zhang*

Main category: cs.PL

TL;DR: Filament是一个无需编译器修改的Rust静态信息流控制库，通过类型推断、轻量级隐式流控制和安全互操作性宏，提供细粒度安全检查和更宽松的编程模型。


<details>
  <summary>Details</summary>
Motivation: 现有语言级信息流控制工具面临两难：Denning式系统需要编译器修改，而粗粒度方法（如Cocoon）编程模型限制较多。需要一种无需编译器修改又能提供细粒度控制的实用方案。

Method: 1. 利用Rust类型推断实现细粒度显式流检查，减少标注开销；2. 引入pc_block!宏通过编译时程序计数器标签强制执行隐式流；3. 提供fcall!和mcall!宏支持与标准库和第三方库的安全互操作。

Result: Filament编译时开销可忽略，仅需适度标注。相比Cocoon，提供更宽松的编程模型，减少绕过安全检查的频繁需求。

Conclusion: Filament成功构建了无需编译器修改的实用Denning式静态IFC库，通过创新方法解决了Rust中IFC的关键挑战，在细粒度控制和编程灵活性间取得良好平衡。

Abstract: Existing language-based information-flow control (IFC) tools face a fundamental tension: Denning-style systems that track explicit and implicit flows at the variable level typically require compiler modifications, while more coarse-grained approaches, including recent work Cocoon, avoid compiler changes but impose more restrictive programming models. We present Filament, a Denning-style static IFC library for Rust that requires no compiler modifications. Filament addresses three key challenges in building a practical IFC library for Rust. First, it enables fine-grained explicit-flow checking with minimal annotation overhead by leveraging Rust's type inference. Second, it introduces pc_block!, a lightweight construct for enforcing implicit flows via a compile-time program counter label, without requiring compiler support. Third, it provides fcall! and mcall! macros to support seamless and safe interoperability with standard and third-party libraries. Our evaluation shows that Filament incurs negligible compile-time overhead and requires only modest annotations. Moreover, compared to Cocoon, Filament offers a more permissive programming model, reducing the need for frequent escape hatches that bypass security checks.

### [2] [Nautilus: An Auto-Scheduling Tensor Compiler for Efficient Tiled GPU Kernels](https://arxiv.org/abs/2604.14825)
*Yifan Zhao,Yuchen Yang,Matei Budiu,Sasa Misailovic*

Main category: cs.PL

TL;DR: Nautilus是一个新型张量编译器，能够从高级数学描述自动优化生成高效的GPU内核，首次实现从注意力机制数学描述自动发现类似FlashAttention-3的内核。


<details>
  <summary>Details</summary>
Motivation: 当前张量编译器需要程序员手动优化，无法从高级数学描述自动生成高效内核。Nautilus旨在将优化负担从程序员转移到编译器，实现完全自动化的数学到内核优化。

Method: 采用连续降低设计，将高级代数规范编译为高效的GPU分块内核。提出新型自动调度器，在保持规则程序结构的同时发现高级优化序列，支持高级融合等全局变换。

Result: 在NVIDIA GH200和RTX 5090 GPU上测试5个Transformer模型和150个配置，Nautilus在GH200上比最先进编译器吞吐量高23%，在RTX 5090上高42%，在许多长序列配置上匹配或超过手动编写的cuDNN内核。

Conclusion: Nautilus是首个端到端张量编译器，能够从类似数学的注意力描述自动发现FlashAttention-3类内核，将优化负担完全从程序员转移到编译器，实现了显著的性能提升。

Abstract: We present Nautilus, a novel tensor compiler that moves toward fully automated math-to-kernel optimization. Nautilus compiles a high-level algebraic specification of tensor operators into efficient tiled GPU kernels. Nautilus's successive lowering design allows high-level optimizations, expression rewrites, and tile optimizations to be jointly applied in a single end-to-end system. Nautilus presents a novel auto-scheduler that discovers sequences of high-level optimizations, while preserving the regular program structure needed by tile optimizers. Nautilus's auto-scheduler captures complex interactions and trade-offs in the high-level optimizations, including aggressive global transformations like advanced reduction fusion. Nautilus is the first end-to-end tensor compiler capable of starting from a math-like description of attention and automatically discovering FlashAttention-3-like kernels, offloading the entire burden of optimization from the programmer to the compiler. Across five transformer-based models and 150 evaluation configurations on NVIDIA GH200 and RTX 5090 GPUs, Nautilus achieves up to 23% higher throughput than state-of-the-art compilers on GH200 and up to 42% on RTX 5090, while matching or exceeding manually written cuDNN kernels on many long-sequence configurations.

</details>

### [4] [Prism: Symbolic Superoptimization of Tensor Programs](https://arxiv.org/abs/2604.15272)
*Mengdi Wu,Xiaoyu Jiang,Oded Padon,Zhihao Jia*

Main category: cs.PL

TL;DR: Prism是首个用于张量程序的符号超级优化器，采用sGraph符号层次表示，通过两级搜索实现优化，在LLM工作负载上达到2.2倍加速，优化时间减少3.4倍。


<details>
  <summary>Details</summary>
Motivation: 现有张量程序优化方法在搜索空间和优化质量之间存在权衡，需要一种既能保持搜索严谨性又能扩展到现代ML工作负载的优化方法。

Method: 提出sGraph符号层次表示，将优化组织为两级搜索：生成表示程序族的符号图，然后实例化为具体实现。使用符号推理进行结构化剪枝，结合符号图生成、等价验证和参数实例化技术。

Result: 在五个常用LLM工作负载上评估，Prism比最佳超级优化器快2.2倍，比最佳基于编译器的方法快4.9倍，同时将端到端优化时间减少3.4倍。

Conclusion: Prism成功地将穷举搜索的严谨性与现代ML工作负载所需的可扩展性相结合，为张量程序优化提供了新的有效方法。

Abstract: This paper presents Prism, the first symbolic superoptimizer for tensor programs. The key idea is sGraph, a symbolic, hierarchical representation that compactly encodes large classes of tensor programs by symbolically representing some execution parameters. Prism organizes optimization as a two-level search: it constructs symbolic graphs that represent families of programs, and then instantiates them into concrete implementations. This formulation enables structured pruning of provably suboptimal regions of the search space using symbolic reasoning over operator semantics, algebraic identities, and hardware constraints.
  We develop techniques for efficient symbolic graph generation, equivalence verification via e-graph rewriting, and parameter instantiation through auto-tuning. Together, these components allow Prism to bridge the rigor of exhaustive search with the scalability required for modern ML workloads. Evaluation on five commonly used LLM workloads shows that Prism achieves up to $2.2\times$ speedup over best superoptimizers and $4.9\times$ over best compiler-based approaches, while reducing end-to-end optimization time by up to $3.4\times$.

### [5] [Pure Borrow: Linear Haskell Meets Rust-Style Borrowing](https://arxiv.org/abs/2604.15290)

<details>
  <summary>Details</summary>
Motivation: 探索在纯函数式语言中安全支持Rust风格的非局部借用机制，解决线性类型扩展后仍存在的借用模式限制问题

Abstract: A promising approach to unifying functional and imperative programming paradigms is to localize mutation using linear or affine types. Haskell, a purely functional language, was recently extended with linear types by Bernardy et al., named Linear Haskell. However, it remained unknown whether such a pure language could safely support non-local \emph{borrowing} in the style of Rust, where each borrower can be freely split and dropped without direct communication of ownership back to the lender.
  We answer this question affirmatively by \emph{Pure Borrow}, a novel framework that realizes Rust-style borrowing in Linear Haskell with purity. Notably, it features parallel state mutation with affine mutable references inside pure computation, unlike the IO and ST monads and existing Linear Haskell APIs. It also enjoys purity, lazy evaluation, first-class polymorphism and leak freedom, unlike Rust. We implement Pure Borrow simply as a library in Linear Haskell and demonstrate its power with a case study in parallel computing. We formalize the core of Pure Borrow and build a metatheory that works toward establishing safety, leak freedom and confluence, with a new, history-based model of borrowing.

### [6] [VeriGraphi: A Multi-Agent Framework of Hierarchical RTL Generation for Large Hardware Designs](https://arxiv.org/abs/2604.14550)
*Sazzadul Islam,Tasnim Tabassum,Hao Zheng*

Main category: cs.AR

TL;DR: VeriGraphi框架使用知识图谱驱动层次化Verilog生成，解决LLM在复杂硬件设计中上下文丢失、接口幻觉等问题


<details>
  <summary>Details</summary>
Motivation: 大型语言模型在生成层次化Verilog时面临挑战：难以维持跨模块上下文、产生接口幻觉、虚构模块间连线、缺乏结构一致性，尤其在处理包含非正式文本、图表和表格的复杂规格说明时

Method: 提出VeriGraphi框架，通过多智能体迭代分析规格说明构建HDA知识图谱，明确编码模块层次、端口接口、连线语义和模块间依赖关系，然后基于图谱进行渐进式代码生成

Result: 在NIST基准测试和RV32I处理器案例研究中，VeriGraphi能够以最少人工干预可靠生成层次化RTL代码，保持强大的功能正确性

Built through iterative multi-agent analysis of the specification, this Knowledge Graph provides a deterministic, machine-checkable structural scaffold before code generation. Guided by the KG, a progressive coding module incrementally generates pseudo-code and synthesizable RTL while enforcing interface consistency and dependency correctness at each submodule stage. We evaluate VeriGraphi on a benchmark of three representative specification documents from the National Institute of Standards and Technology and their corresponding implementations, and we present a RV32I processor as a detailed case study to illustrate the full pipeline. The results demonstrate that VeriGraphi enables reliable hierarchical RTL generation with minimal human intervention for RISC-V, marking a significant milestone for LLM-generated hardware design while maintaining strong functional correctness.

### [7] [Simplifying Safety Proofs with Forward-Backward Reasoning and Prophecy](https://arxiv.org/abs/2604.15266)
*Eden Frenkel,Kenneth L. McMillan,Oded Padon,Sharon Shoham*

Main category: cs.LO

TL;DR: 提出一种增量式安全证明方法，将复杂归纳不变式分解为多个简单证明步骤，结合前向推理、后向推理和预言步骤，降低不变式搜索空间复杂度。


<details>
  <summary>Details</summary>
Motivation: 复杂系统的安全证明通常需要复杂的归纳不变式，这些不变式具有复杂的布尔结构、量词和量词交替，使得自动生成和验证变得困难。需要一种方法将复杂证明分解为更简单的步骤。

Method: 提出增量式证明系统，结合三种推理规则：(1)使用归纳不变式的前向推理，(2)使用时间反转系统归纳不变式的后向推理，(3)添加存在量化属性见证的预言步骤。证明每个规则的正确性，并提供从增量证明恢复单个安全归纳不变式的构造方法。

Result: 在自然限制条件下，每个证明规则都严格增加证明能力，允许用相同公式集证明更多安全问题。增量方法能减少证明给定系统安全性所需的不变式公式搜索空间。Paxos、其变体和Raft的案例研究表明，前向后向步骤可以消除复杂布尔结构，预言步骤可以消除量词和量词交替。

Conclusion: 增量式安全证明方法通过分解复杂归纳不变式为简单步骤，有效降低了证明复杂度，提高了证明能力，为复杂分布式协议的安全验证提供了实用工具。

Abstract: We propose an incremental approach for safety proofs that decomposes a proof with a complex inductive invariant into a sequence of simpler proof steps. Our proof system combines rules for (i) forward reasoning using inductive invariants, (ii) backward reasoning using inductive invariants of a time-reversed system, and (iii) prophecy steps that add witnesses for existentially quantified properties. We prove each rule sound and give a construction that recovers a single safe inductive invariant from an incremental proof. The construction of the invariant demonstrates the increased complexity of a single inductive invariant compared to the invariant formulas used in an incremental proof, which may have simpler Boolean structures and fewer quantifiers and quantifier alternations. Under natural restrictions on the available invariant formulas, each proof rule strictly increases proof power. That is, each rule allows to prove more safety problems with the same set of formulas. Thus, the incremental approach is able to reduce the search space of invariant formulas needed to prove safety of a given system. A case study on Paxos, several of its variants, and Raft demonstrates that forward-backward steps can remove complex Boolean structure while prophecy eliminates quantifiers and quantifier alternations.

</details>
