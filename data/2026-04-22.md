### [1] [RECURSUM: Automated Code Generation for Recurrence Relations Exceeds Expert Optimization via LayeredCodegen](https://arxiv.org/abs/2604.18585)
*Rubén Darío Guerrero*

Main category: cs.PL

TL;DR: RECURSUM是一个Python DSL，用于生成优化的C++代码来计算递推关系，相比专家手写代码实现9.8倍加速，消除了领域知识和C++元编程的双重专业壁垒。


<details>
  <summary>Details</summary>
Motivation: 递推关系在正交多项式、特殊函数、数值积分和分子积分评估中无处不在，但手动优化需要同时具备领域知识和C++元编程技能，这构成了双重专业壁垒。现有方法难以系统性地超越专家手动优化。

Method: 开发了RECURSUM，一个基于Python的领域特定语言，使用类似einsum的符号来指定递推关系、有效性约束和基础情况（10-30行Python代码），生成650+行生产级C++代码。提供三个后端：模板元编程、新颖的LayeredCodegen架构优化后端、运行时循环评估。

Result: LayeredCodegen在McMurchie-Davidson Hermite系数计算中比专家手写实现快9.8倍，比模板元编程快1.9倍。架构分析揭示了三个可量化的优化效果：零拷贝输出参数（70-80%加速）、保证函数内联（15-20%）、精确大小的栈缓冲区（5-10%）。在24种递推类型上验证，生产基准测试显示加速效果能传播到完整算法，生成代码与专家基线相差在3.3%以内。

Conclusion: RECURSUM表明系统化代码生成是递推算法性能的上限。通过消除双重专业壁垒，该框架民主化了高性能科学计算，建立了一个自动化生成系统性地超越手动优化的新范式。

Abstract: Automated code generation can systematically exceed expert hand-optimization for recurrence relations-computational primitives ubiquitous in orthogonal polynomials, special functions, numerical integration, and molecular integral evaluation. We present RECURSUM, a Python-based domain-specific language generating optimized C++ for arbitrary recurrence relations via three backends: template metaprogramming for compile-time evaluation, a novel LayeredCodegen backend with architectural optimizations, and runtime loop-based evaluation. The DSL uses einsum-inspired notation to specify recurrences, validity constraints, and base cases in 10-30 lines of Python, generating 650+ lines of production C++.
  LayeredCodegen achieves 9.8x speedup over expert hand-written implementations and 1.9x over template metaprogramming for McMurchie-Davidson Hermite coefficients. Architecture analysis reveals three quantifiable effects: (1) zero-copy output parameters eliminate return-by-value overhead (70-80% of speedup), (2) guaranteed function inlining eliminates compiler-refused overhead (15-20%), (3) exact-sized stack buffers achieve 100% cache efficiency vs 27% for MAX-sized arrays (5-10%).
  We validate on 24 recurrence types spanning pure mathematics (Legendre, Chebyshev, Hermite, Laguerre polynomials), numerical analysis (Clenshaw, Golub-Welsch), and quantum chemistry (McMurchie-Davidson, Rys quadrature, Boys function). Production benchmarks show speedups propagate to complete algorithms, with generated code matching expert baselines within 3.3%.
  RECURSUM demonstrates that systematic code generation serves as the performance ceiling for recurrence algorithms. By eliminating the dual expertise barrier (domain knowledge + C++ metaprogramming), the framework democratizes high-performance scientific computing-establishing a paradigm where automated generation systematically exceeds manual optimization.

### [2] [HELIX: Verified compilation of cyber-physical control systems to LLVM IR](https://arxiv.org/abs/2604.18593)
*Vadim Zaliva,Yannick Zakowski,Ilia Zaichuk,Valerii Huhnin,Calvin Beck,Irene Yoon,Steve Zdancewic*

Main category: cs.PL

TL;DR: HELIX是一个端到端验证的代码生成系统，专注于高性能高保证数值计算，通过代数变换和形式化验证生成高效代码，支持多种硬件架构。


<details>
  <summary>Details</summary>
Motivation: 解决高性能数值计算中代码生成效率与正确性保证的平衡问题，为关键系统（如网络物理机器人系统）提供既高效又形式化验证的代码生成方案。

Method: 采用代数变换方法，将向量矩阵计算转换为针对目标硬件优化的数据流；使用算子语言和语义保持项重写进行形式化验证；通过稀疏向量抽象支持并行分解证明；在Coq中实现多层中间语言和验证基础设施。

Result: 成功构建了HELIX系统，能够从高级数学问题描述生成高效的命令式实现，同时提供从原始公式到LLVM IR的语义保持形式化验证，并通过网络物理机器人案例展示了实际应用。

Conclusion: HELIX证明了代数变换与形式化验证相结合的方法能够实现高性能高保证的代码生成，为关键系统的可靠高效实现提供了可行方案。

Abstract: This paper presents the design of HELIX, an end-to-end verified code generation system with a focus on the intersection of high-performance and high-assurance numerical computing. The code generation can be fine-tuned to generate efficient code for a broad set of computer architectures while providing formal guarantees of the correctness of such generated code.
  Using a real-life example of a cyber-physical robot system, this paper demonstrates how, by using HELIX, one can start from a high-level mathematical formulation of the problem, apply a series of algebraic transformations that target intermediate languages, and generate an efficient imperative implementation. This is done while formally verifying semantic preservation from the original formulation down to LLVM IR.
  The method we used for high-performance code compilation is the algebraic transformation of vector and matrix computations into a dataflow optimised for parallel or vectorised processing on target hardware. The abstraction used to formalise and verify this technique is an operator language and accompanying semantics-preserving term rewriting. We use sparse vector abstraction to represent partial computations, enabling us to use algebraic reasoning to prove parallel decomposition properties.
  HELIX's verification infrastructure comprises multiple intermediate languages and verification approaches, all implemented in the Coq proof assistant. In particular, it uses verified term rewriting, translation validation, metaprogramming, verified compilation, layered monadic interpreters; it also supports application-specific uses of (verified) numerical analysis as we demonstrate via the running example.

### [3] [Compile to Compress: Boosting Formal Theorem Provers by Compiler Outputs](https://arxiv.org/abs/2604.18587)
*Guchan Li,Rui Tian,Hongning Wang*

Main category: cs.LG

TL;DR: 提出一种基于学习-精炼框架的高效定理证明方法，利用编译器将多样证明尝试映射到结构化失败模式的特性，通过局部纠错避免长历史累积，在有限计算预算下达到SOTA性能。


<details>
  <summary>Details</summary>
Motivation: 当前大型语言模型在形式定理证明中需要大量测试时计算（大规模展开或长上下文窗口），存在可扩展性瓶颈。本文旨在解决这一瓶颈。

Method: 引入学习-精炼框架，利用编译器将多样证明尝试压缩为结构化失败模式的特性。采用树搜索方法，基于显式验证器反馈进行局部纠错，避免累积长证明历史带来的成本。

Result: 该方法在不同规模上持续增强基础证明器的推理能力。在PutnamBench上，在可比较的测试时预算下，使用约8B和约32B参数模型达到了公开报告中的最先进性能。

Conclusion: 该方法为下一代验证器引导推理提供了一个可扩展的范式，通过利用形式验证中的信息结构，有效解决了定理证明中的计算效率问题。

Abstract: Large language models (LLMs) have demonstrated significant potential in formal theorem proving, yet state-of-the-art performance often necessitates prohibitive test-time compute via massive roll-outs or extended context windows. In this work, we address this scalability bottleneck by exploiting an informative structure in formal verification: the observation that compilers map a vast space of diverse proof attempts to a compact set of structured failure modes. We introduce a learning-to-refine framework that leverages this compression to perform efficient learning and proof exploration. We perform tree search that corrects errors locally conditioned on explicit verifier feedback, thereby circumventing the costs associated with accumulating a long history of proof attempts. Extensive evaluations show that our method consistently amplifies the reasoning capabilities of base provers across varying scales. Notably, our approach achieves state-of-the-art performance on PutnamBench among publicly reported $\sim$8B and $\sim$32B parameter models under comparable test-time budgets, offering a scalable paradigm for next-generation verifier-guided reasoning.

### [4] [Formally Verified Patent Analysis via Dependent Type Theory: Machine-Checkable Certificates from a Hybrid AI + Lean 4 Pipeline](https://arxiv.org/abs/2604.18882)
*George Koomullil*

Main category: cs.AI

TL;DR: 首个基于依赖类型理论的交互式定理证明框架，用于专利分析，将专利权利要求编码为DAG，通过形式化验证确保算法正确性。


<details>
  <summary>Details</summary>
Motivation: 现有专利分析方法依赖人工专家分析（慢、不可扩展）或ML/NLP方法（概率性、不透明、不可组合），需要形式化验证的框架来提供数学保证。

Method: 混合AI+Lean 4管道，将权利要求编码为DAG，匹配强度作为验证完备格的元素，置信度通过已验证的单调函数在依赖关系中传播，形式化六个算法处理五个IP用例。

Result: DAG覆盖核心算法完全机器验证，结构引理、覆盖核心生成器和闭合路径恒等式在Lean 4中验证，高层定理保持非正式证明草图，ML层输出经过内核检查和公理审计。

Conclusion: 这是首个将交互式定理证明应用于知识产权分析的框架，为专利分析提供条件性数学保证，但验证依赖于ML层输出的准确性，需要未来在裁决案例上进行验证。

Abstract: We present a formally verified framework for patent analysis as a hybrid AI + Lean 4 pipeline. The DAG-coverage core (Algorithm 1b) is fully machine-verified once bounded match scores are fixed. Freedom-to-operate, claim-construction sensitivity, cross-claim consistency, and doctrine-of-equivalents analyses are formalized at the specification level with kernel-checked candidate certificates. Existing patent-analysis approaches rely on manual expert analysis (slow, non-scalable) or ML/NLP methods (probabilistic, opaque, non-compositional). To our knowledge, this is the first framework that applies interactive theorem proving based on dependent type theory to intellectual property analysis. Claims are encoded as DAGs in Lean 4, match strengths as elements of a verified complete lattice, and confidence scores propagate through dependencies via proven-correct monotone functions. We formalize five IP use cases (patent-to-product mapping, freedom-to-operate, claim construction sensitivity, cross-claim consistency, doctrine of equivalents) via six algorithms. Structural lemmas, the coverage-core generator, and the closed-path identity coverage = W_cov are machine-verified in Lean 4. Higher-level theorems for the other use cases remain informal proof sketches, and their proof-generation functions are architecturally mitigated (untrusted generators whose outputs are kernel-checked and sorry-free axiom-audited). Guarantees are conditional on the ML layer: they certify mathematical correctness of computations downstream of ML scores, not the accuracy of the scores themselves. A case study on a synthetic memory-module claim demonstrates weighted coverage and construction-sensitivity analysis. Validation against adjudicated cases is future work.

### [5] [ARGUS: Agentic GPU Optimization Guided by Data-Flow Invariants](https://arxiv.org/abs/2604.18616)
*Haohui Mai,Xiaoyan Guo,Xiangyun Ding,Daifeng Li,Qiuchu Yu,Chenzhun Guo,Cong Wang,Jiacheng Zhao,Christos Kozyrakis,Binhang Yuan*

Main category: cs.DC

TL;DR: Argus是一个基于LLM的GPU内核代码生成框架，通过数据流不变量和编译时验证实现接近手工优化性能，在矩阵乘法、注意力等关键计算上达到SOTA 99-104%的性能。


<details>
  <summary>Details</summary>
Motivation: 现有LLM生成的GPU内核虽然功能正确，但性能远低于手工优化库，因为需要协调多个紧密耦合的优化（分块、共享内存、软件流水线等），而现有代理只能获得稀疏的通过/失败反馈，无法诊断全局约束违规。

Method: 提出数据流不变量：编译时规范编码数据在核函数执行中的编排方式。使用基于分块的Python DSL暴露硬件指令和编译器策略，同时隐藏低级表示。通过标签函数传播符号注释，标签断言在用例处强制执行关系约束。编译器在违规时返回具体反例，提供密集结构化反馈。通过布局代数的抽象解释和SMT求解进行编译时验证，零运行时开销。使用上下文强化学习规划器选择优化并合成有效不变量。

Result: 在AMD MI300X GPU上评估GEMM、flash attention和MoE内核（占LLM推理90%以上GPU时间），生成的内核达到SOTA手工优化汇编吞吐量的99-104%，比现有代理系统快2-1543倍。在200个KernelBench任务中，解决100%的Level 1和90%的Level 2问题。

Conclusion: Argus通过数据流不变量和编译时验证，使LLM代理能够生成接近手工优化性能的GPU内核，显著提升了自动代码生成的性能水平，并展示了在复杂GPU优化任务上的强大泛化能力。

Abstract: LLM-based coding agents can generate functionally correct GPU kernels, yet their performance remains far below hand-optimized libraries on critical computations such as matrix multiplication, attention, and Mixture-of-Experts (MoE). Peak GPU performance requires coordinated reasoning over tightly coupled optimizations, including tiling, shared-memory staging, software pipelining, and instruction scheduling, while existing agents rely on sparse pass/fail feedback, leaving them unable to diagnose global constraint violations.
  We present Argus, an agentic framework that addresses this through data-flow invariants: compile-time specifications encoding how data must be choreographed throughout kernel execution. Argus introduces a tile-based, Pythonic DSL exposing hardware instructions and compiler policies while hiding low-level representations. The DSL provides tag functions to propagate symbolic annotations through data and control flow, and tag assertions to enforce relational constraints at use sites. When violations occur, the compiler returns concrete counterexamples identifying the thread, data element, and program point, enabling dense, structured feedback for targeted fixes. Invariants are verified at compile time via abstract interpretation over a layout algebra and SMT solving, with zero runtime overhead. An in-context reinforcement learning planner learns to select optimizations and synthesize effective invariants, supported by a curated knowledge base of GPU optimization techniques.
  We evaluate Argus on the AMD MI300X GPU across GEMM, flash attention, and MoE kernels accounting for over 90% of GPU time in LLM inference. Generated kernels achieve 99-104% of state-of-the-art hand-optimized assembly throughput and are 2-1543x faster than existing agentic systems. Argus further generalizes to 200 KernelBench tasks, solving 100% of Level 1 and 90% of Level 2 problems.

### [6] [Adding Compilation Metadata To Binaries To Make Disassembly Decidable](https://arxiv.org/abs/2604.19628)
*Daniel Engel,Freek Verbeek,Pranav Kumar,Binoy Ravindran*

Main category: cs.CR

TL;DR: 提出一种在二进制文件中嵌入编译器意图元数据的格式，使二进制文件既能保持不透明性，又能支持安全分析和重新编译，不影响运行时性能。


<details>
  <summary>Details</summary>
Motivation: 二进制文件是软件分发和执行的通用格式，但也是最不透明的软件表示形式。如果能在二进制文件中添加安全相关的元数据（如编译器预期的可执行指令数据、内存区域边界等），将极大提高软件的安全性和可维护性。

Method: 提出一种介于剥离的黑盒二进制和开源代码之间的二进制格式。开发工具生成捕获编译器意图的元数据，并将其嵌入到二进制文件中。这些元数据支持将二进制提升为正确且可重新编译的高级表示，使分析和插桩更加可靠。

Result: 添加元数据不影响运行时行为或性能。相比DWARF调试信息，元数据大小仅为其17%。通过编译大量真实世界的C和C++二进制文件验证了正确性，这些文件可以被提升、插桩和重新编译而不改变其行为。

Conclusion: 提出的二进制格式在保持二进制不透明性的同时，通过嵌入编译器意图元数据，显著提升了软件的安全性、可维护性和可分析性，为二进制软件的安全增强提供了实用解决方案。

Abstract: The binary executable format is the standard method for distributing and executing software. Yet, it is also as opaque a representation of software as can be. If the binary format were augmented with metadata that provides security-relevant information, such as which data is intended by the compiler to be executable instructions, or how memory regions are expected to be bounded, that would dramatically improve the safety and maintainability of software. In this paper, we propose a binary format that is a middle ground between a stripped black-box binary and open source. We provide a tool that generates metadata capturing the compiler's intent and inserts it into the binary. This metadata enables lifting to a correct and recompilable higher-level representation and makes analysis and instrumentation more reliable. Our evaluation shows that adding metadata does not affect runtime behavior or performance. Compared to DWARF, our metadata is roughly 17% of its size. We validate correctness by compiling a comprehensive set of real-world C and C++ binaries and demonstrating that they can be lifted, instrumented, and recompiled without altering their behavior.

</details>
