### [1] [Linear Constraints](https://arxiv.org/abs/2604.21467)
*Arnaud Spiwack,Csongor Kiss,Jean-Philippe Bernardy,Nicolas Wu,Richard A. Eisenberg*

Main category: cs.PL

TL;DR: 本文提出线性约束（Linear Constraints）作为Haskell中类约束的线性版本，允许编译器自动填充隐式线性参数，从而减少手动管理资源的代码繁琐性。该系统基于限定类型系统，并扩展了GHC现有的约束求解器算法，通过脱糖到Linear Haskell保证其健全性。本文是先前论文的修订扩展版，简化了形式系统和约束求解器，并新增了多个应用案例。


<details>
  <summary>Details</summary>
Motivation: 线性类型系统能验证资源（如文件句柄、手动管理的内存）的安全使用，但显式线性参数编写代码过于繁琐（"bureaucracy"）。线性约束旨在解决这一缺陷，将隐式线性参数自动填充，减少程序员负担。

Method: 将线性约束作为限定类型系统（qualified type system）实现，扩展GHC现有约束求解器算法以支持线性约束的推理。线性约束最终脱糖（desugar）为Linear Haskell，从而保证其健全性（soundness）。

Result: 相比前版论文，形式系统和约束求解器得到显著简化，并描述了更多应用场景。

Conclusion: 线性约束是一种实用的语言特性，在保留线性类型资源安全保障的同时，降低了编程开销，且可通过现有编译器框架实现。

Abstract: Linear constraints are the linear counterpart of Haskell's class constraints. Linearly typed parameters allow the programmer to control resources such as file handles and manually managed memory as linear arguments. Indeed, a linear type system can verify that these resources are used safely. However, writing code with explicit linear arguments requires bureaucracy. Linear constraints address this shortcoming: a linear constraint acts as an implicit linear argument that can be filled in automatically by the compiler.
  We present this new feature as a qualified type system, together with an inference algorithm which extends GHC's existing constraint solver algorithm. Soundness of linear constraints is ensured by the fact that they desugar into Linear Haskell.
  This paper is a revised and extended version of a previous paper by the same authors (arXiv:2103.06127). The formal system and the constraint solver have been significantly simplified and numerous additional applications are described.

</details>


### [2] [NEST: Network Enforced Session Types (Technical Report)](https://arxiv.org/abs/2604.21795)
*Jens Kanstrup Larsen,Alceste Scalas,Guy Amir,Jules Jacobs,Jana Wagemaker,Nate Foster*

Main category: cs.PL

TL;DR: 本文提出NEST，一种将应用层协议监控下沉到网络数据平面的运行时验证框架。通过P4实现，能直接从会话类型合成网络级监控器，处理数据包丢失和乱序，在微服务和网络功能场景中验证了非平凡协议。


<details>
  <summary>Details</summary>
Motivation: 现有协议监控方法需要修改或包装应用程序代码，缺乏将监控直接嵌入网络基础设施的方案，难以实现透明、高效的应用层协议验证。

Method: 提出NEST框架，设计从会话类型合成数据包级监控器的算法，扩展该算法以容忍数据包丢失和乱序，并在P4可编程数据平面中实现。

Result: 在微服务和网络功能模型上验证，表明网络级监控器能够有效执行现实中非平凡的协议约束。

Conclusion: 将协议监控直接嵌入网络数据平面是可行且有效的，能够在不修改应用代码的前提下实现运行时协议合规性验证。

Abstract: This paper introduces NEST (Network-Enforced Session Types), a runtime verification framework that moves application-level protocol monitoring into the network fabric. Unlike prior work that instruments or wraps application code, we synthesize packet-level monitors that enforce protocols directly in the data plane. We develop algorithms to generate network-level monitors from session types and extend them to handle packet loss and reordering. We implement NEST in P4 and evaluate it on applications including microservice and network-function models, showing that network-level monitors can enforce realistic non-trivial protocols.

</details>

### [3] [Trustworthy Clinical Decision Support Using Meta-Predicates and Domain-Specific Languages](https://arxiv.org/abs/2604.21263)
*Michael Bouzinier,Sergey Trifonov,Michael Chumack,Eugenia Lvova,Dmitry Etin*

Main category: cs.AI

TL;DR: 本文提出元谓词（meta-predicates）方法，通过设计契约（design-by-contract）原则，在临床决策规则的DSL中嵌入认识论约束（epistemological constraints），实现对决策证据类型的事前验证。在基因组变异解读场景中，该方法能将决策树转化为单级联（unate cascade）形式，生成可审计的决策路径，并捕获人类编写或AI生成规则中的认识论错误。


<details>
  <summary>Details</summary>
Motivation: 现有AI医疗监管框架（如EU AI Act、FDA指南）要求临床决策支持系统不仅准确，还要可审计。但现有形式化语言只验证语法的结构性正确性，无法检查决策规则是否使用了认识论上恰当的证据。需要一种能在部署前约束证据类型的方法。

Method: 基于设计契约（design-by-contract）原则，引入元谓词（meta-predicates，即关于谓词的谓词）来断言领域特定语言（DSL）中临床决策规则的认识论约束。建立认识论类型系统，沿四个维度（目的、知识领域、规模、获取方法）对标注进行分类。元谓词规定哪些证据类型在给定规则中是允许的。在AnFiSA平台中实现，使用Brigham基因组医学协议对Genome in a Bottle基准中的560万变异进行验证。

Result: 变异解读中的决策树可重构为单级联（unate cascade），从而为每个变异生成审计轨迹，识别哪个规则分类了该变异及原因。元谓词验证能在部署前捕获认识论错误，无论规则是人类编写还是AI生成。该方法与LIME、SHAP等事后解释方法互补：解释方法事后说明使用了什么证据，元谓词事先约束哪些证据可以被使用，同时保持人类可读性。

Conclusion: 元谓词验证是向展示决策不仅准确且基于适当证据（并可独立审计）迈出的一步。虽然以基因组学为例，该方法可推广到任何需要可审计决策逻辑的领域。

Abstract: \textbf{Background:} Regulatory frameworks for AI in healthcare, including the EU AI Act and FDA guidance on AI/ML-based medical devices, require clinical decision support to demonstrate not only accuracy but auditability. Existing formal languages for clinical logic validate syntactic and structural correctness but not whether decision rules use epistemologically appropriate evidence.
  \textbf{Methods:} Drawing on design-by-contract principles, we introduce meta-predicates -- predicates about predicates -- for asserting epistemological constraints on clinical decision rules expressed in a DSL. An epistemological type system classifies annotations along four dimensions: purpose, knowledge domain, scale, and method of acquisition. Meta-predicates assert which evidence types are permissible in any given rule. The framework is instantiated in AnFiSA, an open-source platform for genetic variant curation, and demonstrated using the Brigham Genomics Medicine protocol on 5.6 million variants from the Genome in a Bottle benchmark.
  \textbf{Results:} Decision trees used in variant interpretation can be reformulated as unate cascades, enabling per-variant audit trails that identify which rule classified each variant and why. Meta-predicate validation catches epistemological errors before deployment, whether rules are human-written or AI-generated. The approach complements post-hoc methods such as LIME and SHAP: where explanation reveals what evidence was used after the fact, meta-predicates constrain what evidence may be used before deployment, while preserving human readability.
  \textbf{Conclusions:} Meta-predicate validation is a step toward demonstrating not only that decisions are accurate but that they rest on appropriate evidence in ways that can be independently audited. While demonstrated in genomics, the approach generalises to any domain requiring auditable decision logic.

</details>

### [4] [The Path Not Taken: Duality in Reasoning about Program Execution](https://arxiv.org/abs/2604.20917)
*Eshgin Hasanov,Md Mahadi Hassan Sibat,Santu Karmaker,Aashish Yadavally*

Main category: cs.LG

TL;DR: 现有基准主要关注与特定输入相关的程序属性预测（如代码覆盖率、程序输出），不能全面评估模型对程序执行的真正理解。本文提出通过两个互补推理任务（预测程序行为和推断输入如何改变以达到特定目标）来评估模型对程序执行因果流的理解，构建了DexBench基准并评估了13个LLM，结果表明双路径推理能提供对动态代码理解的鲁棒且可区分的代理。


<details>
  <summary>Details</summary>
Motivation: 当前LLM在编码任务中表现优异，但其是否真正理解程序执行而非依赖表面模式存疑。现有基准仅关注特定输入下的程序属性预测，视角狭窄且易受数据污染，无法全面评估模型对程序执行因果流的理解。

Method: 提出程序执行理解的双路径推理框架：(i)预测给定输入下程序的观察行为，(ii)推断输入应如何变化以实现特定行为目标。基于此构建DexBench基准，包含445对实例，评估13个LLM。

Result: 双路径推理为动态代码理解提供了鲁棒且可区分的代理指标。

Conclusion: 需要更全面的评估框架来真正衡量LLM对程序执行的理解，双路径推理是一个有效的方向。

Abstract: Large language models (LLMs) have shown remarkable capabilities across diverse coding tasks. However, their adoption requires a true understanding of program execution rather than relying on surface-level patterns. Existing benchmarks primarily focus on predicting program properties tied to specific inputs (e.g., code coverage, program outputs). As a result, they provide a narrow view of dynamic code reasoning and are prone to data contamination. We argue that understanding program execution requires evaluating its inherent duality through two complementary reasoning tasks: (i) predicting a program's observed behavior for a given input, and (ii) inferring how the input must be mutated toward a specific behavioral objective. Both tasks jointly probe a model's causal understanding of execution flow. We instantiate this duality in DexBench, a benchmark comprising 445 paired instances, and evaluate 13 LLMs. Our results demonstrate that dual-path reasoning provides a robust and discriminative proxy for dynamic code understanding.

</details>
