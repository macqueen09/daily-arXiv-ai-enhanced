### [1] [Source-to-Source Transformations for GPU Code Generation](https://arxiv.org/abs/2605.13864)
*Julien de Castelnau,Thomas Koehler,Arthur Charguéraud,Clément Pit-Claudel*

Main category: cs.PL

TL;DR: OptiGPU 是一个通过保证明编译实现GPU安全编程的系统，能在最小化权衡的前提下提供数据竞争自由、死锁自由和完全功能正确性的强安全保证，并通过案例研究证明其能生成与手写参考匹配的CUDA代码。


<details>
  <summary>Details</summary>
Motivation: GPU编程面临并发bug的挑战，现有安全GPU编程方法在保证强度、底层控制程度和额外工作量三个维度上存在权衡，缺乏能同时优化这三个维度的方案。

Method: OptiGPU 采用保证明编译方法，通过精化简单、已验证的CPU程序来推导经过验证的低级优化GPU程序；用户通过自动保持证明的源码到源码转换来指导精化过程，避免直接验证复杂优化GPU程序的高证明负担。该框架基于OptiTrust扩展，支持内核启动、共享内存和同步屏障等GPU特性。

Result: 在矩阵转置和基于树的并行归约两个案例研究中，OptiGPU能推导出与手写参考代码技术匹配的CUDA代码，验证了其有效性和实用性。

Conclusion: OptiGPU通过保证明编译方法在GPU编程中实现了强安全保证与良好可用性的平衡，为安全GPU编程提供了新的有效途径，有望降低GPU并发编程的门槛和风险。

Abstract: GPUs have become essential in modern high performance computing, but programming them correctly remains a significant challenge. This difficulty arises from subtle concurrency bugs that result from the explicit management of synchronization primitives and data movement across intricate hierarchies of memory and parallel threads. At the same time, the ability to control these aspects explicitly is at the core of the performance gains granted by GPUs. These challenges have motivated interest in safe GPU programming: tools and languages that can prevent or detect such bugs statically. However, existing approaches make tradeoffs in three dimensions: the strength of their guarantees, the degree of low-level control they allow, and the amount of additional effort required to achieve these safety guarantees. This thesis presents OptiGPU, a system for GPU programming with strong safety guarantees-data race freedom, deadlock freedom, and full functional correctness-that minimizes tradeoffs in the other two dimensions compared to previous approaches. OptiGPU applies proof-preserving compilation to GPU programming, allowing verification of low-level, optimized GPU programs through refinement of simple, verified CPU programs. An OptiGPU user thus avoids the substantial proof burden of directly verifying complex optimized GPU programs, instead directing this refinement with source-to-source transformations that automatically preserve proofs. OptiGPU is implemented as a set of extensions to OptiTrust, an existing framework for proof-preserving compilation on CPUs. OptiGPU models essential GPU programming features, including kernel launches, shared memory, and synchronous barriers, and produces both device and host-side code. We evaluate OptiGPU on two case studies, matrix transpose and tree-based parallel reduction, showing it can derive CUDA code matching techniques found in handwritten references.

### [2] [Quantitative Symbolic Patch Impact Analysis](https://arxiv.org/abs/2605.13885)
*Laboni Sarker,Abdus Satter,Tevfik Bultan*

Main category: cs.PL

TL;DR: 本文提出定量部分等价分析（Quantitative Partial Equivalence Analysis），通过符号分析量化补丁前后程序在输入域上的行为差异，在90个CVE补丁和Juliet测试集上验证了有效性，并在EqBench中发现5对误标为等价的程序。


<details>
  <summary>Details</summary>
Motivation: 传统等价性检查仅能判断程序是否等价，无法在补丁必然导致行为变化（如漏洞修复场景下）时提供更多信息。作者希望不仅知道程序不等价，还能量化它们在多大比例的输入上存在差异，以及具体的差异条件。

Method: 使用符号分析识别输入条件，将输入域划分为等价区域和差异区域；提出基于范围搜索的启发式方法（range-based search heuristic），为数值域上的等价性提供可靠的下界估计，从而高效量化行为差异程度。

Result: 在90个真实CVE补丁（来自Linux、Qemu、FFmpeg等）及Juliet测试集上验证了方法有效性；在EqBench基准测试中发现5对C程序被误标为等价，并找到了它们行为差异的具体输入条件。

Conclusion: 定量部分等价分析能有效刻画和量化补丁影响，为补丁影响分析、回归测试等任务提供比传统等价性检查更丰富的信息。

Abstract: Traditional equivalence checking classifies programs as equivalent or non-equivalent, providing insufficient information for tasks like patch impact analysis where it is expected the patched version of the program to be non-equivalent to the original program. When two program versions are non-equivalent, determining under what conditions they differ and what percentage of inputs are affected remains an open challenge. In this work, we introduce quantitative partial equivalence analysis, an approach for assessing software patches by quantifying behavioral differences between the original (vulnerable) code and the patched code. Using symbolic analysis, we identify input conditions under which patched and original programs exhibit identical or divergent behaviors. Our approach refines non-equivalence by measuring the extent of behavioral divergence across the input domain. For efficient quantitative analysis of numerical domains, we propose a range-based search heuristic that provides a sound lower bound on equivalence. We demonstrate our approach on 90 CVE patches from widely used open-source projects (Linux, Qemu, FFmpeg), as well as on a Juliet Test Suite-based dataset containing programs with CWEs. Our results show that quantitative partial equivalence analysis effectively characterizes and quantifies patch impact. Additionally, experiments on the EqBench benchmark reveal five C program pairs that are mislabeled as equivalent, and we identify the input conditions under which their behaviors diverge.

### [3] [Linear-Time T-Gate Optimization via Random Abstraction](https://arxiv.org/abs/2605.13929)
*Aws Albarghouthi*

Main category: cs.PL

TL;DR: 本文提出了一种线性时间随机化相位折叠算法（TZAP），通过新颖的静态分析（传播定宽比特串而非符号表达式），在T门优化上比现有工具（PyZX、VOQC、Feynman）快多个数量级，且能在秒级内优化数百万门规模的电路，T计数减少效果与最先进工具相当。


<details>
  <summary>Details</summary>
Motivation: 量子计算优势的实现依赖于容错，而T门在大多数容错方案中不能横向实现，需要通过昂贵的魔法态蒸馏协议，导致T门计数成为大规模量子计算资源预算的主要瓶颈。现有T门优化工具无法扩展到量子优势所需的电路规模。

Method: 提出一种线性时间随机化相位折叠算法。核心创新在于提出一种新颖的随机化静态分析：不追踪符号表达式，而是沿电路传播定宽比特串，以任意高概率可靠地逼近可达量子状态集合。

Result: 实现工具TZAP，在标准基准测试上T计数减少与最先进工具（PyZX、VOQC、Feynman）相当，但速度快多个数量级，能在笔记本电脑上秒级优化包含数百万门的电路。

Conclusion: 这项工作和TZAP工具使得大规模量子电路的T门优化变得实际可行，消除了通往量子优势道路上的一个关键瓶颈。

Abstract: Quantum computers promise exponential speedups for problems in cryptography, chemistry, and optimization. Realizing this promise requires fault tolerance: physical qubits are noisy, so logical qubits must be encoded redundantly across many physical ones using quantum error-correcting codes. In most practical fault-tolerance schemes, T gates cannot be implemented transversally and instead require costly magic-state distillation protocols involving a complex set of operations. As a result, T-gate count can dominate the resource budget of large-scale quantum computations, making T-count minimization a central bottleneck on the path to quantum advantage. Existing T-count optimization tools, however, do not scale to the circuits that quantum advantage demands.
  We present theoretical and practical results on T-gate optimization. On the theoretical side, we give a linear-time randomized algorithm for phase folding, based on a novel randomized static analysis. Our static analysis soundly approximates the set of reachable quantum states with an arbitrarily high probability. Our key insight is a static analysis that does not track symbolic expressions, but propagates constant-width bitstrings down the circuit. On the practical side, our implementation, TZAP, is multiple orders of magnitude faster than state-of-the-art tools -- such as PyZX, VOQC, and Feynman -- closely matches their T-count reductions on standard benchmarks, and within seconds on a laptop computer can optimize circuits with millions of gates.

### [4] [Mat2Boundary: Treating User-Defined Boundary Condition as SpMV for Distributed PDE Solvers on Block-Structured Grids](https://arxiv.org/abs/2605.14780)
*Yanzheng Cai,Mingzhe Zhang,Shengqi Chen,Haoyuan Song,Wenguang Chen*

Main category: cs.PL

TL;DR: 本文提出Mat2Boundary，一个用于边界条件计算的DSL和编译器，将边界条件建模为仿射稀疏线性算子，在结构化/块结构化网格上实现高效、可组合的边界计算，在浅水方程求解器和HPCG上取得最高7.6倍加速，代码量减少70%以上，并扩展到1344个CPU核心。


<details>
  <summary>Details</summary>
Motivation: 边界条件处理是PDE求解器中复杂性的主要来源，尤其是在高阶方法和分布式内存执行场景下，现有方法缺乏统一、高效的抽象。

Method: 将边界条件统一建模为仿射稀疏线性算子，暴露模块化的基本子矩阵接口，结合多阶段编程和多面体分析生成矩阵无关内核，支持用户自定义稀疏矩阵，消除冗余边界计算，合成可复用的分布式通信调度。

Result: 在球面立方体网格上的浅水方程求解器和HPCG中，BC内核加速最高7.6倍，BC代码减少超过70%，可扩展到1344个CPU核心，效率为72%-88%。

Conclusion: Mat2Boundary通过统一抽象和编译优化，有效降低了边界条件处理的复杂性，显著提升了性能和代码简洁性，并支持大规模分布式执行。

Abstract: Boundary-condition (BC) handling is a major source of complexity in PDE solvers on structured and block-structured grids, especially for high-order methods and distributed-memory execution. We present Mat2Boundary, a DSL and compiler for boundary computations that models a broad class of boundary-conditions as affine sparse linear operators. This abstraction unifies halo copying, circular and symmetric mappings, zero padding, block-edge synchronization, and user-defined interpolation, while exposing a modular basic sub-matrix interface for declarative composition. To make this representation efficient, Mat2Boundary combines multi-stage programming and polyhedral analysis to generate matrix-free kernels for structured cases, support user-defined sparse matrices for irregular cases, eliminate redundant boundary work, and synthesize reusable communication schedules for distributed execution. Evaluated on two shallow-water equation solvers on cubed-sphere grids and HPCG, Mat2Boundary achieves up to 7.6$\times$ BC-kernel speedup, reduces BC code by over 70%, and scales to 1,344 CPU cores with 72%-88% efficiency.

*Ruotong Cheng,Azadeh Farzan*

Main category: cs.LO

TL;DR: 本文研究无限状态参数化并发程序在丰富通信拓扑下的算法安全验证问题，通过将问题简化为组合方案（局部证明的集合）来实现完全验证，并提出了一个有效的验证工具。


<details>
  <summary>Details</summary>
Motivation: 无限状态参数化并发程序在复杂通信拓扑下的安全验证是一个挑战性问题，现有方法难以自动生成正确的归纳不变量证明。

Method: 将参数化程序的验证问题简化为一组局部证明的组合方案，基于对底层拓扑的合理假设，自动生成全称量化归纳不变量。

Result: 提出了一个验证算法并实现为工具，在多种不同拓扑的基准测试上证明了方法的有效性。

Conclusion: 该方法能够以完全的方式自动证明参数化并发程序的安全性，且在多种拓扑结构下均表现良好。

Abstract: This paper investigates the algorithmic safety verification problem of infinite-state parameterized concurrent programs over a rich set of communication topologies. The goal is to automatically produce a proof of correctness in the form of a universally quantified inductive invariant, where the quantification is over the nodes in the topology. We illustrate that under reasonable assumptions on the underlying topology, the problem can be reduced to and solved as a compositional scheme, that is, the verification of the parameterized family is reduced to a set of local proofs, in a complete manner. We propose a verification algorithm, which is implemented as a tool, and demonstrate through a set of benchmarks over several different topologies that our approach is effective in proving parameterized programs safe.

### [6] [Neural Code Translation of Legacy Code: APL to C#](https://arxiv.org/abs/2605.13896)
*Abdulrahman Ramadan,Hanen Borchani,Iben Lilholm,Mikkel Almind,Allan Peter Engsig-Karup*

Main category: cs.SE

TL;DR: 本文研究了使用大语言模型将APL（一种语法简洁的编程语言）自动翻译为C#的问题，并提出了一种包含自然语言描述中介、检索增强和迭代优化三种引导策略的框架。实验表明，这些方法能有效提升翻译质量，尤其在处理APL稀疏语法和缺乏大规模平行语料等挑战时表现优异。


<details>
  <summary>Details</summary>
Motivation: 编程语言之间的自动翻译仍然是一个具有挑战性的问题，尤其是当源语言（APL）语法极为简洁且专业化，缺乏大规模平行语料库，且需要专业知识才能解读APL程序时。现有方法难以直接应对这些困难。

Method: 提出一个APL到C#翻译框架，比较了三种引导策略：自然语言描述中介（先翻译为自然语言描述，再生成C#）、检索增强（引入外部上下文信息）和迭代优化（逐步改进翻译结果），并以直接翻译模型作为基线。构建了多个不同复杂度的功能等价代码对数据集，并开发了自动评估流水线，同时验证生成C#代码的语法编译和功能执行正确性。

Result: 神经代码翻译能够成功弥合APL和C#之间的差距，涵盖多种程序类型；引入额外上下文和引导策略显著提升了模型性能。

Conclusion: 本文验证了利用大语言模型结合引导策略进行APL到C#翻译的有效性，证明了在缺乏大规模平行语料的情况下，通过增加上下文信息和多阶段引导可以显著改善翻译质量，为处理高度专业化语言的自动翻译提供了可行路径。

Abstract: Automatic translation between programming languages remains a challenging problem, particularly when the source language is highly concise and specialized. This paper investigates the translation of APL into C# using large language models. The task is difficult due to APL's sparse syntax, the scarcity of large-scale parallel corpora, and the requirement for specialized knowledge to interpret APL programs. To address these challenges, we introduce a novel framework for APL-to-C# translation by comparing three guided strategies, namely natural language description-mediated, retrieval-augmented, and iterative refinement, against a baseline direct translation model. We constructed multiple datasets of functionally equivalent code pairs spanning various levels of complexity, and to rigorously assess translation quality, we developed an automated evaluation pipeline that verifies both syntactic compilation and functional execution of the generated C# code. Our results demonstrate that neural code translation can successfully bridge the gap between APL and C# for a wide range of programs, and that incorporating additional context and guidance significantly improves model performance.
