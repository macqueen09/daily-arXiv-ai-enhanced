### [2] [Ordered Adjoint Logic (Extended Version)](https://arxiv.org/abs/2605.19112)
*Sophia Roshal,Frank Pfenning*

Main category: cs.LO

TL;DR: 本文通过adjoint modalities将不同结构性质的逻辑（如weakening, contraction, mobility）组合到有序逻辑中，扩展了Kanovich等人的工作，提出了支持cut elimination的sequent calculus和可判定proof checking的natural deduction系统，为表达性强的adjoint编程语言或逻辑框架奠定基础。


<details>
  <summary>Details</summary>
Motivation: 已有有序逻辑通常要求资源恰好使用一次（线性约束），Kanovich等人通过subexponentials放宽了这一限制，但仍有局限。作者希望进一步扩展，通过adjoint modalities灵活组合具有不同细粒度结构性质（如弱化、收缩、移动性）的逻辑。

Method: 使用adjoint modalities来组合具有不同结构性质（weakening, left/right contraction, left/right mobility）的逻辑。在sequent calculus中证明cut elimination，并给出natural deduction formulation使得结构规则隐式化，同时证明proof checking的可判定性。

Result: 所提出的sequent calculus支持cut elimination；natural deduction formulation中结构规则隐式化且proof checking可判定。

Conclusion: 该系统适合作为表达性强的adjoint编程语言或逻辑框架的基础。

Abstract: Ordered logics and type systems have been used in a variety of applications
  including computational linguistics, memory allocation, stream processing,
  logical frameworks, parametricity, and enforcing security protocols. In most
  formulations, ordered types are also linear, requiring each resource to be
  used exactly once. Prior work by Kanovich et al. has investigated calculi
  that relax this constraint through subexponentials within a linear ordered
  logic. We generalize their work by using adjoint modalities to combine logics
  with varying fine-grained structural properties, including weakening, left
  contraction, right contraction, left mobility, and right mobility. We show
  that the resulting sequent calculus admits cut elimination.
  We further provide a natural deduction formulation in which structural rules
  are implicit, and show that proof checking for this system is decidable. This
  makes it a suitable foundation for an expressive adjoint programming language
  or logical framework.

</details>

### [3] [Cypher is Turing-Complete: A Formal Proof via 2-Counter Machine Simulation](https://arxiv.org/abs/2605.18757)
*Pierre Halftermeyer*

Main category: cs.CC

TL;DR: 本文证明了Neo4j的图查询语言Cypher 25是图灵完备的，通过三种不同构造方式模拟任意2-计数器机器，并在真实实例上验证。


<details>
  <summary>Details</summary>
Motivation: 探究Cypher图查询语言是否具有图灵完备性，厘清其计算能力边界。

Method: 使用单一RETURN语句结合reduce()、CASE表达式和列表推导式模拟2-计数器机器；通过两种互补方案解决有界步骤异议；提出基于量化路径模式(QPP)和allReduce的图原生模拟方法。

Result: 所有三种构造均在Neo4j Aura真实实例上成功验证，证明Cypher 25确实具有图灵完备性。

Conclusion: Cypher 25具备通用计算能力，可以模拟任意计算过程，这对图查询语言的理论基础和实践应用具有重要意义。

Abstract: We prove that Cypher 25, the graph query language of Neo4j, is Turing-complete. The proof shows that a single RETURN statement using reduce(), CASE expressions, and list comprehensions can simulate any 2-counter machine (Minsky 1967). We address the bounded-step objection via two complementary resolutions and present a third graph-native simulation using quantified path patterns (QPP) with allReduce. All three constructions were verified on a live Neo4j Aura instance.

### [4] [Code-Guided Reasoning for Small Language Models: Evaluating Executable MCQA Scaffolds](https://arxiv.org/abs/2605.18827)

Main category: cs.IR

TL;DR: 提出Code-Guided Reasoning (CGR)评估协议，测试可执行推理脚手架（Python代码+工具）能否提升小型语言模型在多项选择题上的表现。实验显示辅助准确率66.21% vs 直接回答38.11%，提升28.10个百分点。


<details>
  <summary>Details</summary>
Motivation: 当前MCQA基准主要评估SLM作为直接回答者的能力，但实际部署的LM系统越来越多地依赖外部脚手架（工具、代码、多次模型调用）。现有评估方式与真实使用场景脱节，缺乏衡量推理脚手架提升效果的标准化方法。

Method: 提出CGR协议，标准化六个组件：规范化条目接口、直接求解提示、生成器提示、Python脚手架、求解器调用与提取辅助函数、三通道结果记录。在20,498条结果上对6个注册求解模型进行对比评估。

Result: 非零基线分区：辅助准确率66.21% vs 直接准确率38.11%，提升28.10个百分点（自助法置信区间[20.32, 36.43]）。在严格门限下（Ab > 30%）差异为+14.11个百分点。

Abstract: Multiple-choice QA benchmarks usually evaluate small language models (SLMs) as direct answerers, but deployed language-model systems increasingly rely on external scaffolds such as tools, code, and repeated model calls. We introduce Code-Guided Reasoning (CGR), an evaluation protocol and generated-program resource for measuring when executable reasoning scaffolds improve SLM performance on MCQA tasks. CGR standardizes six components: a normalized item interface, a direct solver prompt, a generator prompt, a Python scaffold, solver-call and extraction helpers, and a three-channel result record. On 20,498 retained result rows from a locally prepared MCQA bundle and six metadata-registered solver models, the observed non-zero-baseline partition shows 66.21% macro assisted accuracy versus 38.11% direct accuracy, a +28.10 percentage-point difference with a pair-bootstrap interval of [20.32, 36.43]. Under a stricter Ab > 30% direct-signal gate, the macro difference is +14.11 points. These estimates are descriptive. Assisted inference uses a larger solver-call budget, answer extraction is brittle, Time-MQA contains the observed regressions, and some generated programs violate the no-hard-coding instruction. CGR provides the trace package needed to interpret these results, including direct, assisted, and generator-side answers, partition definitions, generated programs, response metadata, and audits.

Main category: cs.SE

TL;DR: 提出了一种基于MAPE-K模型的模块化自愈框架，集成AutoFix机制实现自适应故障恢复。实验表明，该框架在故障检测F1-score达90.7%，恢复成功率93.2%，平均恢复时间3.92秒，吞吐量保持88%-95%，响应时间仅增加3.1%。


<details>
  <summary>Details</summary>

Method: 采用设计与发展研究（DDR）方法，基于MAPE-K模型构建模块化自愈框架，集成AutoFix自适应故障恢复机制，通过受控故障注入实验（涵盖服务崩溃、内存泄漏、数据库断连等20种场景）进行验证。

### [6] [DRReduce: Enhancing Syntax-Guided Program Reduction with Dependency Reconstruction](https://arxiv.org/abs/2605.19412)
*Qiong Feng,Xiaotian Ma,Yongqiang Tian,Wei Song,Peng Liang*

Main category: cs.SE

TL;DR: 本文提出DRReduce，一种通过依赖重构来增强语法引导的归约器的方法，在保持语言无关性的同时显著提升了程序归约的效果和效率。


<details>
  <summary>Details</summary>
Motivation: 现有语言无关的语法引导归约器（如Perses）在删除节点或子树时容易破坏程序语义一致性，导致属性检查器拒绝删除操作，限制了归约效果和效率。作者希望在不依赖特定语言转换规则的前提下，解决这一问题。

Method: DRReduce构建语义依赖图，执行语义连贯的删除操作并通过依赖重构（dependency reconstruction）修复因删除而破坏的程序依赖，然后将进一步最小化交由语法引导归约器处理。

Result: 相比Perses、WDD、CDD等SOTA语法引导归约器，DRReduce平均大小减少分别达51.9%、14.9%、19.8%，且大部分程序归约更快。与语言特定工具相比，DRReduce达到与CReduce和Latra相当的效果，效率分别是它们的3.3倍和1.2倍。消融实验显示依赖重构减少了80.2%的查询调用、58.7%的归约时间以及55.1%的最终token数。

Conclusion: DRReduce通过轻量级语义层有效弥合了语言无关语法归约与语义一致性之间的鸿沟，在不牺牲语言通用性的前提下大幅提升归约性能。

Abstract: Program reduction is a technique for simplifying large, failure-inducing programs into minimal reproducible test cases. Language-specific tools such as CReduce achieve strong performance by leveraging deep semantic knowledge of C/C++, but are tightly coupled to a single language family. Language-agnostic reducers such as Perses address this by applying syntax-guided search across any grammar, yet share a fundamental limitation: deleting a node or subtree in isolation often breaks semantic coherence causing the property checker to reject the deletion and forcing the reducer to backtrack, limiting overall reduction effectiveness and efficiency. In this paper, we propose DRReduce, a framework that bridges this gap by augmenting language-agnostic syntactic reduction with a lightweight semantic layer: dependency reconstruction, which repairs program dependencies broken by a deletion in order to preserve the semantic validity of intermediate programs and increase the acceptance rate of the property checker. DRReduce constructs a semantic dependency graph from the input program, performs semantically coherent deletions with dependency reconstruction, and delegates further minimization to a syntax-guided reducer. We implement DRReduce for C and Java and evaluate it on real-world bug-triggering programs. Compared to SOTA syntax-guided reducers, DRReduce achieves average size reductions of 51.9%, 14.9%, and 19.8% over Perses, WDD, and CDD respectively, while completing reduction faster on the majority of programs. Compared to language-specific tools, DRReduce achieves results comparable to CReduce and Latra without any language-specific transformation rules, at 3.3x and 1.2x higher efficiency than CReduce and Latra on average, respectively. An ablation study confirms that dependency reconstruction reduces query invocations by 80.2%, reduction time by 58.7%, and final token count by over 55.1%.

</details>
