### [1] [Contract Based Verification of Non-functional Requirements for Embedded Automotive C Code](https://arxiv.org/abs/2605.21532)
*Jesper Amilon,Merlijn Sevenhuijsen,Mattias Nyberg,Karl Palmskog*

Main category: cs.PL

TL;DR: 本文提出了一种针对安全关键嵌入式系统中C代码非功能性需求（如控制流和数据流约束）的模块接口规范契约语言，并实现了Frama-C插件来检查这些契约，结合ACSL形式化验证，在斯堪尼亚卡车的两个案例研究中验证了其有效性。


<details>
  <summary>Details</summary>
Motivation: 现有的ACSL契约和Frama-C Wp插件能验证C代码的功能需求，但无法约束非功能需求（如禁止调用不安全函数、使用未初始化变量）。在嵌入式计算中，来自分包商、初级开发者或大语言模型生成的不可信代码可能通过功能验证却存在非功能安全问题。

Method: 1. 构建了一套针对C模块非功能需求（控制流和数据流）的通用规则集，正交于MISRA-C等规则；2. 提出了一种模块接口规范契约语言，用于描述模块的允许行为；3. 实现了Frama-C插件作为检查器，输入C模块及其契约，检查控制流和数据流属性（如只允许调用许可函数）；4. 集成到工具链中，支持模块契约与ACSL契约的联合规范与验证。

Result: 在斯堪尼亚卡车的两个安全关键C代码案例研究中，基于非正式系统需求定义了模块契约和ACSL函数契约，并使用所提工具链进行了验证，证明了方法的可行性。

Conclusion: 所提出的模块契约语言及其检查器能有效补充ACSL在非功能需求验证方面的不足，为安全关键嵌入式系统中不可信代码的验证提供了更全面的解决方案。

Abstract: Code contracts provide a robust way to specify functional requirements of safety-critical software in embedded systems. For example, the ANSI/ISO C Specification Language (ACSL) can be used to specify the functional behavior of C code that is then formally verified by the Frama-C framework's Wp plugin. However, non-functional requirements, such as restrictions on control flow and data flow, are also important for embedded systems safety. Untrusted code developed by subcontractors, junior developers, or generated by large language models, can be verified by Wp but may nevertheless call unsafe functions or use uninitialized program variables. To address this problem, we constructed a set of general rules concerning non-functional requirements of C code in safety-critical embedded systems. Our rules are orthogonal to popular C rulesets such as MISRA-C and center on modules and their interaction through interfaces. To enable checking our rules, we propose an interface specification contract language for C modules. We implemented a checker for our rules as a Frama-C plugin, which takes as input a C module and its contract and checks control flow and data flow properties, ensuring, e.g., that only permitted functions are called by the module. We integrated our checker in a toolchain to enable specification and verification of module contracts and ACSL contracts for untrusted code. We report on two case studies on safety-critical C code using software in Scania trucks, where we defined module contracts and ACSL function contracts based on informal system requirements and verified them using our toolchain.

### [2] [QuCtrl-BELL: A Compiler-Driven Sub-Microsecond Feedback Control Stack for Scalable Trapped-Ion Quantum Experiments](https://arxiv.org/abs/2605.22433)
*Junpeng She,Ruoyu Yan,Zhizhen Qin,Zhanyu Li,Zhongtao Shen,Zichao Zhou,Binxiang Qi,Luming Duan*

Main category: quant-ph

TL;DR: 本文提出QuCtrl-BELL (Bell)，一个基于编译器的编译驱动软件栈，通过解耦控制流与硬件状态数据，在保持亚微秒级板级反馈的同时实现可编程性和模块化，解决了离子阱量子计算中经典控制系统面临的硬件耦合与软件抽象之间的根本性权衡。


<details>
  <summary>Details</summary>
Motivation: 离子阱量子计算向更大规模扩展时，经典控制系统面临根本性权衡：亚微秒级板级反馈需要紧密的硬件耦合，而可维护性和可扩展性又需要清晰、模块化的软件抽象。

Method: 采用Python嵌入式DSL，通过六阶段转译管线（CFG构建、SSA转换、活跃性分析、图染色寄存器分配）产生确定性分布式板级程序和紧凑的步进表数据，并设计跨板同步协议支持低于700ns延迟的反馈循环。

Result: 在QuCtrl-BELL平台（RISC-V + PXIe）上部署和评估，验证了基于编译器的基础设施能够提供可编程性、确定性时序和模块化。

Conclusion: Bell证明了编译器驱动的方法可以有效解决离子阱量子控制中的硬件耦合与软件抽象矛盾，为可扩展的量子控制提供了可行的基础设施方案。

Abstract: As trapped-ion quantum computing scales to larger qubit registers and more complex control protocols, classical control systems face a fundamental tradeoff: sub-microsecond board-level feedback requires tight hardware coupling, whereas maintainability and extensibility require clean, modular software abstractions. This paper presents QuCtrl-BELL (Bell), a compiler-driven software stack for trapped-ion quantum control. The design resolves this tradeoff by decoupling control flow -- including loops, branches, and synchronization -- from hardware state data. A Python-embedded domain-specific language (DSL) is lowered through a six-stage transpilation pipeline covering control flow graph (CFG) construction, static single-assignment (SSA) conversion, liveness analysis, and graph-coloring register allocation. The compiler generates deterministic distributed board-level programs and compact step-table data. A cross-board synchronization protocol supports feedback loops with latency below 700~ns without host intervention. Bell is deployed and evaluated on the QuCtrl-BELL platform (RISC-V + PXIe), demonstrating that a compiler-based infrastructure can provide programmability, deterministic timing, and modularity for scalable trapped-ion quantum control.

</details>
