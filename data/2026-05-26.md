### [1] [Intent-Driven Computing: A Computational Model for Governed Autonomous Systems](https://arxiv.org/abs/2605.24036)
*Alan L. McCann*

Main category: cs.PL

TL;DR: 本文提出"意图驱动计算"范式：程序不再直接执行效应，而是生成描述提议行为的有限数据值（意图），由受治理的运行时根据可判定策略检查、记录且不可篡改地存证后再执行。该模型将所有效应交互具体化为有限意图数据，将治理从不可判定的程序语义域转移到可判定的意图数据域，从而获得事件溯源、治理模拟、审计完整性等涌现特性。作者在BEAM虚拟机上实现了具体语言，并通过Rocq形式化验证（454个定理，零未证明引理）和属性基础测试（70,000+随机输入，零分歧）验证了模型。


<details>
  <summary>Details</summary>
Motivation: 现有编程语言假定程序直接执行效应，但在自主系统动态生成行为时，从决定行动到执行行动之间缺乏结构化的中介点。这导致无法在"决策"和"执行"之间进行有效的治理审核。

Method: 提出意图驱动计算模型：程序产生"意图"（描述提议动作的有限数据值）而非直接执行效应；受治理的运行时对每个意图进行检查（可判定策略语言）、记录到防篡改账本、然后才执行效应。语言本身不提供绕过该路径的其他方式。

Result: 在BEAM虚拟机上实现具体语言，Rocq形式化验证证明454个定理（36个模块，零未证明引理），属性基础测试用70,000+随机输入验证实现与规范一致（零分歧）。

Conclusion: 意图驱动计算将治理从不可判定的程序语义域转移到可判定的意图数据域，获得事件溯源、治理模拟、审计完整性等关键涌现特性，为自主系统的安全可控行为提供了形式化的编程基础。

Abstract: Programming languages assume programs directly execute effects. When autonomous systems generate behavior dynamically, this assumption becomes problematic: there is no structural mediation point between deciding to act and acting. We define intent-driven computing: a programming model where programs produce intents (finite data values describing proposed actions) rather than directly executing effects. A governed runtime examines each intent against a decidable policy language, records every decision in a tamper-evident ledger, and only then realizes the effect. The language provides no alternative path to effects. The model does not decide arbitrary behavioral properties of programs (which Rice's theorem shows is impossible). Instead, it constrains the language so that all effectful interaction is reified as finite intent values, shifting governance from the undecidable domain of program semantics to the decidable domain of intent data. This yields emergent properties: event sourcing by construction, governance simulation via intent replay, structural audit completeness, and improved human comprehensibility. We specify the model formally, implement it in a concrete language compiling to the BEAM virtual machine, and verify key properties in Rocq (454 theorems, 36 modules, zero admitted lemmas). Property-based testing (70,000+ random inputs, zero disagreements) validates that the implementation matches the specification.

### [2] [Program Synthesis for Non-Linear Real Arithmetic: Going Beyond Realizability](https://arxiv.org/abs/2605.24263)
*S. Akshay,Supratik Chakraborty,R. Govind,Aniruddha R. Joshi*

Main category: cs.PL

TL;DR: 本文研究从非线性实数算术（NRA）规范中合成程序的问题。作者提出了一种方法，即使规范不可实现，也能合成出在有理数输入/输出上要么满足规范、要么报告无解的鲁棒程序。针对单输出变量情形给出完整算法，一般情形给出声音但不完全的算法，并实现了原型工具NQSynth。


<details>
  <summary>Details</summary>
Motivation: 现有SyGuS等技术在规范不可实现时会失败，而实际场景中需要鲁棒的程序——要么给出满足规范的输出，要么明确报告无解。此外，浮点数存在舍入误差，因此作者将程序限定在有理数输入/输出上。

Method: 1) 理论分析：证明该问题与数论中一个长期未解难题难度相当，且一般情形下无法完全合成无环程序。2) 针对单输出变量的情形，提出一个声音且完全的合成算法。3) 对于可实现规范，利用现有SyGuS生成的NRA程序（实数域）可直接作为有理数域的解。4) 对于一般情形，提出一个声音但不完全的合成算法。5) 实现原型工具NQSynth。

Result: NQSynth能够解决许多现有SyGuS工具无法处理的基准测试，即使在规范被调整为可实现的情况下也优于现有工具。

Conclusion: NRA程序合成在有理数域上具有独特的理论挑战和实际价值，作者提出的方法在不可实现规范下仍能生成鲁棒程序，填补了现有技术的空白。

Abstract: We study the problem of synthesizing programs from nonlinear real arithmetic (NRA) specifications. Existing techniques, such as syntax-guided synthesis (SyGuS), fail to synthesize programs when the specification is unrealizable. We argue this is unsatisfactory in many situations, and aim to synthesize programs from arbitrary NRA specifications, such that for any input, the synthesized program either produces outputs satisfying the specification or reports non-existence of any such output. To avoid rounding errors inherent in floating-point arithmetic, we restrict our programs to work on rational inputs and outputs.
  We first show that our variant of the synthesis problem is as hard as a long-standing open problem in number theory, and that synthesizing loop-free programs from arbitrary NRA specifications with rational inputs and outputs is impossible in general. Second, we present a sound and complete synthesis algorithm for the case where the specification involves a single output variable. We also show that for realizable specifications, a program generated by SyGuS for NRA (real inputs and outputs) serves as a solution to our problem, where inputs and outputs are rationals. Third, we provide a sound (but necessarily incomplete) synthesis algorithm for the general case of specifications. We have implemented our approach in a prototype tool called NQSynth that solves many benchmarks beyond the reach of state-of-the-art SyGuS tools, even when we render the specifications realizable.

### [3] [DateSAT: A Framework for Solving Date and Period Constraints](https://arxiv.org/abs/2605.25180)
*Leyi Cui,Shrey Tiwari,Rohan Padhye*

Main category: cs.LO

TL;DR: 本文提出 DateSAT，首个支持日期和日历周期可满足性约束求解的框架，通过五种 SMT 归约策略实现，并在 450 个约束数据集上进行了评估。


<details>
  <summary>Details</summary>
Motivation: 现有程序分析和验证工具缺乏对日期和日历计算的原生支持，而日期逻辑因不规则月份、歧义（如"1月31日之后一个月"）和闰年等边界情况极易出错，难以进行符号推理。

Method: 形式化定义了日期和周期算术的输入语言与语义，提出五种基于归约到整数 SMT 公式的求解策略，并使用 Z3 作为后端实现。

Result: 通过 LLM 提示、语法采样和法律文档挖掘构建了 450 个 DateSAT 约束数据集，并对求解器性能进行了实证评估。

Conclusion: DateSAT 是首个为日期约束提供原生支持的求解框架，有效填补了程序分析中日期推理的空白，多种归约策略为后续优化提供了基础。

Abstract: Dates and calendar periods (i.e., days, months, years) appear frequently in tasks involving analysis of software, data, and documents. Prior research has shown that computer logic involving dates and calendrical calculations is error-prone due to tricky rules (e.g., irregularly sized months), ambiguities (e.g., scheduling one month from "Jan 31st"), and edge cases (e.g., leap years). However, existing program analysis and verification tools do not provide native support for dates, making it hard to reason about operations involving calendrical arithmetic symbolically.
  This paper presents DateSAT, the first framework for expressing and solving satisfiability constraints involving dates and calendar periods. The paper first formalizes an input language and the semantics of date and period arithmetic. The paper then presents five separate strategies for solving DateSAT constraints based on reductions to SMT formulas involving integers, which we have implemented using Z3 as a backend. We curate a dataset of 450 DateSAT constraints synthesized using LLM prompting, grammar-based sampling, and mining legal documents, and then present an empirical evaluation of DateSAT solver performance.

### [4] [Context: Proactive Goal-Directed Intelligence via Composable Sandboxed Programs, Declarative Wiring, and Structured Interaction](https://arxiv.org/abs/2605.23928)
*Gregory Magarshak*

Main category: cs.AI

TL;DR: 提出Context架构，通过预计算上下文、组合式沙盒程序、主动式目标流状态机三种机制，将被动问答式聊天机器人转变为主动推进目标的智能体，并证明了六个形式化定理。


<details>
  <summary>Details</summary>
Motivation: 传统聊天机器人采用被动响应模式，需要用户主动提示才能推进对话，效率低下。作者希望构建能主动推进共享目标的智能层，减少用户交互负担和LM调用成本。

Method: 三个核心机制：1) 写入时上下文组装 - 通过Groker代理预计算丰富类型属性，生成确定性纯函数上下文，支持KV缓存近100%复用；2) 可组合沙盒智慧程序 - LM生成的命令式程序通过类型流关系声明式绑定到目标类型，无需额外LM调用即可执行；3) 主动目标流状态机 - 通过检查图状态并主动发出结构化交互内容，驱动对话至终止状态。

Result: 证明了六个形式化定理：上下文稳定性定理（每轮LM成本与语义变化率相关）、程序组合正确性定理、声明式绑定健全性定理、主动主导定理（主动代理在期望轮数上弱主导被动代理）、协调开销消除与质量保持定理（多方目标聊天中的帕累托改进）、跨平台投票一致性定理。

Conclusion: Context架构为构建主动式、高效、可证明的对话智能体提供了理论基础和实现框架，在Qbix/Safebox/Safebots开源栈中实现。

Abstract: We present Context, the intelligence layer of the Magarshak Architecture, which replaces reactive query-response chatbots with proactive goal-directed agents that advance shared tasks without waiting for user prompts. The architecture rests on three mutually reinforcing mechanisms. Write-time context assembly precomputes enriched typed attributes via Groker agents, assembling interaction context as a deterministic pure function of graph state; context blocks are byte-identical across turns between semantic changes, enabling near-100% KV-cache reuse. Composable sandboxed wisdom programs form a governed library of LM-generated imperative programs declaratively wired to goal types via typed stream relations, composed via phase ordering, and executed at interaction time without further LM calls. Proactive goal stream state machines drive conversations toward terminal states by inspecting graph state and emitting structured interaction content (option arrays, governance affordances, clarification prompts) without awaiting user input. We prove six formal results: the Context Stability Theorem, bounding per-turn LM cost as a function of semantic change rate; a Program Composition Correctness Theorem; a Declarative Wiring Soundness Theorem; the Proactive Dominance Theorem, proving proactive agents weakly dominate reactive agents on expected turns-to-terminal-state; Coordination Overhead Elimination and Quality Preservation, establishing Pareto improvements in multi-participant goal chats; and a Cross-Platform Vote Consistency Theorem. Implemented in the open-source Qbix / Safebox / Safebots stack.

### [5] [BODHI: Precise OS Kernel Specification Inference](https://arxiv.org/abs/2605.23931)
*Zhiming Chang,Ziyang Li*

Main category: cs.AI

TL;DR: 本文提出BODHI方法，通过结构化C-to-Python翻译指南增强大语言模型在操作系统内核形式化规约生成任务上的表现，最佳配置(Claude Opus 4.6 + BODHI)达到96.73% Pass@1，相比基线提升11%-32%。


<details>
  <summary>Details</summary>
Motivation: 操作系统内核系统调用的形式化规约编写需要深厚的领域专业知识，人工编写成本高昂，因此希望利用LLM自动化该过程。然而现有方法在OSV-Bench基准上Pass@1仅为55.10%，性能不足。

Method: 提出BODHI方法，受结构化思维链(SCoT)启发，设计了一个覆盖15类领域特定翻译模式的结构化C-to-Python翻译指南，将前置条件提取和后置条件生成作为不同类别进行关注点分离，以增强标准few-shot提示。

Result: 在来自6个提供商的9种模型上评估（覆盖密集、混合专家和推理架构），BODHI使所有模型性能提升11%-32%；最佳配置(Claude Opus 4.6 + BODHI)达到96.73% Pass@1；减少了语法错误和语义错误；对指令遵循能力充足的模型效果最强。

Conclusion: 领域知识注入是一种模型无关的技术，能显著弥合通用代码生成与形式化规约合成之间的差距。

Abstract: The formal verification of operating system kernels requires precise specifications that capture the intended behavior of system calls. Writing these specifications manually demands deep domain expertise, motivating the use of large language models (LLMs) to automate the process. However, in OSV-Bench, a benchmark of 245 specification generation tasks derived from the Hyperkernel OS kernel, the best reported Pass@1 is 55.10%. We propose a domain knowledge prompting method (BODHI), which augments the standard few-shot prompt with a structured C-to-Python translation guide covering 15 categories of domain-specific translation patterns. Inspired by Structured Chain-of-Thought (SCoT) prompting, the guide organizes translation by separation of concerns, addressing pre-condition extraction and post-condition generation as distinct categories. Evaluated on nine models from six providers (Anthropic, Mistral, Amazon, DeepSeek, Meta, Alibaba), covering dense, mixture-of-experts and reasoning architectures, BODHI improves every model tested, with gains ranging from +11% to +32%. The best configuration (Claude Opus 4.6 + BODHI) reaches 96.73% Pass@1. BODHI reduces both syntax and semantic errors, with the strongest effect on models that have sufficient instruction-following capability to utilize structured reference material. These results demonstrate that domain knowledge injection is a model-agnostic technique that substantially bridges the gap between general-purpose code generation and formal specification synthesis.

</details>
