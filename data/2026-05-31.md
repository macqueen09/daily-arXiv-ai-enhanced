### [1] [Generalized Software Product Line Extraction](https://arxiv.org/abs/2605.28989)
*Federico Bruzzone,Walter Cazzola,Luca Favalli*

Main category: cs.SE

TL;DR: 本文提出了一种与工作台无关的软件产品线（SPL）工程协议，基于轻量级依赖单元"atoms"，通过客户端-服务器架构实现特征模型的提取、配置和产品衍生，支持异构开发环境，并在语言产品线（LPL）场景中进行了验证。


<details>
  <summary>Details</summary>
Motivation: 现有SPL工程方法通常与特定技术空间（编程语言、组合器）和IDE紧密耦合，依赖的提取机制对底层技术空间有强假设，这种紧耦合阻碍了异构开发环境下的复用、演化和采用。

Method: 提出一种通用的、与工作台无关的SPL协议：采用自底向上的方法，以轻量级依赖单元"atoms"为基础；通过SPL服务器（独立于工作台）和SPL客户端（工作台特定后端+通用前端）组织提取和配置过程；对底层软件工件假设较少，适用于多种SPL场景。

Result: 通过原型实现验证了架构可行性，多个子系统可自由交互和替换而不影响其他部分。在语言产品线（LPL）场景中具体实现了：Neverlang语言工件、Java SPL客户端后端、Go+Prolog编写的可重用SPL服务器、JavaScript SPL客户端前端。

Conclusion: 该协议在保持工作台无关性的同时，成功应用于具体场景（如LPL），证明了其通用性和实用性。为异构开发环境下的SPL工程提供了一种松耦合、可演化的解决方案。

Abstract: Software product line (SPL) engineering has been successfully applied to software development by obtaining software systems as compositions of modular features. Existing approaches to SPL engineering, however, are typically bound to a specific technological space (such as, a programming language and a composer) and integrated development environment (IDE), and rely on extraction mechanisms that make strong assumptions on the underlying technological space. This tight coupling hinders reuse, evolution, and adoption of heterogeneous development environments. We propose a general, workbench-agnostic protocol for extracting feature models from existing software artifacts and for configuring and deriving software products. The protocol follows a bottom-up approach based on lightweight dependency units called "atoms", and organizes the extraction and configuration process around an SPL server (workbench-independent) and an SPL client with a workbench-specific backend and a generic frontend. The protocol makes few assumptions on the underlying software artifacts and is therefore applicable to varied SPLs. The applicability of this approach is presented through a prototypical implementation of the architecture in which several subsystems interact and can be swapped freely without affecting the others. In particular, we focus on the application of such a protocol in the context of language product lines (LPLs), demonstrating its applicability to concrete scenarios while preserving workbench-agnosticism. From bottom to top, the implementation comprises: Neverlang language artifacts, a Java SPL client backend, an agnostic and reusable SPL server written in Go and Prolog, and a JavaScript SPL client frontend.

### [2] [A Bayesian Approach to Membership Inference for Statistical Release](https://arxiv.org/abs/2605.30203)
*Lisa Oakley,Sam Stites,Cameron Moy,Steven Holtzen,Alina Oprea,Marco Gaboardi*

Main category: cs.CR

TL;DR: 本文研究成员推断攻击问题，提出利用贝叶斯网络建模人口属性依赖结构，并基于概率编程实现贝叶斯后验计算，实验证明该方法在复杂依赖结构上优于传统似然比检验和内积攻击。


<details>
  <summary>Details</summary>
Motivation: 现有成员推断攻击大多假设人口服从乘积分布，仅利用边缘分布信息，未能有效利用属性依赖结构。但在实际场景（如美国人口普查与IRS数据）中，多个数据源可能共享相似的结构信息，利用这些依赖关系可以更好地评估真实的隐私威胁。

Method: 将人口建模为贝叶斯网络（BN），基于贝叶斯决策理论构建攻击框架，将先验信息纳入攻击决策；通过概率编程语言Roulette实现贝叶斯后验计算，证明该攻击在强属性依赖的两群体验证下等价于最优似然比检验变体。

Result: 在五个常用贝叶斯网络上进行实验，结果表明本文提出的攻击方法在人口属性依赖结构复杂时，优于似然比检验（Likelihood Ratio Test）和内积攻击（Inner Product Attack）。

Conclusion: 利用贝叶斯网络建模属性依赖结构可以显著提升成员推断攻击的效果，为更真实的隐私威胁评估提供了新思路。

Abstract: The membership inference problem for publicly released statistics from a private dataset is well-studied. When developing and formally analyzing attack strategies, however, the focus has been on attacks that model the population using only its marginals. In practice, these attacks can perform well on various populations, however most formal analysis is for populations that follow a product distribution. These strategies may fail to leverage useful information about the population that is important for understanding a realistic privacy threat.
  In this work, we explore the impact of providing an attacker with additional information about the attribute dependency structure of the population, motivated by examples where multiple parties may have access to similarly structured data, for example the US Census and the IRS. To model this scenario, we re-frame the membership inference problem with respect to a population represented as a Bayesian network (BN). We develop a framework based on Bayesian decision-making which can incorporate prior information about the population to launch more effective, specialized attacks.
  To evaluate our framework, we introduce a specific attack instantiation which computes the Bayesian posterior using a probabilistic program, and prove its equivalence to an optimal variant of the likelihood ratio test attack for two populations with strong attribute dependency. We implement our program in the Roulette probabilistic programming language and show experimentally that it outperforms the likelihood ratio test and inner product attacks on five commonly used BNs, where the population dependency structure is too complex for the existing attacks to be manually adapted.

### [3] [PassNet: Scaling Large Language Models for Graph Compiler Pass Generation](https://arxiv.org/abs/2605.29357)
*Yiqun Liu,Yingsheng Wu,Ruqi Yang,Enrong Zheng,Honglei Qiu,Sijun He,Tai Liang,Jingjing Wu,Yuhan Zhou,Yiwei Zhang,Dongyan Chen,Weihan Yi,Xinqi Li,Siqi Bao*

Main category: cs.AI

TL;DR: 本文提出PassNet，首个面向LLM驱动的编译器优化的大型生态系统，包含18000+计算图数据集和200个长尾融合任务的基准PassBench，实验表明LLM在个别子图上可达3倍加速，但一致性是瓶颈，小模型微调后性能提升2.67倍。


<details>
  <summary>Details</summary>
Motivation: 现代张量编译器（如TorchInductor）在主流的模型上表现良好，但在长尾计算图上存在系统性性能瓶颈——43%的真实子图在默认编译下出现端到端减速。现有LLM优化工作集中于独立内核生成，缺乏更合适的编译器通量化抽象。

Method: 提出PassNet生态系统：(1) PassNet-Dataset：从10万个真实模型中提取超过1.8万个独特计算图；(2) PassBench：200个精选的长尾可融合任务（共2060个子图），采用纠错感知加速分数(ES_t)作为统一指标，并引入分层完整性防御机制防止LLM系统性利用。

Result: PassBench具有高区分度和非饱和性：最佳前沿模型在聚合指标上落后TorchInductor 37%，但在个别子图上LLM可实现高达3倍加速。在约4000个PassNet轨迹上微调小模型获得2.67倍改进，接近前沿模型性能。

Conclusion: 瓶颈在于一致性而非能力，PassNet可作为LLM驱动编译器优化的实时训练基础设施，所有数据、基准和工具均已公开。

Abstract: Modern tensor compilers such as TorchInductor deliver substantial speedups on mainstream models, yet face a systematic performance ceiling on long-tail workloads -- our profiling shows that 43% of real-world subgraphs experience end-to-end slowdowns under default compilation. While LLMs offer a path toward automated optimization, existing efforts focus on standalone kernel generation. We argue that pass generation -- where LLMs author structured graph transformations that integrate directly into compiler pipelines -- is the more appropriate abstraction. We propose PassNet, the first large-scale ecosystem for LLM-based compiler pass generation, comprising: (1) PassNet-Dataset, over 18K unique computational graphs from 100K real-world models; and (2) PassBench, 200 curated long-tail fusible tasks (comprising 2,060 subgraphs in total) evaluated under the Error-aware Speedup Score (ES_t) -- a metric unifying correctness, stability, and performance -- with layered integrity defenses against systematic LLM exploitation. Experiments reveal that PassBench is both highly discriminative and genuinely unsaturated: the best frontier model trails TorchInductor by 37% in aggregate, yet on individual subgraphs LLMs achieve up to 3x speedup over the same compiler -- indicating that the bottleneck is consistency, not capability. Fine-tuning a small model on merely ~4K PassNet trajectories yields a 2.67x improvement approaching frontier-model performance, demonstrating substantial headroom and validating PassNet as live training infrastructure for advancing LLM-driven compiler optimization. All data, benchmarks, and tooling are publicly available.

</details>
