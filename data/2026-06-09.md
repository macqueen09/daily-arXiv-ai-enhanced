### [1] [AgentCompile: An LLM-Guided Compiler for Direct CUDA Inference](https://arxiv.org/abs/2606.07665)
*Xuanzhe Li,Ziyan Weng,Zhiyu Zhu,Junhui Hou*

Main category: cs.PL

TL;DR: 提出AgentCompile，一种利用LLM提供搜索元数据（语义标签、候选优先级、参数提示、风险注释）来指导CUDA推理编译的系统，在多个模型上实现4-5.6倍加速。


<details>
  <summary>Details</summary>
Motivation: Transformer推理越来越依赖专门的编译器和运行时支持，但真实模型图仍需要语义决策来判断哪些区域值得特化、哪些CUDA实现族可行，现有方法缺乏高效自动化的方案。

Method: LLM仅提供咨询性搜索元数据；编译器通过模板生成CUDA候选，检查接口和硬件约束，通过实验验证候选，选择延迟最优的实现，在特化不支持或无收益时回退。

Result: 在端到端自回归生成中，AgentCompile在Qwen3-1.7B、Qwen3-4B和Llama-3.2-1B-Instruct上分别实现5.66x、4.05x和4.26x的加速（对比PyTorch eager）。

Conclusion: LLM引导的编译方法能在不依赖LLM直接生成代码的情况下，显著加速Transformer推理，项目将开源。

Abstract: Transformer inference increasingly depends on specialized compiler and runtime support, but real model graphs still require semantic decisions about which regions are worth specializing and which CUDA implementation families are plausible. We present AgentCompile, an LLM-guided CUDA inference compiler that uses LLM outputs only as advisory search metadata. Given compiler-derived region summaries and bounded candidate spaces, the LLM proposes semantic labels, candidate priorities, parameter hints, and risk annotations; the compiler materializes CUDA candidates through templates, checks interface and hardware constraints, validates candidates empirically, selects implementations by measured latency, and falls back when specialization is unsupported or unprofitable. In end-to-end autoregressive generation, AgentCompile averages 5.66x, 4.05x, and 4.26x speedup over PyTorch eager on Qwen3-1.7B, Qwen3-4B, and Llama-3.2-1B-Instruct, respectively, across five representative workloads. We will open-source the project.

### [2] [SNN-MLIR: An MLIR Dialect for Compiling Neuromorphic SNNs from NIR to Bare-Metal C](https://arxiv.org/abs/2606.09213)
*Alejandro García Gener,Alvaro Rollón de Pinedo*

Main category: cs.PL

TL;DR: 本文提出 snn-mlir，一个基于 MLIR 的 SNN 方言，通过 NIR-MLIR-C 编译桥接实现了脉冲神经网络的统一中间表示与跨平台部署，支持浮点和量化数据类型，可自动生成无依赖的 C11 代码。


<details>
  <summary>Details</summary>
Motivation: 现有 SNN 框架各自为政，缺乏统一的模型交换与部署标准。NIR 解决了模型交换问题，但未提供从描述到实际运行的编译路径，各后端仍需各自实现部署。

Method: 设计了 out-of-tree MLIR 方言，提供类型多态操作（统一处理浮点与量化数据）；Python 前端解析 NIR 文件并生成方言 IR，自动插入重缩放操作以保持量化尺度一致性；参考 lowering 过程将方言转换为标准 linalg/arith 操作，最终生成无依赖 C11 代码。

Result: 针对前馈全连接网络和 CPU 后端进行了数值精度验证、跨平台可移植性测试以及量化代价评估。r/> 

Conclusion: snn-mlir 提供了一条从 NIR 模型到可运行 C 代码的完整编译路径，统一了 SNN 的仿真与硬件部署表示，已开源在 GitHub 上（Apache-2.0 许可）。

Abstract: Spiking neural networks (SNNs) are increasingly trained in a wide range of frameworks (SnnTorch, Lava, Norse, and others) each with its own model format. The Neuromorphic Intermediate Representation (NIR) addresses this fragmentation by providing a common, framework-independent format for exchanging trained SNN models. NIR solves the exchange problem, but it stops there. It provides a description of a network, not a path to running one. Each backend is still left to implement deployment on its own, with no shared, transformable compiler representation in between. This paper presents snn-mlir, an outof-tree MLIR dialect for SNNs together with a NIR-MLIR-C compilation bridge. The dialect provides a small set of typepolymorphic operations that work identically on floating-point (f32/f64) and quantized data, so a single intermediate representation serves both simulation and hardware-oriented deployment. A Python front end reads any NIR file and emits dialect IR, automatically inserting rescaling operations to keep quantization scales consistent across layers. A reference lowering pass converts the dialect to standard linalg and arith operations, from which the toolchain produces self-contained, dependency free C11 code that compiles and runs on any C-capable CPU or embedded target. We evaluate numerical fidelity against reference outputs, portability across CPU targets, and the cost of quantization. The current scope is feedforward, fully-connected networks with a CPU backend. snn-mlir is released as open source under the Apache-2.0 license with LLVM-exception and it is already available on Github.

### [3] [An Empirical Comparison of General Context-Free Parsers](https://arxiv.org/abs/2606.08465)
*Huan Vo,Danushka Liyanage,Hong Jin Kang,Sasha Rubin,Rahul Gopinath*

Main category: cs.FL

TL;DR: 本文对六种通用上下文无关解析算法（CYK、Valiant、Earley、GLL、RNGLR、BRNGLR）及确定性LL(1)/LR(1)基线在Rust中统一实现并基准测试，结果表明通用解析器的性能代价远低于普遍预期，其中GLR族是通用解析中的性能优胜者。


<details>
  <summary>Details</summary>
Motivation: 现有软件工程实践中普遍使用确定性解析器（LL/LR），迫使开发者为适配解析器而扭曲文法并牺牲语言表达能力。尽管通用上下文无关解析器能消除这一约束，但缺乏对各类算法的严格横向对比，使得从业者难以做出知情选择。

Method: 在Rust中统一实现六种通用解析算法（CYK、Valiant、Earley、GLL、RNGLR、BRNGLR）及LL(1)/LR(1)基线，共享数据结构和解析树提取逻辑，在22个文法（从简单表达式到完整C++/Java）上进行受控基准测试。

Result: 通用解析的性能代价低于广泛假设：在确定性文法上，GLR族相对于LR(1)的中位数延迟仅3倍，方差窄且可预测。GLR是通用解析器中的性能优胜者，适合作为软件工程工具的默认选择。

Conclusion: 通用解析器的性能代价远低于普遍认知，GLR是实际工程中的理想默认选择，为软件工程社区采用更表达性的语言设计提供了实证支持。

Abstract: Parsing underpins a vast range of software engineering tasks, from compilers and static analyzers to language servers and fuzz testing tools. Yet most parsers deployed in practice are deterministic (LL or LR), forcing developers not only to contort their grammars to fit the parser, but to simplify the very languages they design sacrificing expressiveness for the sake of parseability. General context-free parsers eliminate this constraint. Yet, despite decades of algorithmic development, no rigorous head-to-head comparison exists across the major families of parsing algorithms.
  We present the first unified, controlled benchmark of six generalized parsing algorithms: CYK, Valiant, Earley, GLL, RNGLR, and BRNGLR, plus deterministic LL(1) and LR(1) baselines, all implemented in Rust with shared data structures and parse-tree extraction, and evaluated across 22 grammars ranging from simple expressions to full C++ and Java. Our results show that the cost of generality is lower than widely assumed. On deterministic grammars, the GLR family incurs only a 3x median slowdown over LR(1), with a narrow and predictable variance. GLR is the clear performance winner among generalized parsers and a practical default choice for software engineering tools.

### [4] [When Types Intersect and Effects Get Handled](https://arxiv.org/abs/2606.09526)

<details>
  <summary>Details</summary>
Motivation: 为含代数效应和处理器的λ演算提供首个具备经典交集类型性质（主题归约与扩展）的类型系统，以刻画终止性并解决可达性问题。

Method: 引入一种新的交集类型系统，该系统具有行为性特征，满足主题归约和扩展性质；通过类型推断解决可达性问题；并诱导出一个简单类型系统。

Result: 新系统能够刻画终止计算项集合，将可达性问题归约为类型推断；其诱导的简单类型系统具有类型安全性，且HOMC问题可判定（优于Dal Lago和Ghyselen的HEPCF系统）。

Abstract: We introduce a novel intersection type system for a $λ$-calculus with algebraic effects and handlers. The system, inherently behavioral in nature, enjoys the classical properties of intersection type systems, in particular subject reduction and expansion. This new system, the first with these features for a calculus with handlers, induces a system of simple types which, although not guaranteeing termination, is type sound and admits a decidable HOMC problem, unlike similar type systems like Dal Lago and Ghyselen's HEPCF.

### [5] [Toward Compiler World Models: Learning Latent Dynamics for Efficient Tensor Program Search](https://arxiv.org/abs/2606.09312)
*Haolin Pan,Lianghong Huang,Xvlin Zhou,Mingjie Xing,Yanjun Wu*

Main category: cs.LG

TL;DR: 提出了一种受世界模型启发的评估器，将调度评估建模为程序状态上的动作条件潜在动力学，在TVM框架中显著提升了张量程序优化的效率与性能。


<details>
  <summary>Details</summary>
Motivation: 现有自动调度器使用学习到的代价模型评估候选程序，但仅将其视为静态代码快照，忽略了产生该候选的调度轨迹，导致对动作依赖关系不敏感，且易受表面代码变化影响。

Method: 将调度评估建模为动作条件潜在动力学：从初始程序开始，在连续潜在空间中利用轻量级转移模型展开调度动作，避免昂贵的AST变异和重复编码；最终将动态表示与动作及硬件特征结合进行排序。

Result: 在TVM AutoScheduler上实现，在64次试验预算下，GPU代表性子图延迟比Ansor提升1.37倍，CPU提升1.54倍；使用10倍更少测量达到Ansor-10K的2.2%几何均值内；全模型推理比PyTorch/PyTorch-opt(cuDNN)提升4.61倍/3.67倍几何均值。

Conclusion: 通过引入世界模型启发的潜在动力学评估器，有效利用了调度轨迹信息，大幅提升了张量程序优化的搜索效率和最终性能。

Abstract: Tensor program optimization is essential for modern machine learning systems, but its search space is enormous. Existing auto-schedulers reduce measurement cost with learned cost models, yet they usually evaluate each candidate as a static code snapshot, ignoring the schedule trajectory that produced it. This makes them insensitive to action dependencies and vulnerable to superficial code variations. We propose a \emph{world-model-inspired} evaluator that models schedule evaluation as action-conditioned latent dynamics over program states. Starting from the initial program, it rolls out scheduling actions in a continuous latent space with a lightweight transition model, avoiding expensive AST mutation and repeated code encoding. The final dynamic representation is combined with action and hardware features to rank candidates. Implemented in TVM AutoScheduler, our method improves representative-subgraph latency over Ansor by 1.37$\times$ on GPU and 1.54$\times$ on CPU under the same 64-trial budget. It also matches Ansor-10K within 2.2% geometric mean using 10$\times$ fewer measurements, and accelerates full-model inference over PyTorch/PyTorch-opt(cuDNN) by 4.61$\times$/3.67$\times$ geometric mean.

### [6] [LongRTL: Graph-Similarity-Guided LLM-driven Long Context RTL Optimization](https://arxiv.org/abs/2606.08944)
*Yuyang Ye,Che-Kuan Shen,Xiangfei Hu,Yuchen Liu,Shuo Yin,Xufeng Yao,Bei Yu,Tsung-Yi Ho*

Main category: cs.AR

TL;DR: 提出了一种基于图相似性的可扩展LLM框架，通过三个智能体（分区、优化、重构）协同工作，解决长上下文、低模块化的工业级RTL设计优化问题。


<details>
  <summary>Details</summary>
Motivation: 现有LLM在RTL代码生成和优化中表现良好，但真实世界的RTL设计通常冗长、纠缠且模块化差，受限于上下文长度和缺乏结构，难以处理工业级代码库。

Method: 提出三种协作智能体：1) 分区智能体：基于AST图相似性将RTL设计分解为语义有意义的AST子树；2) 优化智能体：利用多模态RAG（AST和RTL引导）生成子模块代码；3) 重构智能体：基于逻辑感知顺序和Graph-RAG提示重新组装优化后的子模块，确保全局功能等价。

Result: 实现了对长上下文的RTL设计进行鲁棒、结构感知的优化，弥合了玩具示例与工业级硬件代码库之间的差距。

Abstract: Large Language Models (LLMs) show great promise in RTL code generation and optimization. However, real-world RTL designs are typically long, entangled, and poorly modularized, posing a major challenge due to context-length limitations and lack of structure. To overcome these obstacles, we propose a scalable LLM-based RTL optimization framework guided by graph similarity. Our method introduces three collaborative agents: (1) a Partition Agent that decomposes RTL designs into semantically meaningful AST subtrees, guided by AST graph similarity to reusable design templates; (2) an Optimization Agent that generates RTL submodule code based on partitioned subtrees using multi-modal Retrieval-Augmented Generation (RAG) with both AST and RTL guidance; and (3) a Reconstruction Agent that reassembles optimized submodules based on logic-aware ordering and Graph-RAG prompting, ensuring global functional equivalence. Together, these components enable robust, structure-aware optimization of long-context RTL designs, bridging the gap between toy examples and industrial-scale hardware codebases.
