### [1] [Type Checking Project Haystack Grids using JSON Schema and Pydantic](https://arxiv.org/abs/2606.24891)
*Thomas Hirsch,Samina Kadkhoda Masoumali,Gerald Schweiger*

Main category: cs.PL

TL;DR: 本文提出了一个基于Python的工具链，用于解析Project Haystack本体定义文件（Trio格式），并自动生成Pydantic模型和JSON Schema，从而在Python生态内外实现静态类型检查和结构化验证，旨在打破Haystack的技术壁垒并增强其生态系统。


<details>
  <summary>Details</summary>
Motivation: Project Haystack是一种广泛采用的建筑本体，但其基于标签的语义模型存在标签使用歧义、缺乏自动化验证的问题，且依赖Haxall生态的自定义文件格式和领域特定语言，造成了实际的集成障碍。

Method: 开发了一套Python工具链，包括：（1）Haystack定义文件（Trio格式）的解析器；（2）从解析结果生成Pydantic模型和JSON Schema定义的代码生成器。

Result: 生成的Pydantic模型支持Python内的静态类型检查和结构验证，JSON Schema支持Python生态外的JSON表示验证。所有工具、模型和Schema均已开源发布。

Conclusion: 该工具链强化了Haystack生态系统，并开辟了一条超越其当前技术边界的实用路径，降低了集成门槛。

Abstract: Ontologies enable scalable energy services in buildings by supporting interoperability and automation. Project Haystack is a building ontology that is widely adopted due to its flexible, tag-based semantic model, openness, and extensibility, but suffers from ambiguous tag usage and limited automated validation. Although Project Haystack is formally open, its reliance on custom file formats and domain-specific languages that originate from the Haxall ecosystem creates a de facto barrier to integration. In this paper, we address these limitations by introducing a Python-based toolchain for Haystack. We present (i) a parser for Haystack definition files (Trio file format), and (ii) a code generator that derives Pydantic models and JSON Schema definitions from these parsed specifications. The resulting models enable static type checking and enable structural validation of Haystack grids within Python, as well as schema-based validation of JSON representations outside the Python ecosystem. All tools, generated models, and schemas are released publicly under an open-source license, with the goal of strengthening the Haystack ecosystem and opening a practical pathway beyond its current technical boundaries.

### [2] [Reading AI Model Compilation in MLIR Through the Lens of Formal Theories](https://arxiv.org/abs/2606.25244)
*Javed Absar*

Main category: cs.PL

TL;DR: 本文论证编译器基础设施（如MLIR）的设计原则与形式理论（项重写、精化演算、抽象解释等）之间存在对应关系，并指出理解这些形式概念有助于设计更好的抽象，尤其是在AI编码代理降低实现成本后，抽象设计成为关键问题。


<details>
  <summary>Details</summary>
Motivation: 随着编码代理降低实现成本，好的设计和抽象成为主要关注点，因此需要理解形式理论如何指导编译器基础设施中的抽象设计。

Method: 通过对比MLIR的设计原则（如match-and-rewrite、staged lowering、range analysis）与形式理论（项重写系统、精化演算、抽象解释）之间的对应关系，论证形式理论对编译器抽象设计的指导价值。

Result: 揭示了MLIR中多个核心机制与形式理论的结构性对应，指出形式理论能为抽象设计提供精确的词汇和理解完整性的框架。

Conclusion: 形式理论的认知有助于明确抽象应达到的理想设计，以及实际设计在何处做出了权衡偏离，这对编译器基础设施的设计具有重要意义。

Abstract: Compiler infrastructures such as MLIR rest on a set of design principles: IR abstractions, interfaces, match-and-rewrite, flow analysis, type conversion, staged lowering, and so on. These concepts have proven themselves in practice. Good designs typically arrive through engineering knowledge, intuition and experience. Many of them, however, have correspondences in formal theory. MLIR's match-and-rewrite engine has correspondence to a \emph{term-rewriting-system}~\cite{baadernipkow1998}; staged lowering has the structure of \emph{refinement calculus}~\cite{back1998}; and range analysis is grounded in \emph{abstract interpretation}~\cite{cousot1977,cousot1979}. Highlighting these correspondences is useful because each theory supplies vocabulary precise enough to discuss structural questions. Moreover, as coding agents lower the cost of implementation, good design and abstractions become the main concern~\cite{Lattner2026ClaudeCCompiler}. A coding agent can generate a pass, but it can only reason over the semantics the representation exposes. When essential structure is missing, the limitation is one of abstraction, not of implementation. The natural next question is how to design that substrate well. Well-chosen abstractions emerge from experience and intuition, but they often mirror concepts given a more precise treatment in formal theory. We argue that knowledge of these formal concepts clarifies what completeness means for a given abstraction, what the ideal design would be, and where practical trade-offs depart from it.

### [3] [Information flow security on persistent memory](https://arxiv.org/abs/2606.25422)
*Graeme Smith*

Main category: cs.LO

TL;DR: 本文提出了一种面向持久内存架构的信息流安全逻辑，通过重排序干扰自由（RIF）概念来推理无序指令传播带来的安全问题，并应用于x86汇编语言。


<details>
  <summary>Details</summary>
Motivation: 现有研究主要关注持久内存上程序的**功能正确性**（如故障恢复），但**信息流安全**（即防止机密信息泄露）尚未得到充分研究。随着持久内存的普及，需要形式化方法来保证其上程序的安全性。

Method: 1) 为包含goto而非循环的**非结构化语言**（模拟简单汇编语言）设计了信息流逻辑；2) 提出**重排序干扰自由（RIF）** 概念，用于推理指令乱序传播到内存时对信息流的影响；3) 将上述方法应用于x86汇编和持久内存场景。

Result: 成功将信息流逻辑应用于x86汇编语言，并展示了RIF概念既能处理传统架构也能处理持久内存架构上的信息流安全问题。

Conclusion: 重排序干扰自由（RIF）是一个统一且有效的概念，可用于形式化推理不同内存架构（包括持久内存）下的信息流安全性，填补了该领域安全验证的空白。

Abstract: Persistent memory is a recently proposed memory paradigm that delivers many system-wide benefits, including improved runtime efficiency and the ability of programs to recover from power outages and system crashes. While recent research has investigated techniques for proving functional correctness of programs running on related architectures, this is not the case for the orthogonal concept of information flow security. In this paper, we provide an information flow logic for an unstructured language (i.e., with gotos rather than loops) modelling a simple assembly language. We apply this logic to x86 assembly using a notion of reordering interference freedom (rif) to reason about potential out-of-order propagation of instructions to memory. We then show how this same notion of rif can be used to similarly reason about information flow on persistent memory.

</details>
