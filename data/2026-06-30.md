### [1] [Langshaw: Declarative Interaction Protocols Based on Sayso and Conflict](https://arxiv.org/abs/2606.29601)
*Munindar P. Singh,Samuel H. Christie,Amit K. Chopra*

Main category: cs.PL

TL;DR: Langshaw是一种基于sayso、nono和nogo三种新构件的声明式多智能体协议语言，在保持灵活性的同时通过信息模型表达协议含义，并提供了形式化语义、安全活性验证及异步执行的消息生成方法。


<details>
  <summary>Details</summary>
Motivation: 现有协议语言要么过度约束协议执行，要么难以准确表达协议含义，需要在灵活性与语义表达之间取得平衡。

Method: 提出三种核心语言构件：sayso（定义属性设置的优先级）、nono和nogo（表达动作间的冲突）；给出形式化语义；设计安全性和活性判定程序；开发面向消息的协议生成方法以支持灵活异步执行。

Result: 实现了兼具灵活性和信息模型表达能力的协议语言，并证明了其形式化语义的正确性。

Conclusion: Langshaw通过三种新构件的组合，有效解决了现有协议语言在灵活性与语义表达之间的权衡问题。

Abstract: Current languages for specifying multiagent protocols either over-constrain protocol enactments or complicate capturing their meanings. We propose Langshaw, a declarative protocol language based on (1) sayso, a new construct that captures who has priority over setting each attribute, and (2) nono and nogo, two constructs to capture conflicts between actions. Langshaw combines flexibility with an information model to express meaning. We give a formal semantics for Langshaw, procedures for determining the safety and liveness of a protocol, and a method to generate a message-oriented protocol (embedding needed coordination) suitable for flexible asynchronous enactment.

</details>


### [2] [Reactive Graphs for Efficient Markov Chain Monte Carlo Inference in Probabilistic Programming Languages](https://arxiv.org/abs/2606.30137)
*Viktor Palmkvist,Fredrik Ronquist,David Broman*

Main category: cs.PL

TL;DR: 本文提出了一种自动将概率程序转换为动态图的方法，利用MCMC中样本的局部变化特性，只重新计算依赖已改变随机变量的部分，从而提升推理效率。该方法基于函数式编程接口，应用函子部分表达贝叶斯网络，单子部分实现通用概率编程。


<details>
  <summary>Details</summary>
Motivation: 概率程序推理的关键挑战是效率。MCMC采样中，大多数提议样本是前一样本的微小变化，但传统实现会重复计算大量不变部分，造成资源浪费。因此需要一种自动方法，只重新计算被修改随机变量所影响的部分。

Method: 提出一种自动将概率程序翻译为动态图的方法，类似函数式响应式编程（FRP）。该图显式表示数据依赖关系，MCMC提议仅重新计算图中依赖被重新抽取的随机变量的部分。接口遵循函数式编程风格，应用函子部分对应贝叶斯网络，单子部分对应通用概率编程语言。

Result: 通过动态图表示数据依赖，MCMC采样时只需增量式地重新计算受影响部分，避免了大量冗余计算，从而显著提升了概率程序推理的效率。

Conclusion: 利用MCMC样本的局部变化特性，结合函数式编程的接口设计，可以自动实现高效的概率程序推理，且该方法在表达能力上覆盖了贝叶斯网络和通用概率编程两种范式。

Abstract: An important aspect of making inference based on a probabilistic program practical is efficiency; faster evaluation enables more work per unit of time, which can be translated into more precision. Inference via Markov chain Monte Carlo has a property that can be favorably exploited for efficiency: most proposed samples are computed as minor variations of previous samples, i.e., a clever implementation can skip computations pertaining to what is unchanged. This paper provides an approach for automatically translating a probabilistic program to a dynamic graph, reminiscent of functional reactive programming, that explicitly represents data dependencies, enabling proposals to only recompute the parts of the graph that depend on redrawn random variables. The graph-building interface follows familiar functional programming interfaces, which also connect to their expressiveness in terms of probabilistic programming: models using the applicative functor portion express Bayesian networks, while those using monads represent universal probabilistic programming languages.

</details>

### [3] [KoAT: Automatic Complexity and Termination Analysis of Integer Programs](https://arxiv.org/abs/2606.28542)
*Nils Lommen,Éléanore Meyer,Jürgen Giesl*

Main category: cs.LO

TL;DR: KoAT是一个自动推断整数程序复杂度上界并证明其终止性的工具，通过模块化交替推理运行时和大小上界，结合多种技术分析子程序，实验证明了其有效性。


<details>
  <summary>Details</summary>
Motivation: 自动分析程序的复杂度和终止性，特别是针对可能递归的整数程序，以提供形式化保证。

Method: 采用模块化交替推理方法，分别推断运行时上界和大小上界；使用多种技术组合（portfolio）来分析子程序。

Result: 通过大量实验评估，展示了该方法的有效性。

Conclusion: KoAT能够有效地自动推断复杂度上界并证明程序终止性，其实验结果支持了该方法的实用性。

Abstract: KoAT is a tool to automatically infer complexity bounds and prove termination of (possibly recursive) integer programs. To this end, KoAT implements an alternating modular inference of upper runtime and size bounds for program parts. In particular, KoAT uses a portfolio of different techniques to analyze subprograms. The power of our approach is demonstrated by an extensive experimental evaluation.

### [4] [Symbolon: Symbolic Execution by Learning Code Transformation](https://arxiv.org/abs/2606.29108)
*Jie Zhu,Penghui Li,Zhongxuan Li,Chihao Shen,Ziyang Li,Yizheng Chen,Kexin Pei*

Main category: cs.CR

TL;DR: Symbolon 是一个自动学习多样化代码变换并上下文敏感地应用它们以提升符号执行效率的框架。通过将变换发现建模为搜索问题，离线学习可复用的变换技能，在 32 个真实程序上平均行覆盖提升 3.69 倍，峰值内存和求解时间分别降低 29.2 倍和 123 倍，并在 Linux 内核中发现 21 个新漏洞。


<details>
  <summary>Details</summary>
Motivation: 符号执行面临路径爆炸、复杂约束等可扩展性问题。现有方法通过代码变换缓解问题，但变换过于僵化，无法利用多样化局部语义，且依赖为具体执行设计的编译器优化，与符号执行目标不一致。

Method: 将变换发现建模为程序表示上的搜索问题。离线在小程序上低成本学习变换，提炼为可复用的智能体技能库；在线由智能体根据目标上下文实例化这些技能，实现上下文敏感的代码变换应用。

Result: 在 32 个真实程序上，平均行覆盖提升 3.69 倍，峰值内存降低 29.2 倍，每查询求解时间降低 123 倍。在 Linux 内核中发现 21 个新漏洞并已上报。

Conclusion: Symbolon 通过自动化、上下文敏感的代码变换学习有效提升了符号执行的可扩展性和漏洞发现能力，验证了将变换发现作为搜索问题的可行性。

Abstract: Symbolic execution is a powerful program analysis technique with broad applications, such as vulnerability detection, security testing, and malware analysis. However, this technique is known to suffer from scalability issues, e.g., path explosion, complex constraints, due to certain structural and semantic patterns commonly presented in real-world programs. Existing approaches attempt to escape these patterns by transforming programs into new representations to reduce the execution cost. Unfortunately, these transformations are often too rigid to exploit diverse local program semantics and sometimes rely on compiler optimizations designed for concrete execution that may misalign with the goals of symbolic execution.
  We present Symbolon, a framework that automatically learns diverse code transformations and applies them context-sensitively to improve symbolic execution. Our key insight is to formulate transformation discovery as a search problem over program representations. To make the search practical, Symbolon learns transformations cheaply offline on small programs, distills them into a reusable library of agent skills, and uses an agent to instantiate these skills on repo-level targets. Our evaluation shows that Symbolon substantially improves the symbolic execution engine KLEE across 16 search strategies on 32 real-world programs, increasing line coverage by 3.69x on average while reducing peak memory and per-query solver time by 29.2x and 123x, respectively. When applied to the latest Linux kernel, Symbolon uncovers 21 previously unknown bugs, all of which have been reported to the kernel maintainers.

</details>
