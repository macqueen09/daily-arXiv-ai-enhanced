### [1] [Folding an e-graph in pure egglog](https://arxiv.org/abs/2607.01249)
*Zeng Ren,Maddy Bowers*

Main category: cs.PL

TL;DR: 本文探讨了在纯egglog中定义和实现catamorphism（一种通用的fold操作）的初步尝试，利用e-graph的memoization特性来减少递归计算中的冗余传播。


<details>
  <summary>Details</summary>
Motivation: 传统函数式编程中的折叠操作（catamorphism）在e-graph环境下可以实现自动memoization，并能通过e-classes而非具体子项来定义子问题，从而大幅减少计算量。

Method: 在纯egglog中定义catamorphism，并采用一种可复用的实现技巧来分阶段安排推理规则，避免更新值的重新传播。

Result: 成功在egglog中实现了catamorphism，并解决了值更新后重复传播的问题。

Conclusion: 这项工作展示了在e-graph中实现通用折叠操作的可行性，并为后续在egglog中高效实现递归计算提供了基础。

Abstract: Folding over a recursive data structure (also known as "catamorphism") is an essential operation in functional programming. It allows one to declaratively define a recursive computation by focusing on how solutions of subproblems are combined into the solution of the parent problem. Folding in the context of e-graphs can have several benefits. Not only do we get memoization for free, but also computation is drastically reduced since the subproblems are defined over e-classes as opposed to concrete sub-terms. This short report presents our initial attempt at defining and implementing catamorphism, a general notion of a fold, in pure egglog. We discuss a reusable implementation trick to stage the inference rules so that there is no re-propagation of updated values.

### [2] [Trustworthy Runtime Verification via Bisimulation (Extended Experience Report)](https://arxiv.org/abs/2607.01363)
*Ryan G. Scott,Ivan Perez,Alwyn E. Goodloe,Mike Dodds,Robert Dockins*

Main category: cs.PL

TL;DR: 本文介绍 CopilotVerifier，它为 Copilot 运行时监控框架生成的 C 代码提供形式化正确性证明，通过符号执行和 SMT 求解建立监控器与其编译输出之间的互模拟等价性，以适中的成本大幅提升编译器可信度。


<details>
  <summary>Details</summary>
Motivation: 在安全关键系统中，运行时监控代码必须行为正确。Copilot 框架自动生成 C 监控程序，但审计人员难以确信编译后的监控器不会崩溃且符合 Copilot 语义。需要一种方法为编译输出提供令人信服的正确性保证。

Method: CopilotVerifier 与 Copilot 编译器并行运行，利用 Crucible（LLVM 符号执行库）和 What4（SMT 求解器接口库）技术，生成以互模拟（bisimulation）形式呈现的正确性证明。该证明分解为一组验证条件，建立 Copilot 监控器与其编译输出在等价输入下产生等价输出，且崩溃条件一致。

Result: 实验结果表明，通过构建现有工具，可以以适中的成本显著提升编译器可信度，为最终生成令人类审计人员信服的形式化保证论证铺平道路。

Conclusion: CopilotVerifier 成功实现了对 Copilot 编译器输出代码的形式化验证，证明了运行时监控器与其编译形式的等价性（包括行为和崩溃一致性），为安全关键系统中的代码部署提供了更强的可信保证。

Abstract: When runtime verification is used to monitor safety-critical systems, it is essential that monitoring code behaves correctly. The Copilot runtime verification framework pursues this goal by automatically generating C monitor programs from a high-level DSL embedded in Haskell. In safety-critical domains, every piece of deployed code must be accompanied by an assurance argument that is convincing to human auditors. However, it is difficult for auditors to determine with confidence that a compiled monitor cannot crash and implements the behavior required by the Copilot semantics.
  In this paper we describe CopilotVerifier, which runs alongside the Copilot compiler, generating a proof of correctness for the compiled output. The proof establishes that a given Copilot monitor and its compiled form produce equivalent outputs on equivalent inputs, and that they either crash in identical circumstances or cannot crash. The proof takes the form of a bisimulation broken down into a set of verification conditions. We leverage two pieces of SMT-backed technology: the Crucible symbolic execution library for LLVM and the What4 solver interface library. Our results demonstrate that dramatically increased compiler assurance can be achieved at moderate cost by building on existing tools. This paves the way to our ultimate goal of generating formal assurance arguments that are convincing to human auditors.

</details>
