### [1] [Compiling Bioinformatics Recurrences](https://arxiv.org/abs/2607.06225)
*Bala Vinaithirthan,Shiv Sundram,Sneha Goenka,Fredrik Kjolstad*

Main category: cs.PL

TL;DR: FILTR是一个专为生物信息学递推算法设计的DSL及编译器框架，它将核心递推规则与剪枝和调度策略分离，自动生成优化的C++代码，性能可比肩甚至超越手工调优的实现（0.95x至30x加速）。


<details>
  <summary>Details</summary>
Motivation: 生物信息学中的动态规划算法（如序列比对、结构预测）在大规模数据上高效实现需要改变矩阵计算顺序并剪枝无效区域，但这些技术使实现变得复杂。作者希望设计一种DSL，让用户能独立描述递推规则、剪枝策略和调度策略，从而简化实现并加速新启发式方法的探索。

Method: 提出FILTR框架，包含一个DSL和编译器。DSL将核心递推规则与剪枝（近似限制DP矩阵计算区域）和调度（决定单元格迭代顺序）分离开来；编译器将这些高层描述编译为优化的C++代码。

Result: 在生物信息学基准测试中，FILTR生成的代码性能与手工优化的序列比对库相当，加速比在0.95x到30x之间。

Conclusion: 通过分离关注点（递推规则、剪枝、调度），FILTR既保证了性能竞争力，又大幅降低了实现复杂度，有助于快速探索新的启发式策略。

Abstract: Many bioinformatics algorithms, such as sequence alignment and structure prediction, can be expressed as recurrence equations over a dynamic programming matrix. Efficient implementations of these algorithms for large-scale biological data often require changing the order in which matrix cells are calculated and pruning ineffectual regions of the matrix from consideration altogether, but these techniques typically complicate implementation. We introduce FILTR, a domain-specific language (DSL) and compiler framework for bioinformatics recurrences. FILTR keeps the core recurrence rules separate from the pruning and scheduling strategies, where pruning acts as an approximation to limit where in the DP matrix cells are computed, and scheduling determines the iteration order for how cells are explored. FILTR compiles these high-level descriptions into optimized C++ code that matches the performance of hand-tuned implementations while enabling rapid exploration of new heuristics. FILTR is competitive with hand-optimized sequence-alignment libraries, ranging from 0.95x to 30x faster across biological benchmarks.

### [2] [CANONIC: Governance Is Compilation](https://arxiv.org/abs/2607.05410)
*Dexter Hadley*

Main category: cs.CY

TL;DR: 提出CANONIC框架，用编译器理论（语法/作用域解析/类型系统）对应三个公理（Triad/Inheritance/Introspection）来治理大模型生成内容。实验证明内容准入无法可靠过滤"slop"（不可靠内容），因此治理层的目标不是判断slop，而是建立可审计的证据账本。


<details>
  <summary>Details</summary>
Motivation: 大语言模型生成内容速度过快，产生了大量无法被及时核查的"slop"（牛津2025年度词汇），需要一种机制来规范内容准入。

Method: 借鉴编译器理论：三个公理（Triad、Inheritance、Introspection）分别对应编译器的语法层、作用域解析层和类型系统层；将内容准入转化为可判定的线性时间检查。

Result: 跨四个场景的预注册跨提供商基准测试显示：结构化的内容准入机制无法可靠区分可靠与不可靠内容——slop不是算法可计算的属性。

Conclusion: 治理层的核心目的不是判定slop（这需要领域专家判断），而是保持记录的可审计性：每个声明都锚定到定义、提交记录和证据窗口，实现端到端的可复现和可核查。

Abstract: We present CANONIC: governed intelligence that compiles digital artifacts into an evidence ledger at scale. Large language models generate prose faster than anyone can check it, the failure Oxford Languages named 'slop', its 2025 Word of the Year. CANONIC governs whether content may enter a corpus the way a compiler decides whether a program is well-formed: mechanically, by a grammar, at the boundary of admission. Governance reduces to three axioms (Triad, Inheritance, Introspection) that map one-to-one onto compiler theory's syntax, scope-resolution, and type-system layers, and admission is a decidable, linear-time check. We then ask, with a pre-registered cross-provider benchmark across four regimes, whether structural admission keeps slop out. It does not: no prose-reading gate reliably separates reliable from unreliable content. Slop is not a property an algorithm computes. It is a verdict of domain expertise. So a governance layer does not decide slop; it keeps the record auditable -- every claim anchored to a definition, a commit, and an evidence window, reproducible and checkable end to end.

### [3] [InvWeaver: Deductive Feedback for Invariant Synthesis in Interacting-Loop Programs](https://arxiv.org/abs/2607.05478)
*Guangyuan Wu,Weining Cao,Zehui Tan,Yuan Yao,Hengfeng Wei,Taolue Chen,Xiaoxing Ma*

Main category: cs.LG

TL;DR: InvWeaver是一个面向多循环程序的神经符号化循环不变式推理框架，通过循环级抽象、义务引导推理和最弱前置条件优化，在82个多循环基准中解决了72个，显著优于现有方法。


<details>
  <summary>Details</summary>
Motivation: 现有LLM辅助的猜测-验证技术在单循环程序上表现良好，但在处理包含多个交互循环的程序时效果不佳，因此需要一种能处理循环间依赖关系的推理方法。

Method: 提出InvWeaver框架，核心思路是通过循环级抽象暴露循环间依赖关系，结合义务引导推理和最弱前置条件优化来传播证明义务，从而实现多循环程序的不变式合成。

Result: 在82个多循环基准问题中解决了72个，同时在单循环任务上保持强性能，全面优于现有不变式推理方法。

Conclusion: InvWeaver有效解决了多循环程序的不变式合成难题，验证了神经符号化方法在处理复杂程序验证问题上的潜力。

Abstract: Loop invariant inference is a fundamental yet challenging problem in program verification. Recent LLM-aided guess-and-check techniques have shown strong performance on single-loop programs, but they often struggle with programs containing multiple interacting loops. This paper presents InvWeaver, a neuro-symbolic framework for synthesizing invariants for such programs. The key idea is to expose inter-loop dependencies and propagate proof obligations through a combination of loop-level abstraction, obligation-guided inference, and weakest-precondition-based refinement. We evaluate InvWeaver on a comprehensive benchmark suite, including a newly curated dataset derived from classic algorithms. Experimental results show that InvWeaver substantially outperforms existing invariant inference methods, solving 72 out of 82 multi-loop benchmark problems and maintaining strong performance on single-loop tasks.

### [4] [PatchOptic for Shared-State LLM Workflows with Projected Views and Verified Structured Updates](https://arxiv.org/abs/2607.05483)
*Zhaoyu Bai,Jiaqi Cai*

Main category: cs.LG

TL;DR: PatchOptic 提出了一种基于光学（optic）启发的接口，用于共享状态下的 LLM 工作流。它通过声明式的投影读取（projected reads）、授权写入区域（authorized write region）和补丁源区域（patch-source region），在局部更新与全局有效性之间建立契约，并通过运行时验证确保工作流步骤的合法性。实验表明，该方法能降低信息泄露和 token 成本，同时保持输出质量。


<details>
  <summary>Details</summary>
Motivation: 现有 LLM 工作流在结构化共享状态上运行时，通常使用渐进式披露（progressive disclosure）来展示当前步骤所需的状态片段。但现有方法（如 grep、RAG、AST 查询等）只解决了读取端的问题，却未定义局部写入修改后如何保证对全局状态的有效性。缺少一个“局部更新与全局有效性的契约”。

Method: 受光学（optic）概念的启发，提出三要素声明：①投影读取视图（projected read view），指定当前步骤可读的状态片段；②授权写入区域（authorized write region），指定可修改的范围；③补丁源区域（patch-source region），指定补丁可引用的来源。该声明支持运行时验证、路径级足迹用于子工作流组合与重排序证明。

Result: 在包含 46 个用例的 PatchBench 基准测试中，投影读取减少了信息泄露和 token 消耗，同时保持了强 actor 下的可接受输出质量。运行时验证能在提交前阻止违反声明的写入，补丁读取机制能拒绝使用隐藏来源的伪造补丁。

Conclusion: PatchOptic 通过光学启发的声明式接口，为 LLM 工作流中的局部更新提供了全局有效性保障，兼顾了安全性和效率，并为工作流的组合与重排序提供了理论基础。

Abstract: Agentic workflows often operate over shared, structured state. Because LLM context windows are limited, each model invocation is typically shown only the state fragment needed for the current workflow step, a pattern commonly known as progressive disclosure. Modern systems construct such model-facing views using grep-like keyword search, retrieval-augmented generation (RAG), abstract-syntax-tree (AST) queries, and task-specific agent skills. These methods make the read side manageable, but they do not define when a locally proposed rewrite is valid after it is applied back to the full state. The missing piece is a contract between local updates and global validity. We introduce PatchOptic, an optic-inspired interface for shared-state LLM workflows. Optics are compositional bidirectional accessors that describe how views of structured data are read and updated. PatchOptic borrows this view/update intuition and realizes it through projected reads and verified structured patches. Each workflow step declares a projected read view, an authorized write region, and a patch-source region. Beyond runtime enforcement, the same declaration yields a path-level footprint that supports delegation, sub-workflow composition, and static certificates for reordering independent steps within the same phase. We evaluate this design with PatchBench, a benchmark with 46 cases across domains. The results show that projected reads reduce reported leakage and token cost while preserving accepted-output quality under the strong actor. Runtime verification blocks declared workflow-contract violations before commit, and patch-read enforcement rejects compromised patch artifacts that use hidden sources.

</details>
