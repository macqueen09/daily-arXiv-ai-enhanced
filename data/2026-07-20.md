TL;DR: 本文提出了展开表达式和纯函数在符号执行的静态验证器中的形式语义，并将其扩展至渐进验证器（如Gradual C0），同时给出了可靠性证明。该成果同样适用于基于Silicon后端的Viper系列静态验证工具（Gobra、Nagini、SnaKt）。

Abstract: Unfolding expressions, which temporarily unfold a predicate to leverage its owned fields when evaluating a heap-dependent expression, and pure functions, which are heap-dependent functions that can be used in specifications, are used in deductive program verifiers based on implicit dynamic frames, such as Gradual C0, Gobra, Nagini, and SnaKt, to increase the modularity of specifications involving ownership. In this paper, we present the formal semantics for unfolding expressions and pure functions for a static verifier using symbolic execution, extend it for a gradual verifier, and provide a proof of soundness. To support Gradual C0, our proof is in the setting of gradual verification, a deductive program verification system that combines static and dynamic verification to allow partial specifications. However, because the gradual verifier is a conservative extension of a static verifier, our results also apply to static verifiers that use symbolic execution, such as the Silicon symbolic execution backend for the Viper verification infrastructure used by Gobra, Nagini, and SnaKt.

### [2] [Programming with Quantum-Controlled Quantum Channels](https://arxiv.org/abs/2607.15873)
*Kengo Hirata,Takeshi Tsukada*

Main category: cs.PL

TL;DR: 本文提出了一种新的量子编程语言，通过线性类型系统解决了量子控制操作中的"对应问题"，能够安全地表达量子SWITCH等程序叠加机制。


<details>
  <summary>Details</summary>
Motivation: 经典比特与量子比特的差异引发了"能否不仅叠加数据，还能叠加程序"的思考。量子SWITCH作为一种程序叠加实例已被广泛研究，但传统的受控操作在通用量子信道上的定义存在良定义性问题。

Method: 通过语义分析识别出受控操作不良行为的根源是"对应问题"——量子条件分支中then分支和else分支的测量缺乏协调。提出一种线性类型系统来强制对齐两个分支中的量子操作，从而构建出能够表达量子SWITCH的量子控制与测量编程语言。

Result: 开发了一种新颖的量子编程语言，支持量子控制和测量，能够表达量子信道上的量子SWITCH；通过线性类型系统解决了对应问题，保证了语言的良好行为特性。

Conclusion: 受控操作与量子SWITCH应被视为不同类型的量子控制机制；线性类型系统是确保量子条件分支中测量协调的有效手段，为量子编程语言的设计提供了新方向。

Abstract: In contrast to a classical bit, which can only take the value $0$ or $1$, its quantum counterpart -- a qubit -- can exist in a superposition of $0$ and $1$. This is a superposition of data values, naturally raising the question of whether one can superpose not only data but also programs. For example, a particular superposition of programs, known as the quantum SWITCH, has attracted much attention, and its implementations and computational advantages have been studied extensively within the physics community.
  A naive way to control a program by a qubit is by means of a controlled operation. Given an operation $F$, this amounts to considering an operation that behaves as $F$ when the control qubit is $|1\rangle$, and as the identity operation when the control qubit is $|0\rangle$. This idea works well when $F$ is a unitary operation, but it is not well-defined for a general quantum channel. By contrast, the quantum SWITCH is free from the well-definedness issue. This contrast leads to the key insight of this paper: controlled operations and the quantum SWITCH should be regarded as different kinds of quantum control mechanisms.
  Building on this insight, we develop a novel quantum programming language with quantum control and measurement that can express the quantum SWITCH over quantum channels. Using a semantic analysis based on program transformations, we identify the source of the ill-behavedness of controlled operations as the \emph{correspondence problem}: a lack of coordination between the measurements performed in the then- and else-branches of quantum conditional branching. We address this problem with a linear type system that enforces alignment of the quantum operations used in the two branches, yielding a well-behaved language capable of expressing the quantum SWITCH.

### [3] [Yarrow: Reconciling Effects Handlers and Region-Based Memory Management](https://arxiv.org/abs/2607.15876)
*Anders Alnor Mathiasen,Amin Timany,Lars Birkedal*

Main category: cs.PL

TL;DR: 本文提出了一种名为Yarrow的新编程语言，将代数效应与基于区域的内存管理相结合。针对代数效应破坏区域内存管理的栈式调用纪律这一挑战，设计了Yarrow Logic（YL）程序逻辑，支持对单次和多次效应处理器的区域进行安全模块化推理。通过OCaml启发的操作语义证明YL逻辑的可靠性，并在多个案例（检查点、异步计算、LIFO数据结构）中验证了正确性，所有案例均使用区域分配内存而非垃圾回收堆。已在Rocq Prover中通过Iris分离逻辑框架形式化实现。


<details>
  <summary>Details</summary>
Motivation: 代数效应的非局部控制流会破坏函数调用/返回的栈式纪律，而多次效应处理器则打破了"区域只能被退出一次"的不变量，这使得将代数效应与基于区域的内存管理结合面临根本性挑战。现有方法难以同时保证内存安全与效应处理的灵活性。

Method: 设计Yarrow语言（ML风格+代数效应+区域管理），开发Yarrow Logic（YL）程序逻辑，该逻辑基于OCaml运行时启发的操作语义，利用Iris分离逻辑框架在Rocq Prover中形式化，支持对单次和多次效应处理器的区域安全推理。

Result: YL逻辑相对于操作语义被证明是可靠的；在检查点、异步计算和LIFO数据结构三个案例研究中，证明了使用区域分配内存的程序正确性，避免了垃圾回收堆的开销。

Conclusion: 代数效应与基于区域的内存管理可以安全共存，YL逻辑提供了模块化推理能力，使得在保持内存效率的同时享受代数效应的表达力成为可能。

Abstract: We present a new ML-like programming language Yarrow with algebraic effects and region-based memory management. Reconciling these programming language features into one language is challenging: the non-local control flow of algebraic effects break the stack discipline of function calls and returns that region-based memory management relies on, and multi-shot effect handlers break the invariant that regions can be exited at most once. We present a program logic, called Yarrow Logic (YL), that supports safe and modular reasoning about regions in the presence of one-shot and multi-shot effect handlers. We prove the logic sound w.r.t. the operational semantics of Yarrow which is inspired by the runtime of OCaml but refined for regions. We use YL to prove correctness of a number of case studies with algebraic effects, including checkpointing, asynchronous computation and a LIFO data structure implementation. Since all memory locations used in these case studies are allocated in regions, these case studies avoid using the less efficient garbage collected heap memory. We have formalized Yarrow's operational semantics, the Yarrow program logic, and all our case studies using the Iris separation logic framework on top of the Rocq Prover.

### [4] [Causality in Pure Quantum Computation with Quantum Control](https://arxiv.org/abs/2607.15926)
*Kengo Hirata,Takeshi Tsukada*

Main category: cs.PL

TL;DR: 本文提出了一个结合量子控制和高阶函数的类型化λ演算，并赋予其基于直觉主义BV逻辑的类型系统以强制执行因果性，通过构建与Caus构造相关的新模型，证明了一些物理上不可实现的过程在该语言中不可定义。


<details>
  <summary>Details</summary>
Motivation: 量子计算中存在不确定因果序现象（如量子SWITCH可实现而OCB过程不可实现），这种可实现性差异通常归因于物理因果性约束。然而缺乏一个形式化框架来严格区分哪些不确定因果序过程是因果合法的。

Method: 提出一个扩展了纯量子计算的高阶类型化λ演算，支持量子条件分支和高阶函数；采用直觉主义BV逻辑作为类型系统以强制执行因果性约束；构建了一个与Casual构造密切相关的新范畴论模型。

Result: 在新模型中证明了某些已知物理上不可实现的过程（如OCB）在该语言中不可定义，表明类型系统能够有效捕获物理因果性约束。

Conclusion: 通过高阶类型系统与范畴论模型，为量子计算中因果性约束的刻画提供了形式化工具，有助于理解哪些不确定因果序过程在物理上是可实现的。

Abstract: Indefinite causal order is a characteristic phenomenon in quantum computation, with examples including the quantum SWITCH and the OCB process. Not all such processes are believed to be physically realizable: while some implementations of the quantum SWITCH have been proposed, the OCB process is suspected to be unrealizable. This difference in realizability is commonly attributed to constraints imposed by physical causality.
  This paper studies such a causality issue in a higher-order setting, proposing a typed lambda calculus with quantum control and its categorical semantics. Our calculus extends pure quantum computation with higher-order functions and quantum conditional branching, and it is equipped with a type system based on intuitionistic BV logic to enforce causality. We also present a novel model that is closely related to the Caus construction, by which we prove that some physically-unrealizable processes are not definable in our language.

</details>


### [5] [A Modular Framework for Stack-Heap and Value Abstractions (Extended Version)](https://arxiv.org/abs/2607.15932)
*Giacomo Boldini,Luca Negrini,Luca Olivieri,Pietro Ferrara*

Main category: cs.PL

TL;DR: 本文提出并形式化了一个通用的内存分析框架，基于抽象解释理论，通过分离值和内存分析为两个模块化抽象域，支持多种编程语言（C、C++、Java、Python）的栈和堆内存行为分析，能够准确检测缓冲区溢出、释放后使用、空指针异常等内存错误。


<details>
  <summary>Details</summary>
Motivation: 高级静态程序分析需要处理指针、复杂数据结构、函数和对象等非平凡程序行为的语义，以及它们如何影响内存。不同编程语言的内存分配和管理方式差异显著，而准确的内存模型对在程序执行前检测缓冲区溢出、释放后使用错误和空指针异常等关键问题至关重要。现有方法缺乏一个既能支持多种语言行为又保持参数化的通用框架。

Method: 基于抽象解释理论，提出分裂状态抽象（split state abstraction），将值分析和内存分析分离为两个模块化抽象域。它们通过一组内存标识符及域定义的操作用于操作这些标识符进行交互，从而同时捕获值信息和结构化的内存关系。该框架支持栈和堆内存管理，可参数化地选择不同内存和值分析组合。

Result: 形式化定义了一个通用的内存框架，能够支撑C、C++、Java和Python等不同编程语言的内存行为分析，并支持不同的内存模型和值分析独立选择与组合，实现了对多种程序行为的内存安全问题的准确近似分析。

Conclusion: 提出的通用内存框架通过模块化的抽象域设计，在保持参数化灵活性的同时，能够准确近似不同编程语言的内存模型和行为，为静态分析中检测内存相关错误提供了理论基础。

Abstract: Advanced static program analysis requires reasoning on the semantics of non-trivial program behaviors (e.g., pointers and complex data structures such as lists and sets, functions, and objects) and how they affect the memory. In most programming languages, static and dynamic allocations are typically managed by the stack and the heap, respectively. However, how allocations behave and how the memory is managed at runtime can vary significantly depending on the programming language being analyzed. Proper handling of these aspects is essential, as an accurate memory model enables the detection of critical issues such as buffer overflows and underflows, use-after-free errors, and null pointer exceptions prior to execution, that is, before such erroneous behaviors occur. In this paper, we propose and formalize a generic memory framework to handle stack and heap memory during the analysis, that is able to support various behaviors from different programming languages (e.g., C, C++, Java, and Python), while remaining parametric, allowing different memory and value analyses to be independently chosen and combined. It relies on the Abstract Interpretation theory and enables sound approximation of different memory models and program behaviors. We introduce a split state abstraction that separates value and memory analyses into two modular abstract domains. These domains interact through a set of memory identifiers, along with a set of operations defined by the domains to manipulate them, allowing the framework to capture both value information and structural memory relationships.

</details>


### [6] [Bidirectional Typing with Freezing, Skeletons, and Ghosts](https://arxiv.org/abs/2607.16061)
*Wenhao Tang,Shengyi Jiang,Aghilas Y. Boussaa,Sam Lindley,Bruno C. d. S. Oliveira*

Main category: cs.PL

TL;DR: Fresco是一种新型双向类型推断方法，通过skeletons和ghosts实现函数与参数间的灵活信息流动，并允许用户通过freezing自定义方向，从而支持一等多态的预测性推断。


<details>
  <summary>Details</summary>
Motivation: 传统的双向类型检查仅支持单向信息流（函数到参数），难以推断一等多态；现有方法要么对混合信息流支持有限，要么需要ad hoc机制损害可预测性。

Method: 提出Fresco方法，利用skeletons（骨架）让类型信息在函数与参数间来回流动，用ghosts表示未知类型信息，并通过freezing机制允许用户自定义信息流动方向。

Result: 提供了Fresco的声明式规范、一个简单且相对于声明系统sound和complete的类型推断算法，以及一个可进一步推广到推断模态效应类型的原型实现。

Conclusion: Fresco的灵活信息流使得一等多态的推断既具表现力又具可预测性。

Abstract: Bidirectional typing makes use of local information flow between functions and arguments. Conventional bidirectional typing only supports unidirectional information flow, typically from functions to arguments, which is insufficient to infer first-class polymorphism. Existing work on improving information flow either has limited support for mixed information flow or requires ad hoc mechanisms that harm predictability.
  We propose Fresco, a novel bidirectional type inference approach to first-class polymorphism. Fresco enables local type information to flow back and forth between functions and arguments via skeletons, with ghosts representing unknown type information, and allows users to customise the direction of information flow by freezing. The flexible information flow of Fresco enables expressive and predictable inference for first-class polymorphism. We provide a declarative specification for Fresco, a simple type inference algorithm that is sound and complete with respect to the declarative system, and a prototype implementation that further generalises Fresco to infer modal effect types.

</details>
